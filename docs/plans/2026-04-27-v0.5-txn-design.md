## 1. Goal & framing

Persistence OS today ships four-of-six Clojure design pillars: immutable values + reified time (`fact`), code-as-data (`plan` parse-side), boundary contracts (`spec`), and algebraic-effect handlers with audit (`effect`), plus counterfactual replay (`replay`) and v0.4.0a1 substrate primitives (Dispatcher, Provenance, fork, causal_history). The two missing pillars are:

1. **Transactional shared state** — Clojure's `ref` + `dosync` (Software Transactional Memory)
2. **Live-programming surface** — Clojure's REPL (production-steering interface)

This design covers the first. v0.5.0a1 ships a full Clojure-shaped STM (refs + `dosync` + `commute` + `ensure` + effect-intents) over the existing bitemporal datom log, without modifying any other module.

**Why this release is paper-protective:** STM is orthogonal to Propositions 1–5 — branch (Prop 1), well-formedness (Prop 2), tamper-evidence (Prop 3), plan AST (Prop 4), and replay no-op (Prop 5) all hold without modification. The next paper revision (v0.9, post-NeSy notification) adds *one* sentence noting the v0.5 extension; R3 verifies zero proposition impact.

**What this release unlocks for AI Box product surfaces:**
- Adaptive Trader v2 — parallel signal aggregation via `tx.commute(signals_ref, sum-into, ...)` without lost increments
- GuestFlow per-property workers — atomic `dosync` over conversation-state + escalation-flag refs
- BankabilityAI memo + status pipeline — atomic memo-row + status-row updates
- Insurance Comparator (Trottinette) — cart updates inside `dosync`
- Juba OS / Agent Teams — coordinator merges via `txn`-managed shared state
- Memory Palace bitemporal retrofit — every `/vault/remember` becomes a transactional commit, not a manual lock

## 2. Architecture

`persistence.txn` is a single new module added alongside `fact`, `effect`, `spec`, `replay`, and `plan`. It does not modify any existing module. Only `fact.db.DB` gains four new methods (`ref`, `new_ref`, `dosync`, `dosync_decorator`) added in a thin `_db_extension.py` that imports at module init.

**Runtime model:**
- A **`Ref`** is an immutable handle to an entity-id `(eid, db_id_or_branch_id)` with `__hash__` and `__eq__` over `eid`.
- A **`Transaction`** is the object passed into a `dosync` body. It captures:
  - `t_start` — snapshot tx-time
  - `read_set` — refs the body has dereferenced
  - `write_set` — pending `tx.assoc` writes (dict)
  - `commute_set` — pending `tx.commute` deltas (dict of ref → list of (fn, args))
  - `ensure_set` — refs declared read-conflict-sensitive without dereffing
  - `effect_intent_log` — effect calls queued during the body
- On exit, the transaction validates against the current log (any datom on `read_set ∪ ensure_set ∪ write_set.keys()` with `tx > t_start`?). If clean, it atomically applies write_set + commute_set + effect_intent_log as one `db.transact()` call. If dirty, it discards the changeset, increments retry count, re-snapshots, and re-runs the body.

**Six invariants the protocol enforces:**

1. **Snapshot isolation.** Every `tx.deref` reads from `db.as_of(t_start)` — no in-flight writes are visible.
2. **Read-write conflict detection.** Any datom on a read-set entity with `tx > t_start` triggers retry.
3. **Write-write conflict detection.** Same query covers writes (writes are datoms in the same log).
4. **Commute exemption.** Refs in `commute_set` but NOT in `read_set` skip the read-write conflict check; their writes still go through atomic commit. Two concurrent `tx.commute(counter, inc_by, 1)` calls both succeed, counter ends up at +2, neither retries.
5. **Effect-intent atomicity.** Effects fire only after commit succeeds. If the commit fails the conflict-check, no effect ever fired. Counterfactual replay sees the commit datom and intent-log datoms together as one unit.
6. **Replay determinism.** Bounded-retry `dosync` blocks are byte-identical across two runs of the same body. Deadline-using transactions get tagged `:txn/non-deterministic-retry true` so replay tooling can flag them.

All transactional state changes are themselves datoms. Replay, `branch()`, `fork()`, and `causal_history()` work on transactional state without modification — refs are just entities, transactions are just multi-datom transactions with a coordination protocol on top.

```
├── intents.py           # Effect-intent envelope dataclass + replay-at-commit helper
├── commutes.py          # Commute-fn registry + starter set (inc-by, sum-into, set-union, dict-merge-shallow)
├── errors.py            # TxnRetryExhausted, TxnDeadlineExceeded, RefBranchMismatch, EffectInIoBlock, CommuteFnNotRegistered
└── _db_extension.py     # DB.ref() / DB.new_ref() / DB.dosync() / DB.dosync_decorator() — imported by fact/db.py
```

`fact/db.py` gains a 6-line bottom-of-file import block:
```python
from persistence.txn._db_extension import _attach_txn_methods
_attach_txn_methods(DB)
```

This keeps `txn` a clean module with **one inbound coupling-point to `fact`** and **zero outbound coupling** to `effect`/`replay`/`plan`/`spec` (they all consume `txn` later, none of them are dependencies). Tests live at `tests/persistence/txn/` mirroring the file structure.

## 4. Data model & datom shape

Three new datom attribute clusters ship in v0.5.0a1, all under the `:persistence.txn/*` namespace. **No changes to existing attributes.** `PLAN_CANONICAL_VERSION` stays at 1.

### 4.1 Per-commit datom (one per successful `dosync`)

```
:persistence.txn/commit-id              UUID,    identity
:persistence.txn/started-at             inst                  (t_start snapshot)
:persistence.txn/committed-at           inst                  (commit tx-time)
:persistence.txn/retry-count            long                  (attempts before success)
:persistence.txn/read-set               [ref :many]           (entity-ids dereferenced)
:persistence.txn/ensure-set             [ref :many]           (entity-ids tx.ensure'd)
:persistence.txn/non-deterministic-retry boolean              (true iff deadline= was used)
:persistence.txn/intent-log             [ref :many]           (refs to post-commit audit-chain entries; populated AFTER intents fire)
```

### 4.2 Per-write datoms (one per `tx.assoc`)

Regular entity datoms as you'd write through `db.transact()` today, with one extra attribute pinning them to the transaction:

```
:persistence.txn/commit                 ref → commit-id       (which transaction wrote this)
```

### 4.3 Per-commute datoms (one per `tx.commute`)

Identical to per-write, plus:

```
:persistence.txn/commute-fn             symbol                (canonical-name fn applied)
:persistence.txn/commute-args            [any :many]           (positional args passed; list, possibly empty)
```

### 4.4 Effect-intent envelopes

Not separate datoms during the body. They fire through the existing `effect` handler stack at commit-time as if they were called directly, with their resulting audit-chain entries pinned to the commit-id via a new audit field:

```
:effect/txn-commit                      UUID | nil            (commit-id this effect fired under)
```

This is a single new field on the existing `AuditEntry` schema; it stays nullable for backwards-compat with non-transactional effect calls.

### 4.5 Spec registration

All `:persistence.txn/*` attributes register in the existing `spec` registry at module init, alongside `:persistence.plan/node` etc., to keep parse-don't-validate discipline. Spec definitions live in `txn/__init__.py` and call `persistence.spec.registry.register(...)`.

## 5. Public API contract

```python
# Constructors
db.ref(eid: str) -> Ref                                # handle to existing/future entity
db.new_ref(initial: dict | None = None) -> Ref         # uuid7-allocated; optional initial assoc

# Transaction entry — both forms
with db.dosync() as tx: ...                            # context-manager sugar
with db.dosync(deadline=5.0) as tx: ...                # opt-in wall-clock deadline
with db.dosync(max_retries=64) as tx: ...              # override default retry cap (32)

@db.dosync                                             # decorator — canonical retryable form
def transfer(tx): ...

@db.dosync(deadline=2.0, max_retries=64)               # decorator with options
def transfer(tx): ...

# Transaction body API
tx.deref(ref: Ref) -> dict | None                      # snapshot read; adds to read_set
tx.assoc(ref: Ref, **attrs) -> None                    # queue write
tx.commute(ref: Ref, fn: str, *args) -> None           # queue commutative write (no read-conflict)
tx.ensure(ref: Ref) -> None                            # add to ensure_set without dereffing
tx.effect(op: str, **kwargs) -> None                   # queue effect intent for commit-time fire

# Errors (all subclass TxnError)
class TxnRetryExhausted(TxnError): ...                 # bounded retry cap hit
class TxnDeadlineExceeded(TxnError): ...               # wall-clock deadline elapsed
class RefBranchMismatch(TxnError): ...                 # ref from another DB used here
class CommuteFnNotRegistered(TxnError): ...            # commute fn symbol unknown
class EffectInIoBlock(TxnError): ...                   # raw effect call inside dosync
class NestedDosyncNotSupported(TxnError): ...          # nested dosync — explicit error in v0.5.0a1
```

### 5.1 Commute function registry

Each `tx.commute(ref, fn, arg)` records `:persistence.txn/commute-fn` as a canonical-name symbol. A registry maps symbols to actual functions; functions must be deterministic and provably commutative.

**v0.5.0a1 starter set (4 functions):**

| Symbol | Signature | Semantics |
|---|---|---|
| `persistence.txn.commutes/inc-by` | `(value: int, n: int) -> int` | Numeric increment by n |
| `persistence.txn.commutes/sum-into` | `(value: list, items: list) -> list` | Append items (order-independent for downstream commutes) |
| `persistence.txn.commutes/set-union` | `(value: set, items: set) -> set` | Set union |
| `persistence.txn.commutes/dict-merge-shallow` | `(value: dict, items: dict) -> dict` | Shallow dict merge (last-write-wins on conflict — caller responsibility) |

Custom commutes register at module init time via `persistence.txn.commutes.register(symbol, fn)`. Unknown symbols at commit raise `CommuteFnNotRegistered`.

### 5.2 Effect intent shape

Effect intents capture `(handler_id, canonical_call, args)` — the same Provenance-typed shape v0.4.0a1 introduced. On commit, intents replay through the existing handler stack in queue order; on retry they're discarded. Inside the body, calling raw `effect(...)` (without `tx.effect`) raises `EffectInIoBlock` — enforced by a thread-local guard.

## 6. Conflict-detection & commit protocol

```python
def _run(db, body, max_retries, deadline):
    attempt = 0
    deadline_at = time.monotonic() + deadline if deadline else None
    while True:
        if attempt >= max_retries:
            raise TxnRetryExhausted(...)
        if deadline_at and time.monotonic() >= deadline_at:
            raise TxnDeadlineExceeded(...)

        t_start = db.now()
        tx = Transaction(db=db, t_start=t_start, attempt=attempt)
        try:
            with _txn_thread_local(tx):    # blocks raw effect() calls
                body(tx)
        except TxnRetryRequested:
            attempt += 1
            continue
        # Other exceptions escape — no commit, no retry, no effects fire.

        # Commit gate: MVCC conflict check
        touched = (tx.read_set
                   | tx.ensure_set
                   | set(tx.write_set.keys())
                   | (set(tx.commute_set.keys()) - {r for r in tx.commute_set
                                                    if r not in tx.read_set}))
        # Note: refs in commute_set that are NOT in read_set skip the conflict check.
        conflicts = db.query_datoms_since(t_start, entity_ids=touched)
        if conflicts:
            attempt += 1
            continue

        # Apply atomically as one db.transact()
        datoms = (tx._build_write_datoms()
                  + tx._build_commute_datoms()
                  + tx._build_commit_datom())
        db.transact(datoms)
        # Replay effect intents through real handler stack — pinned to commit_id
        for intent in tx.effect_intent_log:
            effect_handler.dispatch(**intent.args, _txn_commit=tx.commit_id)
        return tx.commit_id
```

**`tx.commute` semantics:** the queued `(fn, arg)` is applied at commit time against the latest committed value — not the snapshot value. That's what makes commutes contention-free.

**`tx.ensure` semantics:** adds a ref to the conflict-check set without reading it. Lets you say "I depend on this ref *not* changing" without paying the deref cost.

**Exceptions:** any exception escapes the `dosync` immediately; no commit, no retry, no effects fire. The transaction's pending changeset is dropped.

**Defaults:**
- `max_retries=32`
- `deadline=None` (bounded-retry only by default)
- Commute starter set registered at module init

## 10. Deferred to post-freeze

Explicitly out of scope for v0.5.0a1.

| Deferred | Target tag | Scope sketch | Why deferred |
|---|---|---|---|
| **Atoms (single-cell CAS)** | v0.5.1 | `db.atom(eid)` + `swap!`-like API; degenerate case of refs (single-cell `dosync`) but with a faster lock-free path | Ergonomic shortcut, not a missing primitive |
| **Agents (async ordered single-cell)** | v0.5.2 | `db.agent(eid)` + `send` / `send-off`; Clojure's async-ordered model; sits naturally on top of `effect` handlers | Async model; clean post-paper sprint |
| **Nested `dosync`** | v0.5.3 | v0.5.0a1 raises `NestedDosyncNotSupported`; choice between flatten-into-outer or keep prohibited | Needs design pass |
| **Multi-process STM** | v0.6.x | Cross-process refs, distributed conflict detection | Requires storage-side conflict-check API; not load-bearing for any current AI Box surface |
| **Distributed STM** | future | Network partitions, eventual consistency model | Speculative; no driver |

### 10.1 Atom sketch (v0.5.1)

```python
counter = db.atom(":metrics/login-count", initial=0)
new_value = counter.swap(lambda v: v + 1)
@v = counter.deref()
```

Single-cell CAS over a datom-backed entity. Degenerate case of `dosync(...)` with one ref — but a lock-free fast-path skips the full transaction protocol. Useful for hot counters where building a full Transaction object is overhead.

### 10.2 Agent sketch (v0.5.2)

```python
log_agent = db.agent(":logs/agent-1", initial=[])
log_agent.send(lambda v: v + ["new entry"])    # async, ordered
log_agent.send_off(lambda v: ...)              # async, IO-allowed, ordered per-agent
```

Async-ordered single-cell. Sends are queued per-agent, applied in order, never block the sender. Sits naturally on top of `effect` handlers (`:agent/send` becomes a new effect-catalog op). The replay story is interesting and worth its own design pass.

### 10.3 Nested dosync sketch (v0.5.3)

Decision: flatten-into-outer (Clojure's actual behavior — inner `dosync` is a no-op, outer transaction's read/write/commute sets absorb the inner body) **or** keep `NestedDosyncNotSupported` as a permanent error to force callers to refactor. Needs design pass before deciding.

## 13. Risks & mitigations

| Risk | Severity | Mitigation |
|---|---|---|
| Concurrency-bug class hides in retry semantics | High | R2a scoped to this surface; Hypothesis property tests mandatory in Phase D; INT phase exercises real-thread parallel transactions |
| Effect-intent atomicity violation under retry | High | Thread-local guard raises `EffectInIoBlock` on raw-`effect` calls inside `dosync`; tests assert "0 intents fire on conflict-failed commit"; audit-chain pinning via `:effect/txn-commit` |
| Schedule slip past 2026-05-15 | Medium | ~3 weeks margin (today 2026-04-26); fallback: drop Phase D commute starter-set 4 → 2 (`inc-by`, `dict-merge-shallow`); other commutes deferred to v0.5.1 |
| `commute` against the latest committed value (not snapshot) surprises callers | Medium | Documented prominently in `tx.commute` docstring + `test_conflict.py` includes a "commute reads latest, not snapshot" test that demonstrates the semantics |
| `db.transact()` partial failure mid-commit corrupts atomicity | Medium | Verify `db.transact()` is single-tx atomic in `fact/db.py` (it is — confirmed via reading lines 137–286); add INT test asserting partial-failure rollback |
| Paper revision (v0.9) introduces proposition wording drift | Low | R3 passes BEFORE tag; no v0.9 paper revision until post-NeSy notification |
