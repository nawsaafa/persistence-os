### A1 — `execute()` core + result envelope

**Files:** `src/persistence/plan/_execute.py` (NEW), `src/persistence/plan/__init__.py`, `src/persistence/plan/_errors.py`
**Tests:** `tests/plan/test_execute_result_envelope.py`, `tests/plan/test_execute_no_dspy_dep.py`
**Lines (estimate):** ~120 LOC + ~150 LOC tests

#### Public API (must match exactly)

```python
@dataclass(frozen=True, slots=True)
class LeafResult:
    node_id: str          # 32-hex-char content-addressed leaf id
    tag: str              # ":llm-call", ":tool-call", ":transact"
    handler_id: str       # registered handler key (caller-supplied via env or registry)
    result: Any           # handler return value (may be None)


@dataclass(frozen=True, slots=True)
class FailureInfo:
    failed_node_id: str
    failed_tag: str
    error_class: str      # type(exc).__name__
    error_repr: str       # repr(exc) — NOT str(exc)


@dataclass(frozen=True, slots=True)
class ExecutionResult:
    plan_id: str          # node.id of the root
    status: Literal["ok", "failed"]
    leaf_results: tuple[LeafResult, ...]   # tuple, frozen
    failure: FailureInfo | None  # populated iff status == "failed"


def execute(plan: Node, dispatcher: Dispatcher, *, env: dict | None = None) -> ExecutionResult: ...
```

#### Semantics

- Walks `plan` via `Dispatcher.dispatch` (which calls `_walk.walk` internally).
- Per-node: if dispatcher has a registered handler, capture its return as a `LeafResult`. `handler_id` is read from `node.attrs.get("handler-id", "<default>")` if the env doesn't pin one — refine in code review.
- On handler raise: halt, record `FailureInfo`, return `status="failed"` with partial `leaf_results`.
- On `UnimplementedNodeKindError` from walker: re-raise (do NOT swallow — caller's contract is broken).

#### Edge cases the tests MUST cover

1. Empty plan (root with no children, no handler) → `status="ok"`, `leaf_results=()`.
2. Single `:llm-call` leaf → `leaf_results` has 1 entry with the right `node_id`.
3. Nested DAG (`:seq` containing two `:llm-call`) → walk-order preserved.
4. Handler raises ValueError → `status="failed"`, `failure.error_class == "ValueError"`, partial `leaf_results` preserved.
5. Walker raises `UnimplementedNodeKindError` for `:code` leaf → `execute()` re-raises (does NOT wrap).
6. **No-DSPy import test:** subprocess `python -c "import sys; sys.modules['dspy']=None; import persistence.plan._execute"` succeeds. (Use `subprocess.run([sys.executable, "-c", ...], check=True)` to fence module-import side-effects.)

#### `_errors.py` extension

```python
class OptimizerNotAvailable(ImportError):
    """`optimize()` invoked but optional dep unavailable."""

    """`MetricRef` lookup miss in the metric registry."""

class GateFailure(RuntimeError):
    """A promotion gate (G1/G2/G3/G4) returned False; raised by `promote()`."""
```

#### `__init__.py` extension

Add to `__all__`: `ExecutionResult`, `LeafResult`, `FailureInfo`, `execute`. Re-export from `persistence.plan`.

#### Exit gate

- Suite green: 931 + N (N = added tests) passed + 7 xfailed.

### A2 — `MetricRef` + `_metric_registry` + `_canonicalize_training_set`

**Files:** `src/persistence/plan/_metric_registry.py` (NEW), `src/persistence/plan/_execute.py` (extend with canonicalizer)
**Tests:** `tests/plan/test_metric_registry.py`, `tests/plan/test_training_set_canonicalize.py`
**Lines:** ~80 LOC + ~120 LOC tests

#### Public API

```python
# _metric_registry.py
class MetricRef(NamedTuple):
    id: str
    version: str

def register_metric(
    ref: MetricRef,
    fn: Callable[[ExecutionResult, dict], float],
    *,
    replace: bool = False,
) -> None: ...

def lookup_metric(ref: MetricRef) -> Callable[[ExecutionResult, dict], float]: ...
    # Raises MetricNotRegistered on miss

def unregister_metric(ref: MetricRef) -> None: ...  # test-only
```

```python
# _execute.py (extension)
def _canonicalize_training_set(
    training_set: list[TrainingExample],
) -> str:  # returns sha256 hex
    """Canonical hash of training set per design §5 rule (1).
    1. Validate schema: only "inputs" (dict) + "expected" (Any) keys.
    2. Encode each example with json.dumps(sort_keys=True, separators=(',',':'), allow_nan=False).
       (Same canonical-JSON rule as Node.id.)
    3. Sort canonicalized examples lexicographically.
    4. Join with '\n', utf-8 encode, sha256, return hexdigest.
    No callable hashing — only canonical-JSON bytes feed the digest.
    """
```

`TrainingExample` is `TypedDict("TrainingExample", {"inputs": dict, "expected": Any})`.

#### Semantics

- `register_metric` raises `ValueError` on collision unless `replace=True`.
- Empty training set returns sha256 of empty string (`e3b0c44...`).
- Schema violation raises `ValueError` with the offending example index.

#### Edge cases the tests MUST cover

1. `register → lookup` round-trip.
2. Re-register same ref without `replace=True` → `ValueError`.
3. Re-register with `replace=True` → returns new fn.
4. `lookup` on missing ref → `MetricNotRegistered`.
5. Reorder same examples → same hash (canonicalizer sorts).
6. Different float representations (`1.0` vs `1`) — canonical-JSON normalizes (test pinning the one we get).
7. Schema violation: example with extra key `{"inputs": {...}, "expected": ..., "extra": 1}` → `ValueError`.
8. Empty training set → known fixed hash.

#### Exit gate

- Tests cover all 8 edge cases.
- `_canonicalize_training_set` is pure (no IO, no time).
- Pyright clean.

### A3 — `_plan_to_dspy_module` forward adapter

**Files:** `src/persistence/plan/_optimize.py` (NEW)
**Tests:** `tests/plan/test_plan_to_dspy_adapter.py`
**Lines:** ~150 LOC + ~200 LOC tests (DSPy mocked)

#### Public API (internal — not in `__init__.__all__`)

```python
# _optimize.py
def _plan_to_dspy_module(plan: Node) -> "dspy.Module":
    """Walk plan AST. Per :llm-call node: instantiate dspy.Predict or
    dspy.ChainOfThought based on node.attrs.get("reasoning") in {"direct","cot"}.
    Returns a callable dspy.Module whose `forward(**inputs)` runs the plan
    via the dispatcher and returns the final leaf result.
    """
```

#### Semantics

- DSPy `import` happens inside the function (not module level) — verified by A1's no-DSPy test which exercises `_execute.py` import; A3's tests verify the lazy-import path is reachable.
- For each `:llm-call` node, signature is read from `attrs.get("signature")` (string like `"question -> answer"` per DSPy convention).
- Adapter result is a `dspy.Module` subclass; `forward` walks the same plan via `execute()`.
- Tests mock `dspy.Predict`, `dspy.ChainOfThought`, `dspy.Module` so the test suite does NOT require dspy-ai installed.

#### Edge cases the tests MUST cover

1. Plan with no `:llm-call` → adapter returns a Module that just walks dispatcher.
2. Plan with one `:llm-call` with `signature="q -> a"` → mocked `dspy.Predict("q -> a")` called once.
3. Plan with `attrs.get("reasoning") == "cot"` → mocked `dspy.ChainOfThought` instead of `Predict`.
4. Missing `signature` attr → clear `ValueError("plan/optimize: :llm-call node missing 'signature' attr")`.

#### Exit gate

- Tests pass with `dspy` not installed (mocked via `sys.modules["dspy"] = MagicMock()`).
- Spec review confirms the adapter does NOT touch `_execute.py`'s import path.

### A5 — `SkillLibrary.register/lookup/list_skills`

**Files:** `src/persistence/plan/_skill_library.py` (NEW), `src/persistence/plan/__init__.py`
**Tests:** `tests/plan/test_skill_library.py`
**Lines:** ~120 LOC + ~180 LOC tests

#### Public API

```python
class SkillLibrary:
    def __init__(self, db: DB) -> None: ...

    def register(self, plan: Node, promotion_record: PromotionRecord) -> str:
        """Returns skill_id. Re-registration of same plan is no-op (returns same id)."""

    def lookup(self, skill_id: str) -> tuple[Node, PromotionRecord] | None: ...

    def list_skills(self, *, tag_filter: str | None = None) -> list[str]: ...
```

#### Skill id derivation

`skill_id = "skill/" + plan.id[:16]` — first 16 hex chars of the 32-hex Node.id. Namespaced so `skill_id` is a fact-store entity id, content-addressed via plan_id.

#### Datom shape (per registration — 3 datoms)

```python
[
    Datom(e=skill_id, a="skill/plan",             v=plan.id,        ...),
    Datom(e=skill_id, a="skill/promotion-record", v=promotion_id,   ...),
    Datom(e=skill_id, a="skill/registered-at",    v=clock_now_ms,   ...),
]
```

`clock_now_ms` is supplied by the caller (initially the test fixture; later, the `:clock/now` effect handler). NOT `time.monotonic()` / `time.time()`.

#### Edge cases the tests MUST cover

1. `register → lookup` round-trip preserves Plan AST + PromotionRecord byte-identity.
2. Same plan registered twice → same `skill_id`, no extra datoms (idempotent).
3. `lookup` on missing skill_id → `None`.
4. `list_skills(tag_filter=":seq")` returns only skills whose `plan.tag == ":seq"`.
5. `list_skills(tag_filter=None)` returns all.
6. Reconstruction: register 3 skills, run `db.asOf(t_after_2nd)`, list_skills returns 2.

#### Exit gate

- Datoms use plain-string `a` keys (no leading colon).
- Skill id derivation is deterministic.
- All 3 datoms emit in a single `db.transact` (atomic).
- Spec review confirms fact-store-backed (no separate index).
