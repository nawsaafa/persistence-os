```python
@dataclass(frozen=True)
class SubstituteLeafAction:
    target_path: tuple[int, ...]
    new_leaf: Node

@dataclass(frozen=True)

    target_path: tuple[int, ...]

    skill_id: str

Action = SubstituteLeafAction | AddStepAction | ComposeWithSkillAction

MAX_PLAN_DEPTH: int = 32   # module-level public constant


def apply_action(
    plan: Node,
    action: Action,
    *,
    skill_library: "SkillLibrary | None" = None,

# private helpers (no re-export)
def _action_hash(action: Action) -> str: ...
def _resolve_path(plan: Node, path: tuple[int, ...]) -> Node: ...
def _replace_at_path(plan: Node, path: tuple[int, ...], replacement: Node) -> Node: ...
```

`_errors.py` extension:

```python
class PlanDepthExceeded(ValueError):
    """`apply_action` produced a Plan AST whose depth exceeds MAX_PLAN_DEPTH."""
```

**Semantics (design §5–§6):**
- `apply_action` is **pure** + structural — leaves untouched subtrees object-identical.
- Dispatch is **`isinstance`-strict** (NOT `match`/structural). Third-party dataclasses with matching field shapes raise `ValueError(f"unknown action kind: {type(action).__name__}")`.
- `_action_hash` recipe: canonical-JSON of `{kind, target_path: list, payload-fields-using-Node.id-not-asdict}`. **Reuses `_ast.py:111-148` canonical-JSON helper — no parallel implementation.** (Per design §5: nested Nodes hash by `Node.id`, NOT `dataclasses.asdict`, which would diverge.)
- `_apply_compose_with_skill` rejects upfront if `_plan_depth(skill_plan) > MAX_PLAN_DEPTH // 2 = 16`; raises `PlanDepthExceeded`.
- `apply_action` post-construction also asserts `_plan_depth(new_plan) <= MAX_PLAN_DEPTH`; raises `PlanDepthExceeded` on overflow.
- `_resolve_path` raises `IndexError` on invalid path; the caller (MCTS loop in B6) converts to a `phase="reject"` datom with `reason="plan_construction_raised"`.
- `_LEAF_REQUIRED_TAGS = frozenset()` (empty in v0.6.5; reserved for v0.7+).

**Tests — `tests/plan/test_action_canonical_hash_stability.py`:**
1. Same `SubstituteLeafAction(target_path=(0,2), new_leaf=Node(":leaf/predict", attrs={"prompt":"x"}, children=()))` → same `_action_hash` across two calls.
2. Same hash across two Python processes (`subprocess.run(...)` round-trip — pin determinism across re-imports).
3. Same hash regardless of `attrs` dict insertion order (canonical-JSON sorts).
4. Different action kinds with identical `target_path` produce different hashes (kind discriminator in payload).
5. `ComposeWithSkillAction(skill_id="skill/abc...")` hash depends only on `target_path` + `skill_id` (NOT on the looked-up plan; expander resolves).

**Tests — `tests/plan/test_action_applicator_invariants.py`:**
1. `apply_action(SubstituteLeafAction)` on a 3-leaf plan: target leaf swapped; siblings object-identical (`is` check).
2. `apply_action(AddStepAction)` inserts at `at` index: parent.children length + 1; sibling order preserved.
3. `apply_action(ComposeWithSkillAction)` with stub `SkillLibrary` returning a known skill plan: result root tag matches skill plan tag; subtree at `target_path` becomes first child of skill root.
4. Invalid `target_path` (out of bounds) → `IndexError`.
5. `_replace_at_path` returns a new Node (no in-place mutation; original `plan` unchanged byte-identical).

### B3 — `MCTSNode` + `MCTSEdge` + `_select_child` PUCT

**Design-doc sections load-bearing:** §4 (node/edge identity, transposition, edge-vs-node statistics), §7 (PUCT formula + cold-start prior + tie-breaking), §17 ADR-1, ADR-4.

**Files to touch:**

**Public surface delta:**

```python
@dataclass(slots=True)  # NOT frozen — visits/total_value/children-dict-membership/is_terminal mutable
class MCTSNode:
    plan_id: str
    visits: int = 0
    total_value: float = 0.0
    children: dict[str, "MCTSEdge"] = field(default_factory=dict)
    is_terminal: bool = False

    @property
    def q_value(self) -> float:
        return 0.0 if self.visits == 0 else self.total_value / self.visits


@dataclass(slots=True)  # NOT frozen — edge counters mutable in BACKUP
class MCTSEdge:
    action_hash: str
    action: Action
    child_plan_id: str
    prior: float
    visits_through_edge: int = 0
    total_value_through_edge: float = 0.0


def _select_child(node: MCTSNode, c_puct: float) -> MCTSEdge | None: ...
```

**Semantics (design §7):**
- `_select_child` implements PUCT:
  ```
  UCB(s,a) = Q(s,a) + c_puct * P(s,a) * sqrt(parent.visits) / (1 + N(s,a))
  ```
- `Q(s,a) = total_value_through_edge / max(1, visits_through_edge)` — returns `0.0` for unvisited edges (cold-start; no FPU bonus).
- `parent.visits = MCTSNode.visits` (NOT edge counter).
- `sqrt(parent.visits)` falls back to `1.0` when `parent.visits == 0` (edge case for first-iteration root).
- **Tie-breaking:** stable — first edge in dict-insertion order wins on equal score (Python 3.7+ dict order preserved).
- Returns `None` when `node.children` is empty.
- **No parent pointer on `MCTSNode`** (DAG, not tree; backup walks the path traversed in the current iteration — covered in B6).

**Tests — `tests/plan/test_puct_formula_pin.py`:**
1. **Closed-form pin:** node with 2 edges, known `(prior, visits, total_value)` per edge, known `c_puct=1.4`, parent.visits=10. Compute UCB1 by hand for each edge; assert `_select_child` returns the edge with the max score.
2. **Cold start:** all edges unvisited (`visits_through_edge=0`); `Q` is 0.0; selection is by prior alone (`c_puct * P * sqrt(parent.visits) / 1`). Assert highest-prior edge wins.
3. **`parent.visits == 0` edge case:** root with 2 edges, parent.visits=0; `sqrt_parent` falls back to 1.0; selection still well-defined.
4. **Tie-break stability:** two edges with mathematically-equal UCB scores (priors equal, visits equal, totals equal) → first inserted wins. Repeat the test with reversed insertion order: the new "first inserted" wins. Pin: dict-insertion-order-stable.
5. **Empty children:** `_select_child` on a node with `children={}` → returns `None`.
6. **`MCTSNode.q_value` property:** `visits=0` → returns `0.0` (not raise / not NaN).

**Acceptance criteria:**
- All 6 tests green.
- `slots=True` on both dataclasses; **NOT** `frozen=True` on either (the design pins mutation as the design intent).
- `_select_child` does NOT mutate node or edge state — pure read.

**Semantics (design §16 + §10):**
- Implements the §16 pseudocode end-to-end: SELECT → EXPAND → EVALUATE → BACKUP.
- `transposition: dict[str, MCTSNode]` — graph MCTS keyed by `plan.id`.
- `expander_cache: dict[str, tuple[tuple[Action, float], ...]]` — frozen list per plan_id.
- `evaluator_cache: dict[str, float]` — float per plan_id.
- `plan_by_id: dict[str, Node]` — materialized Node per plan_id (used by EXPAND to call `expander.propose(plan_by_id[node.plan_id], k=...)`).
- **Prior-sum assertion** at expander cache miss: `abs(sum(p for _, p in proposals) - 1.0) < _PRIOR_TOL` for non-empty `proposals`; else raise `ExpanderContractError`.
- **Evaluator runs on the SELECTed-and-expanded leaf** (Invariant 5). Rollout depth = 1 (no random walk between expansion and evaluation).
- **NaN/Inf reject:** `not math.isfinite(score)` — `pending_datoms` carries a placeholder reject record; BACKUP skipped; `iter_index += 1`.
- **None reject:** evaluator returned `None` (Protocol violation) — same shape as NaN/Inf reject, different `reason`.
- **Evaluator-raises:** caught as `Exception`; reject record with `error_class=type(exc).__name__`, `error_repr=repr(exc)`. BACKUP skipped.
- **`db: DB | None`:** when `None`, `_build_iteration_datoms` is still called (so the prev-hash chain advances correctly across iterations even in db-less test runs), but no `db.transact(...)` call is made. **B6 ships zero datom side-effects.** B8 wires the actual `db.transact()` invocation.
- **Winner selection:** `winner_edge = max(root.children.values(), key=lambda e: e.visits_through_edge, default=None)`. Most-visited convention (AlphaZero). If `winner_edge is None`, `winner = initial_plan`.

  ```python
      "config_hash": _hash_config(config),
      "started_at": started_at_ms,

  ```

- **6 reasons OR-combined:** `max_iter`, `max_unique_plans`, `simple_regret`, `wall_clock`, `exhausted`, `all_evaluations_failed`.
- **First-fired wins:** `_termination_reached(...)` returns the first reason that fires this iteration.
- `wall_clock` is a no-op in v0.6.5 (search loop ignores `wall_clock_budget_ms`); the reason stays in the union for forward-compat. **No `wall_clock` test fixture in v0.6.5** (the reason is unreachable until v0.7+).
- `exhausted`: root has no proposals at iter 0 → exit immediately, `winner = initial_plan`.
- `all_evaluations_failed`: every evaluator call raised → `winner = initial_plan` + `UserWarning`.
- **`simple_regret` Q-gap on visits-sorted top-2:**
  ```python
  if len(root.children) < 2:
      simple_regret_signal = False  # short-circuit
  else:
      sorted_edges = sorted(root.children.values(), key=lambda e: e.visits_through_edge, reverse=True)
      top1, top2 = sorted_edges[0], sorted_edges[1]
      gap = top1.q_value() - top2.q_value()
      # require gap above threshold for K consecutive iterations (config.simple_regret_window)
      ...
  ```
- **Visits-sort, NOT Q-sort** — the "best - second-best" gap is computed by sorting `root.children.values()` by `visits_through_edge` descending. This matches AlphaZero / MCTS literature for "posterior commitment" (most-visited = most-confident); Q-sorted alternative would compute "regret on point estimate" which is not what convergence proofs cover.

**Semantics (design §13):**

## 9. Anti-patterns to refuse

If a subagent proposes any of the following, the spec-reviewer rejects on sight:

| Anti-pattern | Why refused |
|---|---|
| Adding `time.time()` / `time.monotonic()` anywhere in `persistence.plan._mcts` | Breaks the no-wall-clock rule; `started_at_ms` is caller-supplied. |
| Splitting `_mcts.py` into multiple sub-files | Design ADR-11 mandates flat single file at v0.6.5. Lift in v0.7+ if needed. |
| Importing `dspy` in `_mcts.py` (any path) | Closed-loop separation; design §12 / ADR-9. DSPy is the caller's recipe, not MCTS's. |
| Registering a `:branch` dispatcher handler | ADR-12 deferred to v0.6.6. |
| Bumping `PLAN_CANONICAL_VERSION` | Stream B is observational, not structural. |
| Using `match` / structural pattern matching for `Action` dispatch | Design §5 mandates `isinstance`-strict (nominal type). `match` would silently route duck-typed third-party dataclasses. |
| Implementing canonical-JSON twice (one in `_ast.py`, one in `_mcts.py`) | Design §5 + §13 + B1 acceptance: reuse `_ast.py:111-148` helper. |
| Caching at the evaluator-class level (e.g. `LLMJudgeEvaluator._cache: dict`) | Cache is owned by the MCTS loop (B6), not the evaluator instance. Evaluator-side caching would break replay's cache-pre-population path. |
| Allowing `MCTSReplayCacheMiss` to be imported from `persistence.plan` | Test-only exception; lives in B-INT test module. |
| Recording `phase="evaluate"` datoms on cache hits | Cache-miss-only recording (ADR-10). Cache hits emit zero datoms; `iter_index` advances regardless. |
| Recording action hashes only (not `new_leaf_canonical` / `new_child_canonical`) on `phase="expand"` | Closes R2 M4: hash alone is one-way; full canonical Node bytes are required for replay-from-datoms-alone Action reconstruction. |
| Per-datom `db.transact` calls (instead of one per iteration) | Design §13 transaction granularity. One MCTS iter = one transact. |
| Using `Iterable` (instead of `Sequence`) for `Expander.propose` return type | Design §8: single-pass generator interactions break the prior-sum assertion. |
| Returning `None` from `Evaluator.evaluate` as a "no signal" pattern | Design §9: `None` is a Protocol violation; the loop routes through `phase="reject"` with `reason="evaluator_returned_none"`. |
| Letting NaN / Inf propagate through PUCT arithmetic | Design §9 NaN/Inf defense: reject at the boundary. |
| Q-sorted (instead of visits-sorted) top-2 in `simple_regret` Q-gap | Design §11: visits-sort matches AlphaZero / MCTS literature for posterior commitment. |
