## 1. What 2.1a delivers

`src/persistence/coder/` — the 9th module under `persistence.*`, alongside the seven substrate modules (fact / effect / plan / txn / replay / spec / repl) and the curated SDK facade (sdk). Unlike the substrate modules, **`coder` is a CONSUMER** — it imports from `persistence.sdk` only, never from raw substrate modules.

The deliverable from the refresh doc § 3 task table:

> **2.1a** — `persistence.coder` skeleton + CLI entry — `python -m persistence.coder --task "..." ` runs no-op loop.

"No-op loop" lands as: a `Coder` class whose `run()` walks the ReAct loop shape from base § 3.4, hits the first `NotImplementedError("Phase 2.X — <purpose>")` stub on its path, and surfaces a clean stderr banner before exiting non-zero. The skeleton makes the architecture **visible** (ARIS-reviewable, statically analyzable) without doing any LLM, effect, or substrate-write work — those land in 2.1b → 2.3d as each stub fills in.

## 2. Locked decisions

Five decisions taken via 2026-05-03 brainstorming session (Claude Opus 4.7) on top of refresh § 2's locks:

### LD1 — Scaffolding depth: full ReAct skeleton with `NotImplementedError` stubs (~150 LOC class body)

The base design § 3.4 has the agent-loop shape locked. 2.1a brings that architecture forward as stub methods now rather than postponing class shape to 2.1b. Three reasons:

1. ARIS R1 sees the full architecture in one review (no "wait until 2.1b for the real shape" deferral).
2. `NotImplementedError` stubs are cheap to skip past in 2.1b/2.2/2.3 integration tests — you can write tests against the stub interface without import errors.
3. Q2-locked discipline (zero `s.escape.*` callsites) lands on day one with every stub's docstring naming which `s.*` surface fills it. ARIS catches drift if 2.1b/2.2/2.3 reorganize.

The risk is over-committing to a method partition that later wants refactoring. Mitigated by stubs being trivial (`raise NotImplementedError(...)`) — renames are ~5 minutes.

### LD2 — Substrate ownership: dependency injection (DI)

`Coder` takes an open `Substrate` via `__init__`; never opens or closes one itself.

```python
@dataclass
class Coder:
    task: str
    substrate: Substrate
    confidence_threshold: float = 0.65
    missing_confidence_default: float = 0.5
```

Caller (CLI, tests) handles the open/close lifecycle. Matches the `repl/_session.py` precedent — substrate is owned at the long-running-process boundary (`WSServer.serve(host, port, db, ...)` for repl; `__main__.py` for coder), business class stays pure.

Rejected: (i) `Coder` owns substrate (couples lifecycle to behavior, harder to test) and (iii) hybrid factory (more API surface for no benefit at this size).

### LD3 — `Coder.run()` body: walk stubs, first one raises

```python
def run(self) -> None:
    obs = self._observe()                      # raises Phase 2.2a
    decision = self._decide(obs)               # never reached in 2.1a
    if self._should_escalate_branch(decision):
        self._escalate_branch(decision)
    elif self._should_escalate_plan(decision):
        self._escalate_plan(decision)
    else:
        self._act(decision)
    self._check_pause()
```

The first stub on the path (`_observe`) raises `NotImplementedError("Phase 2.2a — substrate read via s.fact.q")`. `__main__.py` catches and exits 1 with a clean stderr banner. No traceback dump.

Rejected: (a) bare `run()` raises (no architectural visibility), (c) all-stubs-`pass` (contradicts LD1's NotImplementedError choice), (d) bracket-with-`:coder/started` datoms (commits contract surface 2.1c may want differently).

### CP1 — `Substrate.open()` URI grammar: bare `"memory"`, not `:memory:`

Per `_facade.py:1354-1442`:

```python
@classmethod
def open(cls, uri: str = "memory", *, audit: bool = True) -> "Substrate":
    """...
    - "memory"                  — in-process in-memory store
    - "sqlite:///<absolute-path>" — file-backed SQLite store
    - "postgres://..."          — raises BackendNotInstalled until PG1
    """
```

Default URI is `"memory"`; the audit handler stack auto-installs (`audit=True` default). Substrate is both a context manager (`__enter__`/`__exit__`) and callable-cleanup (`close()`).

CLI flag `--db-path` defaults to `None`; when omitted, the CLI prints a stderr warning (per `repl/_cli.py:63-77` precedent) and dispatches `Substrate.open("memory")`. Explicit values (`memory`, `sqlite:///abs/path`) pass through unmodified to `Substrate.open(uri)`.

### CP2 — `--confidence-threshold` deferred to 2.3b/2.4a

The class attribute `Coder.confidence_threshold: float = 0.65` lands now (2.1a tests assert default + constructor pass-through). The CLI flag does NOT land in 2.1a — the threshold isn't exercised (every stub raises before `_should_escalate_branch` would consume it). Exposing a flag for unused behavior commits to a UX shape (flag vs. env var vs. config file vs. per-task seed) before knowing what 2.4a's telemetry-tuning workflow needs.

Repl precedent: exposes `--clock-iso` (load-bearing for replay determinism) but no policy thresholds. Land at the CLI when behavior consumes it.

### 3.2 `Coder` class shape (`_session.py`)

```python
from __future__ import annotations

from dataclasses import dataclass

from persistence.sdk import Substrate

from ._types import LLMDecision, Observation


class CoderStubNotImplemented(NotImplementedError):
    """Phase 2.1a skeleton sentinel: a method body has not been
    filled yet by its target sub-phase. The CLI banner catches
    THIS subtype only; bare NotImplementedError raised by real
    code in 2.1b+ propagates as a real failure. (R1 fix.)"""


@dataclass
class Coder:
    """ReAct skeleton with two escalations (plan, branch) and a REPL pause hook.

    Phase 2.1a: every behavioral method is a stub raising
    CoderStubNotImplemented tagged with the sub-phase that fills it.
    Substrate ownership is dependency-injected (LD2 / refresh Q2):
    callers (CLI, tests) open the substrate; Coder never does.

    The loop shape mirrors base design § 3.4 (agent-loop diagram)
    top-to-bottom; reordering is an ARIS-reviewable architectural
    change, not a refactor.
    """

    task: str
    substrate: Substrate
    confidence_threshold: float = 0.65          # base § 3.4; tuned at 2.4a
    missing_confidence_default: float = 0.5     # base § 3.4 fail-safe

    # main entry — Phase 2.1b fills the loop body (one iter → while-loop)
    def run(self) -> None:
        obs = self._observe()
        decision = self._decide(obs)
        if self._should_escalate_branch(decision):
            self._escalate_branch(decision)
        elif self._should_escalate_plan(decision):
            self._escalate_plan(decision)
        else:
            self._act(decision)
        self._check_pause()

    # --- ReAct primitives — Phase 2.1b/2.2a fill these ---------------

    def _observe(self) -> Observation:
        raise CoderStubNotImplemented(
            "Phase 2.2a — substrate read via s.fact.q"
        )

    def _decide(self, obs: Observation) -> LLMDecision:
        raise CoderStubNotImplemented(
            "Phase 2.1b — LLM provider call → :llm/decision datom"
        )

    def _act(self, decision: LLMDecision) -> None:
        raise CoderStubNotImplemented(
            "Phase 2.2a — s.effect.perform on :fs/:shell/:code/:git"
        )

    # --- Escalation gates — Phase 2.3a/b fill these ------------------

    def _should_escalate_plan(self, decision: LLMDecision) -> bool:
        raise CoderStubNotImplemented(
            "Phase 2.3a — checks decision.kind == 'plan'"
        )

    def _escalate_plan(self, decision: LLMDecision) -> None:
        raise CoderStubNotImplemented(
            "Phase 2.3a — Plan AST builder + s.plan.execute"
        )

    def _should_escalate_branch(self, decision: LLMDecision) -> bool:
        raise CoderStubNotImplemented(
            "Phase 2.3b — decision.kind == 'branch' "
            "or confidence < threshold"
        )

    def _escalate_branch(self, decision: LLMDecision) -> None:
        raise CoderStubNotImplemented(
            "Phase 2.3b — s.plan.mcts_search + s.txn.fork + s.plan.judge"
        )

    # --- REPL pause hook — Phase 2.3d fills this ---------------------

    def _check_pause(self) -> None:
        raise CoderStubNotImplemented(
            "Phase 2.3d — :repl/request datom check + pause/resume"
        )
```

`CoderStubNotImplemented(NotImplementedError)` is the **R1 fix-1 mechanism**: it preserves Python's standard "method not implemented" exception identity (so `pytest.raises(NotImplementedError)` still matches) while letting the CLI catch only stub raises and let any *real* `NotImplementedError` from 2.1b+ propagate as a genuine failure rather than a banner-masked silent skip. Re-exported via `__init__.py` so tests can target the subtype explicitly.

Stubs are ordered to match base § 3.4's loop diagram top-to-bottom. Each `NotImplementedError` message names exactly one downstream sub-phase. ARIS R1 catches drift if implementation reorganizes.

Zero `s.escape.*` callsites — every method is a `raise` before reaching any SDK surface. G1 lockfile passes from day one.

### 3.3 Method partition — phase-fill responsibility

| Method | Stub raises | Filled in | What that phase does |
|---|---|---|---|
| `run()` | (calls stubs; doesn't itself raise) | 2.1b | turns one-iter into `while not done:` loop |
| `_observe` | "Phase 2.2a — substrate read via s.fact.q" | 2.2a | reads substrate state via `s.fact.q` |
| `_decide` | "Phase 2.1b — LLM provider call → :llm/decision datom" | 2.1b | calls LLM provider, emits `:llm/messages` + `:llm/decision` |
| `_act` | "Phase 2.2a — s.effect.perform on :fs/:shell/:code/:git" | 2.2a (stub flips green); 2.2b additive | 2.2a wires the dispatch; 2.2b expands the handler set additively (no stub message change) |
| `_should_escalate_plan` | "Phase 2.3a — checks decision.kind == 'plan'" | 2.3a | predicate on `:strategy/plan` |
| `_escalate_plan` | "Phase 2.3a — Plan AST builder + s.plan.execute" | 2.3a | builds Plan AST, calls `s.plan.execute` |
| `_should_escalate_branch` | "Phase 2.3b — decision.kind == 'branch' or confidence < threshold" | 2.3b | predicate on `:strategy/branch` + confidence |
| `_escalate_branch` | "Phase 2.3b — s.plan.mcts_search + s.txn.fork + s.plan.judge" | 2.3b | MCTS + fork + judge |
| `_check_pause` | "Phase 2.3d — :repl/request datom check + pause/resume" | 2.3d | REPL pause hook |

## 4. CLI surface (`_cli.py` + `__main__.py`)

`argparse` (yagni — persistence-os has no click/typer dep; matches `repl/_cli.py`).

| Flag | Required | Default | Purpose |
|---|---|---|---|
| `--task` | yes | — | Free-text task description (`Coder.task`) |

Excluded by CP2: `--confidence-threshold`. The class attribute lands; the CLI flag waits for 2.3b/2.4a.

`__main__.py` shape:

```python
from __future__ import annotations

import sys

from persistence.sdk import Substrate

from ._cli import build_parser
from ._session import Coder, CoderStubNotImplemented


def main(argv: list[str] | None = None) -> int:
    parser = build_parser()
    args = parser.parse_args(argv)

    if args.db_path is None:
        print(
            "warning: no --db-path; using in-memory substrate "
            "(non-persistent — coder state will not survive process exit)",
            file=sys.stderr,
        )
        uri = "memory"
    else:
        uri = args.db_path

    try:
        with Substrate.open(uri) as substrate:
            Coder(
                task=args.task,
                substrate=substrate,
            ).run()
    except CoderStubNotImplemented as exc:
        print(f"persistence-coder skeleton: {exc}", file=sys.stderr)
        return 1
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

The `try/except CoderStubNotImplemented` catch (R1 fix-1) is deliberately narrow: produces the clean stderr banner (`persistence-coder skeleton: Phase 2.2a — substrate read via s.fact.q`) and returns 1 ONLY for un-filled stub raises. Real `NotImplementedError` thrown by 2.1b+ implementation code (e.g. an LLM-provider abstract method that wasn't overridden) will propagate untouched — no banner-masking. Bare uncaught raise would dump a traceback — ugly for a no-op skeleton run; the typed catch keeps the banner UX while preserving genuine failure visibility.

## 6. 2.1c forward-pointer (G1 lockfile contract test)

2.1c (Wed 2026-05-06) lands the G1 contract test against `coder.lockfile.json` in alpha-mode. The skeleton's job is to make that test trivially pass when 2.1c writes it:

| Lockfile field | 2.1a state |
|---|---|
| `alpha_mode: true` | (set by 2.1c, not 2.1a) |
| `escape_callsites: []` | **enforced now** — zero raw-substrate imports + zero `.escape` attribute access in `src/persistence/coder/` |
| `allowed_set: [s.fact.*, s.txn.*, s.plan.*, s.plan.judge, s.replay.*, s.spec.*, s.effect.*]` | (set by 2.1c) |
| Static-analysis target | `src/persistence/coder/` — every `s.<ns>.<method>` callsite must be in `allowed_set` |

The skeleton has **zero `s.*` callsites** in 2.1a (every method body is `raise CoderStubNotImplemented(...)` before reaching any SDK call). 2.1c's contract test trivially passes; 2.1b adds the first real callsite (`s.fact.transact` for the `:llm/decision` datom — must be in the allowed-set the lockfile pins) and the contract test starts doing real work.

### 6.1 Curated-SDK guard — AST-based spec (R1 fix-3)

A grep-based check (`rg 's\.escape'`) is **insufficient** — it can be bypassed by aliasing (`from persistence.sdk import Substrate as sub; sub.escape.fact.q(...)`) or by importing raw substrate modules directly. The 2.1c lockfile contract test MUST be AST-based with the following enforcement (specified now so 2.1a's no-op skeleton trivially passes and 2.1b/2.2/2.3 can't drift):

**Rule G1.A — No raw-substrate imports.** For every `.py` file under `src/persistence/coder/`, walk `ast.Import` and `ast.ImportFrom` nodes. **Reject** any import whose module path matches `r'^persistence\.(fact|effect|plan|txn|replay|spec|repl)(\..*)?$'`. **Allow** only `persistence.sdk` and submodules thereof. (Exceptions: type-only `from typing import …`, `from dataclasses import …`, stdlib, third-party.) The Coder consumes the substrate exclusively via the curated facade.

**Rule G1.B — No `.escape` attribute access regardless of alias.** Walk `ast.Attribute` nodes. **Reject** any attribute whose `attr == "escape"` chained off a name that resolves (statically, via the file's `Substrate` / `s` / curated-SDK type binding) to the curated facade. The detector is name-agnostic: it must catch `s.escape`, `sub.escape`, `self.substrate.escape`, and any rebinding pattern. Implementation hint for 2.1c: track local names whose RHS imports or annotations point at `persistence.sdk.Substrate` or `_FacadeNamespace` types, then flag every `<tracked_name>.escape` reachable.

**Rule G1.C — Allowed-set check on every curated-SDK callsite.** Walk `ast.Attribute` chains rooted at tracked Substrate-typed names; collect `<root>.<ns>.<method>` triples. **Reject** any triple whose `<ns>.<method>` is not in `allowed_set` from the lockfile. (Trivially passes in 2.1a — no callsites.)

**2.1a obligation:** zero raw-substrate imports + zero `.escape` references + zero `s.<ns>.*` callsites in `src/persistence/coder/`. The cheapest possible 2.1c handoff: 2.1a doesn't need to know what `coder.lockfile.json` looks like; it just needs to satisfy these three rules statically — which the all-stubs-raise body trivially does.

Acceptance signals at 2.1a end:
- `rg -n 'from persistence\.(fact|effect|plan|txn|replay|spec|repl)' src/persistence/coder/` → zero matches.
- `rg -n '\.escape\b' src/persistence/coder/` → zero matches (smoke check; the load-bearing detector is the AST walk that 2.1c writes).
- `rg -n '^\s*s\.\w+\.\w+\(' src/persistence/coder/` → zero matches.

## 7. Datom emissions in 2.1a

**Zero.** The skeleton raises `NotImplementedError` from `_observe()` before any `s.fact.transact` or `s.effect.perform` runs. 2.1b lands the first datoms (`:llm/messages`, `:llm/decision`).
