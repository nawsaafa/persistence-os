## 1. Scope & deliverable

Three things land in 2.1b, in order of architectural weight:

### 1.1 Three `:llm/call` handler factories (substrate-side)

The `:llm/call` op in `effect/catalog.py` (`OpSpec` with locked args/returns wire shape) is the cross-provider contract. Each provider is a `Handler` instance produced by a `make_*_llm_handler(...)` factory in `effect/handlers/`. Coder body never knows which handler is at the bottom — it calls `s.effect.perform(":llm/call", ...)` and reads catalog shape.

| Handler | File | Mode | Auth | Deps |
|---|---|---|---|---|
| `make_anthropic_llm_handler` | `effect/handlers/anthropic.py` | Mode 1 — pay-per-token | `ANTHROPIC_API_KEY` env var | `anthropic` (lazy import) |
| `make_claude_code_llm_handler` | `effect/handlers/claude_code.py` | Mode 2 — Max subscription | Claude Code session (signed-in) | `claude-agent-sdk` (lazy import) |
| `make_callable_llm_handler` | `effect/handlers/callable.py` | Mode 3 — library primitive | Whatever caller wires | none |

### 1.2 `Coder._decide` body filled in

Replaces the `CoderStubNotImplemented("Phase 2.1b — LLM provider call → :llm/decision datom")` raise. ~50 LOC body. Calls `s.effect.perform(":llm/call", ...)` with the `EMIT_DECISION_TOOL_SCHEMA` tool definition; reads catalog shape; transacts `:llm/messages` (before perform) and `:llm/decision` (after parsing) datoms; returns structured `LLMDecision`.

### 1.3 `LLMDecision` dataclass body filled in

`_types.py` — was empty in 2.1a. Three fields per base design § 3.4:

```python
DecisionKind = Literal["act", "plan", "branch"]

@dataclass(frozen=True)
class LLMDecision:
    kind: DecisionKind
    confidence: float           # [0.0, 1.0]
    payload: Mapping[str, Any]  # kind-dispatched shape; tightens progressively in 2.2a/2.3a/2.3b
```

### 1.4 What 2.1b deliberately does NOT touch

- **`run()` loop body.** Still single-iteration. The CHANGELOG-coder hint about "one iter → while-loop" was forward-looking — widening is meaningless until `_observe`/`_act` exist (2.2a). Loop widening lands when iterations have substance to repeat over. **2.1b's CHANGELOG-coder entry amends this hint to reflect honest scope.**
- **Other `_*` stubs.** `_observe`, `_act`, `_should_escalate_*`, `_escalate_*`, `_check_pause` remain `CoderStubNotImplemented`. `Coder.run()` will still raise on `_observe()` first; `_decide` is exercised via direct unit tests, not via end-to-end `run()`.
- **CLI `--confidence-threshold` flag.** Still deferred per Phase 2.1a CP2.
- **Real effect dispatch from the LLM.** The LLM sees only `emit_decision`. Real `:fs/`, `:shell/`, `:code/`, `:git/` tool surfaces are 2.2a.

### 1.5 What the user can do after 2.1b ships

```bash

persistence-coder --task "implement parse_csv_row" --provider anthropic

# Mode 2 — Max subscription via Claude Code session
```

In all three CLI modes, `Coder.run()` still raises `CoderStubNotImplemented("Phase 2.2a — substrate read via s.fact.q")` on the first call (because `_observe` is still a stub). The 2.1b deliverable is **`_decide` correctness exercised by direct unit tests**, not end-to-end `run()` success.

## 2. Locked decisions

Six decisions taken via brainstorming session on top of refresh § 2's locks and Phase 2.1a's CP1/CP2.

### LD1 — Provider abstraction lives at the substrate handler layer, not coder-side

The `:llm/call` op in `effect/catalog.py` is the cross-provider contract. Each provider is a `Handler` factory in `effect/handlers/`. Coder calls `s.effect.perform(":llm/call", ...)`; reads catalog shape; never knows which handler is at the bottom.

**Why:** the substrate's effect runtime stack (raw → retry → rate-limit → cache → audit → policy) is already designed for vendor-handler swap-points. A coder-side `LLMProvider` Protocol would duplicate the abstraction (two ways to call an LLM is a smell). Reuses retry/rate-limit/cache/audit on every provider for free. The `:llm/call` audit datom (auto-emitted by the audit handler) is the substrate-truth provenance; `:llm/decision` is the parsed-interpretation layer on top.

**Rejected:**
- (B) coder-side `LLMProvider` Protocol with `AnthropicProvider` impl — duplicates substrate infrastructure, bypasses the runtime stack unless re-added, two abstractions to maintain.
- (C) hybrid Protocol-with-substrate-default — the swap point already exists at the handler layer; adding a Protocol shim is YAGNI.

### LD2 — Three handlers ship in 2.1b: `anthropic`, `claude_code`, `callable`

Refresh § 6 schedule named "LLM provider abstraction + Anthropic adapter." Brainstorming flip on 2026-05-03 expanded scope to three handlers.

**Why three:**
- **Mode 1 (`anthropic`)** — headless baseline anyone with an API key can use via `pip install persistence-os` + `ANTHROPIC_API_KEY`.
- **Mode 2 (`claude_code`)** — Max-subscription path (load-bearing for Mimir Phase D distribution: persistence-coder ships as a Claude Code skill/subagent and inherits the host's auth — no new code needed when Mimir lands).
- **Mode 3 (`callable`)** — provider-agnosticism primitive. ~30 LOC. Lets any host (Codex, Cursor, juba, custom Python apps, OSS local LLMs via Ollama / vLLM) wire its LLM access via `call_fn` translation. Persistence-coder is **not Claude-specific** because the catalog shape is the contract.

`anthropic` and `claude-agent-sdk` deps are **lazy-imported** inside the factory; `pip install persistence-os` does not drag both LLM SDKs unless used. `make_callable_llm_handler` has zero new deps.

**Rejected:**
- Anthropic-only baseline — would force a downstream rewrite once the Mimir distribution path ships.
- Per-host packaging (Codex extension, Cursor plugin) — out of scope for a substrate alpha; weeks-per-host work belongs in Mimir Phase D or downstream `persistence-coder-<host>` packages.

### LD3 — Two-tier parsing in `_decide`: tool-use primary, text-fenced fallback, missing-confidence default last

```python
result = s.effect.perform(":llm/call", model=..., messages=..., tools=[EMIT_DECISION_TOOL_SCHEMA])

# Tier 1: structured path (Anthropic / Claude Code natively populate this)
if result.get("tool_calls"):
    payload = result["tool_calls"][0]["input"]
    return LLMDecision(kind=payload["kind"],
                       confidence=float(payload["confidence"]),
                       payload=payload.get("payload", {}))

# Tier 2: text fallback (any text-only backend wired through callable)
if (parsed := parse_text_decision(result.get("text", ""))) is not None:
    return LLMDecision(**parsed)

# Tier 3: missing-confidence default — forces branch escalation per base § 3.4
return LLMDecision(kind="act",
                   confidence=self.missing_confidence_default,  # 0.5 < 0.65
                   payload={"raw_text": result.get("text", "")})
```

**Why:** the catalog shape's `tool_calls` field IS the cross-provider structured-output contract. Each handler populates it via vendor translation. Anthropic & claude-agent-sdk both support tool-use natively → tier 1 lights up reliably. Callable backends that wrap text-only LLMs flow through tier 2 (caller can prompt `<decision>{json}</decision>`-fenced output) or tier 3 (degraded, branch-escalates). Missing-confidence default = `0.5` is below the `0.65` threshold, so a malformed decision triggers branch-escalation per base § 3.4 fail-safe — **working as designed**, not a bug path.

**Rejected:**
- (B) text-only parsing as primary — fragile; loses Anthropic's native schema validation.
- (C) `s.spec.validate` on parsed text — adds substrate touch surface; spec module's prompt-engineering ergonomics unproven for this use; doesn't help for backends that natively support tool-use.

### LD4 — Decision/action split: LLM sees one tool, substrate executes effects

`EMIT_DECISION_TOOL_SCHEMA` is the **only** tool exposed to the LLM in 2.1b. Not `:fs/write`, not `:shell/run`, not `:code/exec`. The LLM emits structured intent (`kind`/`confidence`/`payload`); the substrate routes the intent in `_act` (2.2a) / `_escalate_plan` (2.3a) / `_escalate_branch` (2.3b). Every effect goes through `s.effect.perform(...)` policy / audit / retry / cache.

**Why:** this is what makes `s.audit.verify_chain()` load-bearing. Some agent frameworks (OpenAI assistants default, Anthropic-tool-use-as-direct-invocation pattern) let the LLM directly invoke tools — easier demo, harder audit. Persistence-os goes the other direction: the LLM cannot bypass policy by directly invoking a tool; it can only ASK the substrate to invoke one, and the substrate decides whether to allow. The "intent log" datom (`:llm/decision`) is separate from the "what actually happened" datom chain (`:audit/*` + `:fact/transact`) — divergence between them is detectable and is itself a substrate signal.

**Rejected:** flat tool-use where the LLM directly calls `:fs/write` etc. — LLM bypass of policy stack; harder to audit; harder to fork/replay (forking after a side-effecting tool call requires substrate-side undo machinery the substrate does not have).

### LD5 — `run()` body unchanged in 2.1b; while-loop widening deferred to 2.2a

`Coder.run()` retains the 2.1a body verbatim. `_decide` is exercised via direct unit tests, NOT via `run()` (which still raises on `_observe()` first).

**Why:** loop widening has no consumer until `_observe`/`_act` are real. Widening now would require either (a) bypassing `_observe` (changes architecture without justification) or (b) wrapping `_observe` raise in try/except inside `run()` (silently masks 2.2a's stub — defeats the architecture-visibility purpose of the skeleton). Either is worse than waiting. The CHANGELOG-coder.md hint about "one iter → while-loop" in 2.1b is **amended in 2.1b's CHANGELOG entry** to reflect this honest scope.

**Rejected:**
- Widening to while-loop with try/except around `_observe` — masks stubs, defeats the skeleton's audit-friendliness purpose.
- Widening with bypass — architectural drift.

### LD6 — Provider-detection ordering for `--provider auto`: `claude-code` → `anthropic` → `echo`

When user runs `persistence-coder --task "..."` without `--provider`, the CLI tries handlers in this order:

1. **`claude-code`** if `claude-agent-sdk` is **importable** (no session-reachability probe at detection time per `_claude_code_available` § 3.6 — first-call `RuntimeError` handles signed-out case with a clean "sign in to Claude Code" hint). Selected → handler installed; stderr note: `"using claude-code provider (Max subscription)"`.
2. **`anthropic`** if `ANTHROPIC_API_KEY` env var set AND `anthropic` importable. Selected → handler installed; stderr note: `"using anthropic provider (API key, pay-per-token)"`.
3. **`echo`** (existing `make_echo_llm_handler` from `effect/handlers/raw.py`). Selected → stderr warning: `"warning: no LLM provider available — using echo handler. Set ANTHROPIC_API_KEY or sign in to Claude Code. Run will exit on first stub."`. Exit code 1 on `CoderStubNotImplemented`.

**Why claude-code first:** prefers the free/Max-subscription path. If user has both available, they probably want Max sub to flow rather than burn API tokens. Explicit `--provider anthropic` overrides.

**Why echo as the floor, not error-out:** lets users smoke-test the CLI / library without any LLM auth (they get the same `CoderStubNotImplemented` they would get otherwise; the echo handler just makes the path reachable to that point). Matches the existing pattern from `__main__.py` in 2.1a where missing `--db-path` falls to in-memory + stderr warning.

### LD7 — Substrate-side prereqs land first in 2.1b (R1 fix-pass)

The original R0 design assumed handler installation could go through `s.escape.effect.push(...)` — a non-existent API (codex R1 F1) — and assumed `:llm/call` was already covered by the canonical audit stack — false (codex R1 F3). The R1 fix-pass adds two small substrate-side changes (§ 3.0.1 + § 3.0.2) that land in 2.1b's same impl plan, in earlier task rows than the coder body:

1. New curated `s.effect.install_handler(handler, *, position)` method on `_EffectNamespace` (`_facade.py`).
2. `:llm/call` added to `CANONICAL_AUDIT_OPS` (`_audit_stack.py`) — audit middleware wraps it, raw terminator does NOT (LLM provider is the terminator).

**Why land in 2.1b instead of a separate sub-phase:** they are load-bearing for 2.1b's coder body and for G4's claim. Splitting them into 2.0g would create a chicken-and-egg with the calendar (2.1b can't ship without them; they have no consumer outside 2.1b). The combined LOC delta is ~85 (substrate) + ~120 (tests). The Tue-Thu calendar absorbs this — no further slip needed.

**Why this resolves the original Q2-violation tension:** with `install_handler` curated, `coder/` has zero `s.escape.*` callsites at any boundary (including `__main__.py`). Refresh § 2 Q2 SDK posture is preserved literally, not by exception.

### LD8 — Provider-default ordering supersedes base ADR-4 (R1 fix-pass F5)

Base design `2026-04-30-phase-2-persistence-coder-design.md` ADR-4 (lines 468-472) locked an Anthropic-first default. The 2026-05-03 brainstorming session expanded to three modes (LD2) and locked `claude-code → anthropic → echo` ordering for `--provider auto` (LD6), motivated by: (a) Mimir Phase D distribution path makes Max-subscription auth the load-bearing user flow; (b) prefer free/Max tier when both are reachable.

**Effect on ADR-4:** 2.1b's LD6 explicitly **supersedes** ADR-4's default-provider posture. ADR-4's strategic frame (single-provider-as-baseline) is preserved as the reasoning behind why we now lift to multi-provider; its specific decision (Anthropic as default) is replaced. Future readers should treat LD6 as the operative ordering and ADR-4 as historical context.

- **CF1 = 2.1a CP1.** `Substrate.open()` URI grammar: bare `"memory"`, `"sqlite:///<abs-path>"`. CLI's `--db-path` defaults to `None` → `"memory"` + stderr warning.

## 3. Architecture

### 3.0 Substrate-side prereqs (R1 fix-pass: F1 + F3 + F6)

Two small substrate-side changes land FIRST (same branch, same impl plan, but in earlier task rows than the coder body) so 2.1b's coder-side code is implementable without violating Q2 SDK posture or making the audit-chain claim.

#### 3.0.1 New curated `s.effect.install_handler` method

Codex R1 F1 surfaced that `s.escape.effect.push(...)` is not a real API: `s.escape.effect` resolves to the raw `Runtime` instance (`_resolve_raw_effect(s) → s._runtime`, `_facade.py:1128-1130`), and `Runtime` (`runtime.py:107-125`) has no `push`/`install`/`add_handler` method — only a public `handlers: list[Handler]`. AND refresh § 2 Q2 explicitly forbids `s.escape.*` callsites in `src/persistence/coder/` with no documented exception for `__main__.py`.

**Fix:** add a curated method on `_EffectNamespace` (`src/persistence/sdk/_facade.py`):

```python
class _EffectNamespace:
    # existing perform(...) here
    ...

    def install_handler(self, handler: "Handler", *, position: str = "bottom") -> None:
        """Install a handler into the substrate's runtime stack.
        ``position="bottom"`` appends to ``handlers`` (innermost — raw terminator slot).
        ``position="top"`` prepends to ``handlers[0]`` (outermost — middleware slot).
        Idempotent: re-installing a handler with the same ``name`` replaces it.

        Phase 2.1b: coder CLI uses position="bottom" to install the chosen
        :llm/call handler under the canonical audit middleware so audit
        wraps the LLM call. Library callers (Mode 3) use the same method."""
        rt = self._substrate._runtime
        # Drop any existing handler with same name first (idempotent)
        rt.handlers = [h for h in rt.handlers if h.name != handler.name]
        if position == "bottom":
            rt.handlers.insert(0, handler)  # innermost = position 0 per Runtime docstring
        elif position == "top":
            rt.handlers.append(handler)
        else:
            raise ValueError(f"position must be 'bottom' or 'top', got {position!r}")
```

(Note: `Runtime.handlers[0]` is innermost per `runtime.py:107` docstring — so "bottom" of the conceptual stack = list position 0. The method documents this explicitly.)

**Why curated, not escape:** keeps refresh § 2 Q2 unchanged (zero `s.escape.*` callsites in `src/persistence/coder/`). The G1 lockfile contract test in 2.1c gets `install_handler` added to the allowed-method set on `_EffectNamespace`. Lifecycle-boundary exceptions for `__main__.py` (which were a bandaid for the original wrong design) are no longer needed.

**Test surface:** `tests/sdk/test_effect_namespace.py` (new or extended) — install + replace-by-name + position-top + position-bottom + invalid-position. ~50 LOC.

**Companion docstring update (R2 NICE-TO-HAVE):** the existing `_EffectNamespace` class docstring at `_facade.py:106-112` currently steers callers needing handler-push to `s.escape.effect`. When `install_handler` lands, the docstring is updated to steer to the curated method first; `s.escape.effect` reservation drops to "raw runtime, advanced/test-only use". One-line CHANGELOG-sdk.md entry under v0.9.0a1.

#### 3.0.2 Extend `CANONICAL_AUDIT_OPS` to include `:llm/call`

Codex R1 F3 surfaced that `Substrate.open()` installs `canonical_audit_stack(...)` (`_audit_stack.py:55-84`) whose canonical ops are `(":plan/edit", ":code/exec", ":fork/probe", ":fork/branch", ":fork/score", ":fork/chosen", ":fold/chosen")`. **`:llm/call` is NOT in this set** — so the design's G4 claim "every `:llm/call` perform produces an `:audit/llm.call` entry … audit chain coverage already exists from substrate-completion" is false. Plus `make_audit_handler` hard-codes `name="audit"`, so simply layering a second audit middleware at coder-CLI install time would collide on name.

**Fix:** R2 fix-pass surfaced that the existing `_audit_stack.py` couples `CANONICAL_AUDIT_OPS` to the raw terminator's clauses (the terminator does `clauses = {op: lambda ...: None for op in CANONICAL_AUDIT_OPS}` per `_audit_stack.py:99`). So simply adding `:llm/call` to that tuple would make the raw terminator handle `:llm/call` (returning `None`) — which would mask the real LLM provider handler at `position=bottom`.

The correct fix is to **split the constant into two purpose-specific tuples** and make the raw terminator use only the raw subset:

```python
# Wrapped by the audit middleware — every op here gets a :audit/<op> entry
# emitted on perform with prev-hash linkage + Mimir Phase B Ed25519 signing.
CANONICAL_AUDIT_WRAPPED_OPS: tuple[str, ...] = (
    # Phase 2.0a — Plan Edit API
    ":plan/edit",
    # Phase 2.0b — :code sandbox effect
    ":code/exec",
    # Phase 2.0c-extended — DB.fork 4-datom shape
    ":fork/probe",
    ":fork/branch",
    ":fork/score",
    ":fork/chosen",
    # Phase 2.0c (Path-A) — DB.fold chosen-marker
    ":fold/chosen",
    # Phase 2.1b — :llm/call wrapping (NEW)
    ":llm/call",
)

# Raw terminator covers ONLY audit-only ops where the request datom IS the
# audit signal. Ops with real return values (here: :llm/call) are NOT in
# this set — their bottom-of-stack handler is installed separately
# (e.g. by coder/__main__.py via s.effect.install_handler at position=bottom).
CANONICAL_AUDIT_RAW_OPS: tuple[str, ...] = (
    ":plan/edit",
    ":code/exec",
    ":fork/probe",
    ":fork/branch",
    ":fork/score",
    ":fork/chosen",
    ":fold/chosen",
    # NOT ":llm/call" — see above
)

# Backwards-compatibility alias for any external code that imported the
# old name. Equal to the wrapped set (the audit-coverage perspective is
# the more frequent caller-of-truth). Marked @deprecated in CHANGELOG-effect.
CANONICAL_AUDIT_OPS = CANONICAL_AUDIT_WRAPPED_OPS
```

And `_make_canonical_raw_terminator()` is updated to consume `CANONICAL_AUDIT_RAW_OPS`:

```python
def _make_canonical_raw_terminator() -> Handler:
    """Raw no-op terminator handler covering every audit-only op.
    Per R2 fix-pass: uses CANONICAL_AUDIT_RAW_OPS (audit-only subset),
    NOT CANONICAL_AUDIT_WRAPPED_OPS (which includes :llm/call whose
    return value comes from the installed LLM provider handler)."""
    clauses = {op: (lambda _args, _k, _ctx: None) for op in CANONICAL_AUDIT_RAW_OPS}
    return Handler(
        name="audit-canonical-raw",
        wraps=set(CANONICAL_AUDIT_RAW_OPS),
        clauses=clauses,
    )
```

`canonical_audit_stack(...)` uses `CANONICAL_AUDIT_WRAPPED_OPS` when constructing the audit middleware (so `:llm/call` IS wrapped), but the raw terminator uses the smaller `_RAW_OPS` set (so `:llm/call` is NOT terminated by it — the LLM provider handler installed at `position="bottom"` answers it instead).

**Why two constants instead of subtracting at runtime:** explicit naming captures intent. A future reader sees "wrapped set" and "raw-terminator set" and immediately understands the distinction. Subtracting (`CANONICAL_AUDIT_RAW_OPS = tuple(set(WRAPPED) - {":llm/call"})`) hides the structural difference behind a derived value.

**Test surface:** extend the existing `test_canonical_audit_stack_covers_phase_2_ops` membership test to include `:llm/call`; add `tests/effect/test_audit_stack_llm_call.py` — install canonical stack + a mock `:llm/call` handler at bottom, perform a call, assert (a) `audit-canonical-raw` does NOT handle `:llm/call` (b) the audit middleware wraps the call and emits an `:audit/llm.call` entry (c) the chain hash extends correctly. ~70 LOC.

**Backwards compatibility:** existing `Substrate.open()` callers automatically get `:llm/call` wrapping. If they have a custom `:llm/call` handler already pushed onto the runtime, it now gets audit-wrapped — observable change (an extra audit entry per call). Documented as a new line in `CHANGELOG-effect.md`. Not a breaking change for any existing public API.

#### 3.0.3 The runtime stack a coder CLI run sees

```python
from __future__ import annotations
from dataclasses import dataclass
from typing import Any, Literal, Mapping

DecisionKind = Literal["act", "plan", "branch"]

@dataclass(frozen=True)
class Observation:
    """Substrate read result — empty in 2.1a/b; fields land 2.2a."""

@dataclass(frozen=True)
class LLMDecision:
    kind: DecisionKind
    confidence: float                   # [0.0, 1.0]
    payload: Mapping[str, Any]          # kind-dispatched shape; tightens 2.2a/2.3a/2.3b
```

`_prompt.py` (new file):

```python
"""Phase 2.1b — prompt construction + tool schema + text fallback parser.

`EMIT_DECISION_TOOL_SCHEMA` is the ONLY tool exposed to the LLM in 2.1b
per LD4 (decision/action split). Real effect tools (:fs/, :shell/, :code/,
:git/) land in 2.2a — never reachable by the LLM directly; the substrate
routes intents instead.
"""
from __future__ import annotations
import json
import re
from typing import Any

from ._types import Observation

EMIT_DECISION_TOOL_SCHEMA: dict[str, Any] = {
    "name": "emit_decision",
    "description": (
        "Emit your structured next-step decision for the persistence-coder agent. "
        "kind='act' for a single tool invocation; kind='plan' for a multi-step "
        "composition; kind='branch' if you are uncertain and want the agent to "
        "fork-and-explore. confidence is in [0.0, 1.0]; values below 0.65 trigger "
        "branch escalation."
    ),
    "input_schema": {
        "type": "object",
        "required": ["kind", "confidence", "payload"],
        "additionalProperties": False,
        "properties": {
            "kind": {"type": "string", "enum": ["act", "plan", "branch"]},
            "confidence": {"type": "number", "minimum": 0.0, "maximum": 1.0},
            "payload": {"type": "object"},  # 2.1b: loose; 2.2a tightens for kind="act"
        },
    },
}

def build_messages(task: str, obs: Observation) -> list[dict[str, Any]]:
    """Construct the LLM message list. 2.1b: minimal (user with task + emit_decision pointer).
    Observation fields land in 2.2a — when populated, observation context is appended."""
    return [
        {"role": "user", "content": f"Task: {task}\n\nUse emit_decision to respond."},
    ]

_DECISION_ENVELOPE_RE = re.compile(r"<decision>\s*(\{.*?\})\s*</decision>", re.DOTALL)

def parse_text_decision(text: str) -> dict[str, Any] | None:
    """Tier-2 fallback parser for `<decision>{json}</decision>` envelope.
    Returns None if envelope absent OR JSON invalid OR shape doesn't match
    the required {kind, confidence, payload} fields. Caller falls to tier 3."""
    if not (m := _DECISION_ENVELOPE_RE.search(text)):
        return None
    try:
        parsed = json.loads(m.group(1))
    except json.JSONDecodeError:
        return None
    if not isinstance(parsed, dict):
        return None
    if parsed.get("kind") not in {"act", "plan", "branch"}:
        return None
    try:
        confidence = float(parsed["confidence"])
    except (KeyError, ValueError, TypeError):
        return None
    if not (0.0 <= confidence <= 1.0):
        return None
    payload = parsed.get("payload", {})
    if not isinstance(payload, dict):
        return None
    return {"kind": parsed["kind"], "confidence": confidence, "payload": payload}
```

```python
        import anthropic

        raise RuntimeError(
            "anthropic SDK not installed. Install with: pip install 'anthropic>=0.40,<1.0'"
```

### 3.4 `_decide` body (replaces the 2.1a stub)

R1 fix-pass F2: uses the real `DB.transact` shape (`list[dict]` with `e/a/v/valid_from`, per `fact/db.py:196-207`) and the real `canonical_dumps` helper from `persistence.effect.canonical` (the closest existing canonicalizer; the design's earlier reference to a non-existent `_canonical_json` was wrong).

```python
def _decide(self, obs: Observation) -> LLMDecision:
    import uuid
    import datetime as dt
    from persistence.effect.canonical import canonical_dumps
    from ._prompt import EMIT_DECISION_TOOL_SCHEMA, build_messages, parse_text_decision

    messages = build_messages(self.task, obs)
    tools = [EMIT_DECISION_TOOL_SCHEMA]
    call_id = uuid.uuid4().hex
    now = dt.datetime.now(dt.timezone.utc)

    # Provenance BEFORE the call — captured even if perform() raises.
    # Single-attribute denormalized form per LD3-vocabulary (R1 fix-pass F4):
    # the 2.1a CHANGELOG-coder.md committed to ":llm/messages" and
    # ":llm/decision" as the datom attribute names, and we honor it.
    self.substrate.fact.transact([
        {
            "e": call_id,
            "a": ":llm/messages",
            "v": canonical_dumps({
                "messages": messages,
                "tools": tools,
                "model": self.model,
            }),
            "valid_from": now,
        },
    ])

    result = self.substrate.effect.perform(
        ":llm/call",
        model=self.model,
        messages=messages,
        tools=tools,
    )

    decision, parsed_via = self._parse_decision(result)

    decision_id = uuid.uuid4().hex
    self.substrate.fact.transact([
        {
            "e": decision_id,
            "a": ":llm/decision",
            "v": canonical_dumps({
                "kind": decision.kind,
                "confidence": decision.confidence,
                "payload": dict(decision.payload),
                "parsed_via": parsed_via,
                "source_call": call_id,
            }),
            "valid_from": now,
        },
    ])
    return decision

def _parse_decision(self, result: dict) -> tuple[LLMDecision, str]:
    """Two-tier parsing per LD3 + tier-3 missing-confidence default."""
    # Tier 1: tool-use
    if result.get("tool_calls"):
        try:
            payload = result["tool_calls"][0]["input"]
            return (LLMDecision(
                kind=payload["kind"],
                confidence=float(payload["confidence"]),
                payload=payload.get("payload", {}),
            ), "tool_use")
        except (KeyError, ValueError, TypeError):
            pass  # malformed tool call → fall through
    # Tier 2: text fallback
    if (parsed := parse_text_decision(result.get("text", ""))) is not None:
        return (LLMDecision(**parsed), "text_fallback")
    # Tier 3: missing-confidence default → branch-escalates per base § 3.4
    return (LLMDecision(
        kind="act",
        confidence=self.missing_confidence_default,  # 0.5 < 0.65
        payload={"raw_text": result.get("text", "")},
    ), "missing_default")
```

### 3.5 Datom shapes

R1 fix-pass F4: vocabulary aligned with 2.1a CHANGELOG-coder.md promise. The 2.1a CHANGELOG committed to "first datoms (`:llm/messages`, `:llm/decision`)". The design uses **exactly those two attribute names**, single-attribute-per-datom (denormalized form: the structured payload is canonical-JSON serialized into the value).

**`:llm/messages`** — one entity per call, value carries the full prompt envelope:

| Attribute | Value (canonical-JSON of dict with these fields) | Purpose |
|---|---|---|
| `:llm/messages` | `{"messages": list[dict], "tools": list[dict], "model": str}` | prompt-as-data; provenance for replay; entity-id is a fresh uuid hex per call |

**`:llm/decision`** — one entity per decision, value carries the parsed structured decision:

| Attribute | Value (canonical-JSON of dict with these fields) | Purpose |
|---|---|---|
| `:llm/decision` | `{"kind": "act"\|"plan"\|"branch", "confidence": float, "payload": dict, "parsed_via": str, "source_call": str}` | structured decision + provenance back to prompt; `source_call` is the entity-id of the matching `:llm/messages` datom |

The `parsed_via` field inside the value is `"tool_use"` / `"text_fallback"` / `"missing_default"` — substrate-side observable signal for "did the LLM behave?" without re-parsing logs. The `source_call` FK lets `s.fact.q` joins materialize full provenance.

**Why single-attribute (denormalized) instead of multi-attribute (normalized).** Two reasons:

1. **Honors the 2.1a CHANGELOG promise verbatim.** That CHANGELOG promised `:llm/messages` and `:llm/decision`; ARIS R1 F4 caught that the earlier multi-attribute form `:llm.call/*` / `:llm.decision/*` was vocabulary drift. v0.9.0a1 ships persistence-coder's first datoms — locking in the exact names the CHANGELOG promised avoids re-tagging the alpha.
2. **Matches the `:plan/ast` precedent.** Plan AST is stored as a single canonical-JSON-serialized value, not as a per-field datom-set. Same reasoning: the value is typically read back atomically; query-by-subfield is rare in dogfood; the audit chain hashes the whole value cleanly. If 2.4c lockfile dogfood reveals query-shape needs, we can split into per-field attributes in a v0.9.x track without re-tagging v0.9.0a1.

**Trade-off acknowledged.** `s.fact.q` against `:llm/decision` cannot index on `confidence` or `kind` directly without parsing the value. For 2.1b this is acceptable: the agent loop reads decisions back via the in-memory `LLMDecision` returned from `_decide`, not via `s.fact.q`. Substrate-side queries that need confidence/kind indexing land as a v0.9.x normalization track if and when dogfood demands it.

**Datom attribute canonicalization note.** Per `fact/datom.py:159-177`, leading `:` is stripped from `a` at construction time — `Datom(a=":llm/messages")` and `Datom(a="llm/messages")` produce identical in-memory values; the wire form re-prepends `:` uniformly. The design writes the colon-prefixed form for readability.

```python
The CLI's --provider auto path tries claude-code first (Max sub), anthropic
second (paid API), echo handler third (smoke-test floor). Explicit
--provider=<name> errors loudly if the chosen provider isn't available.

from __future__ import annotations
import os
import sys
from typing import Any

from persistence.effect.runtime import Handler

ProviderName = str  # "claude-code" | "anthropic" | "echo"

def detect_or_explicit(provider: str) -> tuple[Handler | None, ProviderName]:
    """Returns (handler-to-install, provider_name).
    handler is None for 'echo' floor (caller installs make_echo_llm_handler).
    Raises SystemExit on explicit-provider unavailability."""

    if provider in ("claude-code", "auto"):
        if _claude_code_available():
            from persistence.effect.handlers import make_claude_code_llm_handler
            return make_claude_code_llm_handler(), "claude-code"

                "installed (pip install claude-agent-sdk). "
                "Note: signed-out state is detected lazily on first "
                "LLM call, not at startup."
```

## 5. Test strategy

Three tiers, mirroring the wider substrate's pattern:

### Tier 1 — Pure unit (fast, deterministic, runs in CI without secrets)

- `tests/effect/handlers/test_callable_handler.py` — `make_callable_llm_handler` with mocked `call_fn`. Asserts catalog-shape pass-through.
- `tests/effect/handlers/test_provider_translation_contract.py` — parametric across all three factories. Each is loaded with a deterministic fake (anthropic-shaped fake response → `make_anthropic_llm_handler` with patched `anthropic.Anthropic`; claude-agent-sdk-shaped fake → `make_claude_code_llm_handler` with patched `query`; raw fake → callable). Asserts each handler produces identical catalog wire shape for matching inputs.
- `tests/coder/test_decide.py` — `_decide` body coverage. Uses `make_callable_llm_handler` with fixture-controlled `call_fn` to inject all tier-1/tier-2/tier-3 paths. Asserts datom emissions, `parsed-via` attribution, source-call FK linkage.
- `tests/coder/test_provider_detection.py` — `detect_or_explicit` parametric matrix.
- `tests/coder/test_prompt_schema.py` — `EMIT_DECISION_TOOL_SCHEMA` structural assertions (R1 fix-pass F7: NO `jsonschema` dependency added — the test uses manual structural asserts: required-keys present, `kind` is enum-of-three, `confidence` is number-with-min-max, `payload` is object-typed; the schema is short enough that this is cheaper than adding a dep). `parse_text_decision` parametric over valid/invalid envelopes uses pure assertions on returned dict shape.
- `tests/coder/test_decide_replay.py` — recorded `_decide` call → `s.replay.replay` → byte-identity assertion (G3).

### Tier 2 — Integration with real SDKs (skip-marked when SDK absent)

- `tests/effect/handlers/test_anthropic_handler.py` — `@pytest.mark.skipif(not _anthropic_available(), reason="anthropic SDK not installed")`. With SDK present but no API key → asserts the `RuntimeError` envelope; with key present (CI secret or local env) → makes one real call and asserts catalog-shape return.
- `tests/effect/handlers/test_claude_code_handler.py` — same pattern, gated on `claude-agent-sdk` availability.

Real-API tests are NOT required to pass in CI without secrets. They're falsifiable signals during local dev / when CI is configured with secrets.

### Tier 3 — Property tests (Hypothesis)

- `tests/coder/test_decide.py::test_parse_decision_total_function` — `@given(strategies for catalog-shape result)`, `@max_examples=200`. Asserts `_parse_decision` is total: every input shape produces exactly one of three `parsed_via` outcomes; no exceptions escape; returned `LLMDecision` always satisfies invariants (`kind ∈ {act, plan, branch}`, `0 ≤ confidence ≤ 1`).

### What is NOT tested

- **End-to-end `Coder.run()` success with a real LLM.** `run()` still raises `CoderStubNotImplemented("Phase 2.2a — substrate read via s.fact.q")` because `_observe` is a stub. End-to-end success is 2.2a/b deliverable.
- **Long-running session behavior, retry under flaky vendor, rate-limit interaction.** Those handler stack behaviors are already tested at the substrate layer; 2.1b's handlers slot into the existing stack and inherit coverage.

## 6. Risks & open questions

### Risks

**R1 — `claude-agent-sdk` API shape unverified at design time.** § 3.3 sketches the handler against a plausible API surface but does not pin the actual call signature (`query(prompt=, tools=, ...)` may not be the literal shape). **Mitigation:** during impl planning (writing-plans skill), use `mcp__plugin_context7_context7__query-docs` to fetch the SDK's actual API. If the SDK has only a single-prompt-string surface (no `messages` list, no native `tools` array), the Mode 2 handler will need a prompt-flattening adapter that may degrade tool-use to text fallback — this is **acceptable** (tier 2 handles it). Falsifiable signal: if `tool_calls` is empty 100% of the time on Mode 2, the SDK doesn't support structured output and we route through tier 2.

**R2 — Anthropic SDK version drift.** The `anthropic` package's `messages.create()` API has shifted across versions. **Mitigation:** pin a minimum version in the lazy-import error message; version-test against `anthropic>=0.40,<1.0` (or the current stable line at impl time).

**R3 — Echo handler installed by default in `--provider auto` floor case.** A test in `tests/coder/test_provider_detection.py` could inadvertently mask a missing-handler path. **Mitigation:** the auto-floor explicit stderr warning ("warning: no LLM provider available — using echo handler") is deliberate UX; tests assert the stderr line is emitted.

**R4 — Datom-shape lock-in before MCP server contract.** `:llm/messages` and `:llm/decision` datoms become permanent substrate vocabulary in v0.9.0a1. If MCP server work (SDK3/SDK4) wants different shapes (e.g. per-field normalized attributes for query indexing), we either rewrite or add an alias layer. **Mitigation:** the names match the 2.1a CHANGELOG-coder.md promise verbatim; the value-shape is canonical-JSON-stringified structured data (matches `:plan/ast` precedent); flag for explicit MCP review before v0.9.0a1 tags.

**R5 — Hidden coupling between `Coder.confidence_threshold` (0.65) and `missing_confidence_default` (0.5).** A future change to the threshold (e.g. 0.5 in 2.4a tuning) would silently break the tier-3 fallback's branch-escalation guarantee. **Mitigation:** add a unit test that asserts `missing_confidence_default < confidence_threshold` as a class invariant; flag in 2.4a's plan that any threshold change must be accompanied by a default change.

### Open questions (closeable during impl plan or impl)

**OQ1 — Should `:llm/messages` and `:llm/decision` values be canonical-JSON strings (single denormalized attribute, current design) or per-field structured datoms (multi-attribute normalized form)?** The 2.1b design picks single-attribute denormalized form per § 3.5 (matches CHANGELOG-coder.md promise + `:plan/ast` precedent). Trade-off: cannot index on subfields (e.g. `confidence`, `kind`) via `s.fact.q` without parsing the value. **Default:** denormalized form for 2.1b; revisit in 2.4c lockfile if dogfood reveals query-shape needs (would land as a v0.9.x normalization track).

**OQ2 — Where does `Coder.model` default come from?** Currently `model: str = "claude-opus-4-7"` as constructor default. Alternative: read from env var `PERSISTENCE_CODER_MODEL` or `--model` CLI flag with no constructor default (forces explicit choice). **Default:** keep constructor default; CLI `--model` overrides; env var deferred (not yet justified).

**OQ3 — Should the auto-floor echo case exit non-zero immediately, or proceed to `Coder.run()`?** Currently proceeds (and raises `CoderStubNotImplemented` on `_observe()` first). Alternative: exit 1 with the warning, never construct Coder. **Default:** proceed — preserves the "smoke-test the CLI without LLM auth" affordance and exit-code semantics match 2.1a (1 on stub, 0 on success).

**OQ4 — Should `_provider.py`'s `_claude_code_available()` actually attempt session-handshake or just check importability?** Currently checks importability only (cheaper, doesn't burn API quota on probe). The actual session-reachability error surfaces lazily on first call as a clean `RuntimeError`. **Default:** importability-only; revisit if first-call failures become a frequent UX paper-cut during dogfood.
