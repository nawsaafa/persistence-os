## 1. Why this exists as a separate phase

Phase 2.1b shipped `_decide` (LLM-call + `:llm/decision` datom emission) but left every other behavioral method on `Coder` as `CoderStubNotImplemented`-raising stubs. `run()` is single-iter and crashes on the first `_observe()` call. The agent has no operational footprint — it can decide, but it can neither read substrate state nor act on the world.

Phase 2.2a closes that gap with the **smallest substrate-side surface that lets the coder run a real ReAct loop end-to-end:** the four `:fs/*` ops (`read`, `write`, `glob`, `grep`) and `:shell/exec` per the canonical Phase 2 design § 6 line 373 — plus the coder body changes that put them to use. `:code/exec` already exists from 2.0b; `:git/*` is deferred to Phase 2.2b per the same design row.

The phase is intentionally separate from 2.1b (which was the LLM-provider phase, no effect-tool surface) and from 2.2b (which adds `:git/*` and wires existing `:code/exec` into `_act` dispatch). Splitting keeps each ARIS pass under a tight scope (~5 new handlers + coder body) and matches the existing Phase 2.0a/0b/0c/1a/1b/1c/1c.6 cadence.

## 2. Falsifiable acceptance signal (load-bearing)

A new strict-xfail test lands in `tests/coder/test_loop_e2e.py` at design-time:

```python
@pytest.mark.xfail(strict=True, reason="Phase 2.2a — _observe/_act bodies + fs/shell handlers")
def test_coder_loop_runs_two_iters_emits_act_results():
    """End-to-end: coder reads a file, then writes a summary; ≥2 iters; audit chain advances."""
    ...
```

The test:
1. Opens a substrate with `audit=True`
2. Installs `make_callable_llm_handler` with a scripted call_fn that emits two decisions: iter 1 → `kind=act, payload={"op":":fs/read","args":{"path":"<scratch>/input.txt"}}`; iter 2 → `kind=act, payload={"op":":fs/write","args":{"path":"<scratch>/out.txt","content":"summary"},"done":true}`
3. Installs `make_fs_handler(project_root, scratch_dir)` + `make_shell_handler(allowlist, env_passthrough)`
4. Runs `Coder(task="...", substrate=s).run()`
5. Asserts: `≥2 :llm/decision` datoms exist; `≥1 :fs/read` request+result pair; `≥1 :fs/write` request+result pair; audit chain advances by ≥6 entries; final loop exit via `payload.done=true`

This test is the load-bearing acceptance signal — it currently FAILs because `_observe` raises `CoderStubNotImplemented` and the `:fs/*` ops don't exist. After 2.2a ships, the test PASSes. The strict-xfail decorator is removed in the final 2.2a commit (T9 below); CI catches any regression that re-breaks the loop.

## 3. Approach (locked after design pass)

### 3.1 Substrate-side additions (LD7 — handlers land first)

**Five new effect handlers in `src/persistence/effect/handlers/`:**

| File | Op | Args (canonical-JSON) | Result (canonical-JSON) | Audit-replay action |
|---|---|---|---|---|
| `fs.py` | `:fs/read` | `{path, encoding?}` | `{bytes_or_text, mtime, size, sha256}` | read recorded result; do not touch FS |
| `fs.py` | `:fs/write` | `{path, content, mode?}` | `{bytes_written, sha256_after}` | read recorded result; do not touch FS |
| `fs.py` | `:fs/glob` | `{pattern, root, flags?}` | `{matches: [...]}` | read recorded result |
| `fs.py` | `:fs/grep` | `{pattern, root, flags?}` | `{matches: [...]}` | read recorded result |
| `shell.py` | `:shell/exec` | `{argv, env_allowlist_subset, cwd, allowlist_version, timeout_s?}` | `{exit, stdout, stderr, wall_clock_ms}` | read recorded result; do not spawn subprocess |

Each is a `make_*_handler(...)` factory mirroring the Phase 2.1b LLM-handler pattern. Handler factories take **explicit capability arguments**:

- `make_fs_handler(project_root: Path, scratch_dir: Path) -> Handler`
- `make_shell_handler(allowlist: frozenset[str] = ALLOWLIST_V1, env_passthrough: frozenset[str] = ENV_DEFAULT) -> Handler`

Capability denial is enforced at the handler level (NOT detection):
- `:fs/read`: path must resolve under `project_root` (after `Path.resolve()`); else raises `FsCapabilityDenied`
- `:fs/write`: path must resolve under `scratch_dir`; else raises `FsCapabilityDenied`
- `:fs/glob` / `:fs/grep`: `root` must resolve under `project_root`; else raises
- `:shell/exec`: `argv[0]` must be in `allowlist`; argv must be `list[str]` (never a string — no shell metacharacter parsing); subprocess spawned with `shell=False`; `cwd` is **explicit-required** (no default); env passed is `{k: os.environ[k] for k in env_allowlist_subset if k in os.environ}`; default timeout 30 s; SIGKILL past timeout

**Allowlist constants in `shell.py`:**

```python
ALLOWLIST_V1: frozenset[str] = frozenset({
    "ls", "cat", "echo", "head", "tail", "wc",
    "grep", "find", "git",
    "python", "python3", "uv", "pip", "pytest",
})

ENV_DEFAULT: frozenset[str] = frozenset({"PATH", "HOME", "LANG", "LC_ALL", "LC_CTYPE"})

def _allowlist_version(allowlist: frozenset[str]) -> str:
    return hashlib.sha256(canonical_dumps(sorted(allowlist)).encode()).hexdigest()[:16]

ALLOWLIST_VERSION: str = _allowlist_version(ALLOWLIST_V1)
```

The `:shell/exec` request datom carries `allowlist_version`. Audit-replay reads it; if substrate's current `_allowlist_version(ALLOWLIST_V1)` differs (because someone added/removed a stem), audit-replay raises `ShellAllowlistVersionMismatch` rather than silently passing — matches the design § 3.7 contract.

**Wired into canonical audit stack** (`src/persistence/effect/_audit_stack.py`):

```python
CANONICAL_AUDIT_WRAPPED_OPS: tuple[str, ...] = (
    # ... existing entries ...
    # Phase 2.2a — fs/shell effect ops (substantive return; bottom-of-stack
    # handlers installed via s.effect.install_handler).
    ":fs/read", ":fs/write", ":fs/glob", ":fs/grep",
    ":shell/exec",
)
```

These ops are **NOT** added to `CANONICAL_AUDIT_RAW_OPS` — their bottom-of-stack handler IS the substantive handler (the `make_fs_handler` / `make_shell_handler` output) which produces the real return value. The pattern matches `:llm/call` (wrapped, but raw terminator excluded; provider handler installed separately at `position="bottom"`).

### 3.2 Coder body changes (LD8 — body lands AFTER handlers)

**`Coder._observe` body** (`src/persistence/coder/_session.py`):

```python
def _observe(self) -> Observation:
    """Read recent decision + action history from substrate.

    Reads via `s.fact.since(self._session_start_tx)` then filters by
    attribute; takes trailing N=5 of each. The substrate is the source
    of truth — Python in-memory state is intentionally not threaded
    through the loop body, so replay reconstructs identical observations.
    """
    snapshot = self.substrate.fact.since(self._session_start_tx)
    decisions = [d for d in snapshot if d.a == ":llm/decision"][-self.observe_depth:]
    actions = [d for d in snapshot if d.a == ":act/result"][-self.observe_depth:]
    return Observation(
        iter_count=self._iter_count,
        recent_decisions=tuple(_decode_value(d.v) for d in decisions),
        recent_actions=tuple(_decode_value(d.v) for d in actions),
    )
```

`Coder.observe_depth: int = 5` is a class field, hardcoded for 2.2a; promoted to CLI flag during 2.4a tuning. `Coder._session_start_tx: int` is captured in `__post_init__` via `s.fact.since(0)` to seed the trailing window (or via `s._db.next_tx()` if available — verified at impl time).

**`Coder._act` body**:

```python
def _act(self, decision: LLMDecision) -> None:
    """Dispatch the decision payload to its named effect op + transact :act/result."""
    if decision.kind != "act":
        raise ValueError(f"_act called with kind={decision.kind!r}")  # caller guarantees kind=act
    op = decision.payload.get("op")
    args = decision.payload.get("args", {})
    if not isinstance(op, str) or not op.startswith(":"):
        raise ValueError(f"malformed act payload: missing/invalid op {op!r}")

    t0 = dt.datetime.now(dt.timezone.utc)
    try:
        result = self.substrate.effect.perform(op, args)
        error = None
    except Exception as e:
        result = None
        error = f"{type(e).__name__}: {e}"
        # Provenance survives — :act/result still transacted below; then re-raise.

    t1 = dt.datetime.now(dt.timezone.utc)
    latency_ms = int((t1 - t0).total_seconds() * 1000)

    result_summary = _summarize_result(result) if result is not None else None
    self.substrate.fact.transact([
        {
            "e": uuid.uuid4().hex,
            "a": ":act/result",
            "v": canonical_dumps({
                "op": op,
                "args_hash": _canonical_hash(args),
                "result_summary": result_summary,
                "error": error,
                "latency_ms": latency_ms,
            }),
            "valid_from": t0,
        },
    ])

    if error is not None:
        raise  # propagate after :act/result is recorded
```

`_summarize_result(r)` truncates large blobs (`bytes_or_text` to first/last 256 chars; `stdout`/`stderr` to first/last 1 KB) to keep the prompt cheap when `_observe` re-reads `:act/result` next iter. Full result remains in the audit chain.

**`Coder.run` body widens to a bounded for-loop:**

```python
def run(self) -> None:
    self._session_start_tx = self.substrate._db.next_tx()  # snapshot loop entry
    for self._iter_count in range(self.max_iters):
        obs = self._observe()
        decision = self._decide(obs)
        if self._should_escalate_branch(decision):
            self._escalate_branch(decision)  # raises CoderStubNotImplemented in 2.2a
            return
        if self._should_escalate_plan(decision):
            self._escalate_plan(decision)    # raises CoderStubNotImplemented in 2.2a
            return
        if decision.payload.get("done"):
            return
        self._act(decision)
        # _check_pause skipped — re-added in Phase 2.3d
    # for-else: max_iters reached without explicit done → log + return
```

`Coder.max_iters: int = 20` field; CLI flag `--max-iters` overrides for tests. Loop counter is a coder field (`self._iter_count`) so `_observe` can surface it in the Observation.

**`_should_escalate_*` gates filled as one-liners**:

```python
def _should_escalate_plan(self, decision: LLMDecision) -> bool:
    return decision.kind == "plan"

def _should_escalate_branch(self, decision: LLMDecision) -> bool:
    return decision.kind == "branch" or decision.confidence < self.confidence_threshold
```

`_escalate_plan` and `_escalate_branch` keep their `CoderStubNotImplemented` raises (2.3a / 2.3b territory). The gates are filled so routing is correct; the heavy lifting stays stubbed.

**`_check_pause`** is **NOT called from `run()` in 2.2a** — re-added in Phase 2.3d when the REPL session class lands. The stub remains in `_session.py` for the 2.3d sub-phase to fill. Documented inline.

### 3.3 Schema changes

**`Observation` body** (`src/persistence/coder/_types.py`):

```python
@dataclass(frozen=True)
class Observation:
    iter_count: int
    recent_decisions: tuple[Mapping[str, Any], ...]   # ≤observe_depth
    recent_actions: tuple[Mapping[str, Any], ...]     # ≤observe_depth
```

**`Coder` fields added** (`_session.py`): `max_iters: int = 20`, `observe_depth: int = 5`. Plus internal: `_session_start_tx: int = field(init=False)` and `_iter_count: int = field(init=False, default=0)`.

**`build_messages` extended** (`_prompt.py`): when `obs.recent_decisions` or `obs.recent_actions` non-empty, append a "Recent loop history" section to the user message with truncated JSON dumps of the last N items. Schema stable; no tool-schema change.

**No changes to `LLMDecision`** — `payload.done: bool` is a loose convention (no schema break). If 2.4a tuning wants `kind="done"` we promote then.

**No new curated SDK methods** — `s.fact.since(...)` already shipped at v0.8.0a1; `s.effect.perform(...)` already exists. Per the design § 3.2 SDK-gap protocol, no SDK addition needed.

### 3.4 Op argument shapes (canonical, append-only contract)

**`:fs/read`**:
- args: `{path: str, encoding: str = "utf-8"}` — `path` resolves under `project_root` or raises `FsCapabilityDenied`
- result: `{bytes_or_text: str | bytes, mtime: float, size: int, sha256: str}`

**`:fs/write`**:
- args: `{path: str, content: str | bytes, mode: Literal["w","wb","a","ab"] = "w"}` — `path` resolves under `scratch_dir`
- result: `{bytes_written: int, sha256_after: str}`

**`:fs/glob`**:
- args: `{pattern: str, root: str, recursive: bool = True}`
- result: `{matches: list[str]}` — canonical-sorted

**`:fs/grep`**:
- args: `{pattern: str, root: str, fixed_string: bool = False, ignore_case: bool = False}`
- result: `{matches: list[{path: str, line: int, text: str}]}` — canonical-sorted by `(path, line)`

**`:shell/exec`**:
- args: `{argv: list[str], cwd: str, env_allowlist_subset: list[str], allowlist_version: str, timeout_s: float = 30.0}`
- result: `{exit: int, stdout: str, stderr: str, wall_clock_ms: int}`

**`:act/result`** (substrate fact, not an effect op):
- value (canonical-JSON): `{op: str, args_hash: str, result_summary: Any | None, error: str | None, latency_ms: int}`

All argument and result keys are sorted in canonical-JSON; this is the audit-replay byte-identity contract.

## 5. ARIS posture — pre-mortem topics

### 5.1 New op surface coupling risk

Adding 5 ops to `CANONICAL_AUDIT_WRAPPED_OPS` has the same coupling shape as 2.1c.6 (`:claim/emit` + `:blob/put`). The audit middleware fans out from the wrapped set automatically. Risk: `tests/effect/test_canonical_audit_stack.py` drift-pin needs an update; otherwise it asserts the OLD set membership. Folded as a T1 task.

### 5.2 Allowlist version drift

If a future task adds `"docker"` or `"node"` to `ALLOWLIST_V1`, the sha256 hash changes, and any existing replay of a `:shell/exec` trajectory raises `ShellAllowlistVersionMismatch`. This is the design intent (audit-replay catches stale-allowlist replay). Test gate G2 includes a positive case (recorded version matches) AND a negative case (modified ALLOWLIST_V1 raises on replay).

### 5.3 Loop termination correctness

The `for _ in range(max_iters):` cap is the universal safety. Three exit paths:
- `payload.done=true` → return (cleanest)
- `_escalate_plan` / `_escalate_branch` → raise `CoderStubNotImplemented` (2.2a behavior; honest stub)
- max_iters exhausted → return (for-else; logged via `:loop/exhausted` fact? Or no fact, just return?)

**Locked decision:** no `:loop/exhausted` fact in 2.2a. If the loop runs out of iters, `run()` returns silently. The audit chain shows N decision+action pairs and stops. Adding a "loop exhausted" sentinel datom is a 2.4a tuning concern, not a 2.2a hard requirement.

### 5.4 Path capability constraints (chroot-style)

`Path.resolve()` follows symlinks; the constraint check must use the resolved path. A symlink under `project_root` pointing outside is allowed by the design (the resolved target is what matters; if the resolved target is outside, the constraint denies). `Path.resolve(strict=True)` raises `FileNotFoundError` for missing paths — which is caller-helpful for `:fs/read` (reading a missing file should fail loudly).

For `:fs/write` to a not-yet-existing scratch path: `Path.resolve(strict=False)` then `.is_relative_to(scratch_dir.resolve())` is the constraint shape. T2 verifies.

### 5.5 Replay byte-identity for fs/shell

The audit middleware records `{request, result}` pairs into `_canonical_audit_entries`. Replay reads back the recorded result without invoking the handler. Byte-identity holds as long as canonical-JSON serialization is stable across replays. Risk: `bytes_or_text` field — if the file is binary, base64-encoding on the wire is required for canonical-JSON. **Locked decision:** `:fs/read` returns `bytes_or_text: str` (decoded per `encoding` arg, default utf-8); binary reads via opt-in `encoding="binary"` returning a base64-encoded string. T2 verifies.

### 5.6 Observation read-after-write hazard

`_observe` reads `s.fact.since(self._session_start_tx)`. The previous `_act` in iter N-1 transacted `:act/result` after the perform completed. By iter N's `_observe` call, the `:act/result` is in the fact log — `since(...)` returns it. No concurrent transactor under test; the bare-string `since` query is sequential read-your-writes consistent. Verified.

If a future caller threads multiple coders into the same substrate concurrently, the `since(...)` window may include other coders' decisions in iter N's observation — that's a 2.4c+ multi-coder scoping concern; documented as out-of-scope here.

### 5.7 Plan/branch routing escalation stub semantics

`_should_escalate_plan` returns True when `decision.kind == "plan"`. `_escalate_plan` raises `CoderStubNotImplemented`. Net behavior: a `kind=plan` decision halts the loop with a clear error message. Testable as: scripted call_fn returns `kind=plan` → loop exits via raise; `:llm/decision` was transacted before the raise (provenance); no `:act/result` for the plan decision (correct — `_act` was never called for it).

### 5.8 ARIS-the-spec built-in step (per Mimir D.0 / mimir-design SKILL v0.1.1)

The Mimir D.0 dogfood landed a "ARIS-the-spec" built-in pattern: the design doc itself is reviewed before any implementation. This is that step. Findings from R0 land as inline folds before any worktree commits in `src/`.

## 8. Risks + mitigations

| # | Risk | Mitigation | Where folded |
|---|---|---|---|
| R1 | Canonical-JSON of `bytes` (binary file reads) breaks audit-replay byte-identity | `:fs/read` returns `str`; binary mode → base64-encoded str. T2 verifies. | § 5.5, G1 |
| R2 | Symlink escape from `project_root` / `scratch_dir` | `Path.resolve(strict=False).is_relative_to(...)` after symlink follow | § 5.4, G1 |
| R3 | `subprocess.run` with `shell=False` + argv list — accidentally allowing argv injection if argv contains shell metacharacters | argv is `list[str]`; `shell=False` is structurally safe (no shell interpretation regardless of metacharacters) | § 3.1, G2 |
| R4 | `_observe` returns datoms in non-tx order under bare-string `since(...)` | Verified at impl time; if order isn't tx-ordered, sort by `tx` field on the datom | § 5.6, G3 |
| R5 | LLM emits `payload.done=true` AND `payload.op=...` simultaneously | Locked: `payload.done=true` short-circuits BEFORE `_act` is called; `op` field ignored if done is set | § 3.2 run body |
| R6 | Lockfile drift breaks `test_sdk_consumer.py` if anyone forgets to re-snapshot | Drift-pin G7 catches any lockfile mismatch; re-snapshot is T9 | § 6 G7 |
| R7 | `s.fact.since(self._session_start_tx)` returns ALL datoms not just `:llm/decision` and `:act/result` — Python-side filter is O(N) per iter | Bounded by max_iters × observe_depth; for max_iters=20 + ~5 datoms/iter, total filter is ~100 datoms/iter; trivially fast | § 5.6 |
| R8 | Allowlist V1 may be incomplete — agent needs a stem we forgot | Bumping the set re-derives `ALLOWLIST_VERSION` automatically; old-trajectory replay raises `ShellAllowlistVersionMismatch` (correct behavior per design) | § 5.2, G2 |

## Locked design decisions (LD index)

- **LD1.** Approach A — fs+shell handlers per design § 6 line 373; not folding 2.2b (`:git/*`) in.
- **LD2.** Loop widens via for-range(max_iters); `payload.done=true` sentinel + escalation-as-terminator.
- **LD3.** Observation = `(iter_count, recent_decisions ≤5, recent_actions ≤5)` read via `s.fact.since(session_start_tx)`.
- **LD4.** New `:act/result` attr datomized per `_act` perform; fields `{op, args_hash, result_summary, error?, latency_ms}`.
- **LD5.** Allowlist = static module-level frozenset; version = `sha256(canonical_dumps(sorted(set)))`. Env subset = `PATH/HOME/LANG/LC_ALL/LC_CTYPE`. Cwd explicit-required. shell=False always. 30s default timeout. `git` stem on allowlist for ad-hoc.
- **LD6.** Capability constraints baked into handler factory args (`make_fs_handler(project_root, scratch_dir)`, `make_shell_handler(allowlist, env_passthrough)`).
- **LD7.** Substrate prereqs land FIRST (Tasks 1-3: drift-pin update + fs + shell handlers); coder body second (Tasks 4-7); drift-pin + lockfile + e2e tests last (Tasks 8-9).
- **LD8.** `_should_escalate_plan` and `_should_escalate_branch` filled as one-liners; `_escalate_plan` and `_escalate_branch` keep `CoderStubNotImplemented` raise; `_check_pause` skipped from `run()` for 2.2a (re-added 2.3d).
- **LD9.** New ops added to `CANONICAL_AUDIT_WRAPPED_OPS` (audit middleware wraps); NOT to `CANONICAL_AUDIT_RAW_OPS` (handlers have substantive return values).
- **LD10.** `:fs/read` returns `bytes_or_text: str`; binary reads via opt-in `encoding="binary"` returning base64-encoded str (canonical-JSON safety).
