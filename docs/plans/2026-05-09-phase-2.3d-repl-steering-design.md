## 1. Overview

Phase 2.3d ships `_CoderSteeringSession` — the operator-facing class that turns the persistence-coder ReAct loop into a **live debugger**. Seven ops on the session expose the agent's decision boundary to a REPL operator: `pause` / `resume` / `snapshot` / `context_at` / `branch` / `fold` / `commit`. Each op emits two audit-chained datoms (`:repl/request` + `:repl/response`); the trajectory replays byte-identically because directives are first-class recorded inputs (§ 3.7 row).

This is the most differentiated UX surface persistence-os has. Phase 2 design § 3.6 calls it "the demo we lead with" — a direct counterweight to Mem0 / MemGPT / Zep / Memento, none of which expose their agent's decision boundary to an operator while it runs.

**What 2.3d does NOT do:**
- No new substrate primitive. `db.branch()`, `db.fold()`, `db.fork()`, `make_audit_handler` are all shipped at HEAD.
- No `db.merge()` (merging a branch's writes back to main). Out of scope for v0.9.0a1; would require a `db.fork()` / merge primitive that doesn't exist (REPL design ADR-13 OQ-4).
- No SQLite-backed branch persistence. Branches are per-session-durable; lost on REPL session restart. W3-rescope to v0.9.x.
- No promotion of a branch's facts back to the parent store. `coder.commit(branch_id)` is a session pointer swap, not a write-replay.

## 2. Locked design decisions (LD0-LD5)

### LD0 — SCOPE: full § 3.6 (7 ops), no substrate touches

`_CoderSteeringSession` is a NEW class in `_steering.py` — distinct from the existing v0.7.0a1 REPL `Session` class in `repl/_session.py`. The REPL `Session` is frozen, transport-level (`session_id`, `token_id`, `cap_set`, view cursors, `parent_chain_depth`). `_CoderSteeringSession` is mutable, agent-level — it composes a REPL `Session` reference for capability gating but owns the agent's `active_db: DB`, `branches: dict[branch_id, DB]`, and `_pause_event: threading.Event`.

`_CoderSteeringSession` exposes **7 ops** (R0-fold B2 — `resume` is its own op, was undercounted in initial draft):
- `pause()` — clears the `threading.Event`; the loop blocks at iteration head until resume
- `resume()` — sets the `threading.Event`; the loop unblocks
- `snapshot(n: int = 50, branch_id: str = "active")` — returns the last `n` datoms; `branch_id` selects parent / child / active (R0-fold I2)
- `context_at(t: datetime, branch_id: str = "active")` — `branches[branch_id].as_of(t)` (R0-fold I2)
- `branch(directive)` — opens a counterfactual via `db.branch(fork_at, [directive_assertion])`; records `fork_at` in `:repl/request`
- `fold(probe)` — runs `probe(branch_db)` over **parent + all child branches**, returns dict of scores (R0-fold B4)
- `commit(branch_id)` — pointer swap: `session.active_db = branches[branch_id]`

All seven are session-side Python methods; the WebSocket transport surface (`:coder/*` ops registered in `repl/_ops.py`) is the dispatcher to them.

**Why:** § 3.6 specifies these. The user explicitly chose full scope (vs. minimal pause/snapshot only). Codex confirmed substrate prereqs are zero given the existing `db.branch()` primitive.

**Concurrency invariants (R0-fold N2):**
- Pause/resume are idempotent (multiple `pause()` calls on already-paused session = no-op).
- `pause()` requested mid-iteration takes effect at the NEXT `_check_pause()` call (iteration head, before `_observe()`).
- `commit()`, `branch()`, `fold()` are allowed while paused (mutate operator-side state; agent loop stays blocked).
- Single REPL connection per session in v0.9.0a1. Concurrent operator threads → W3-rescope to v0.9.x with xfail-strict acceptance signal `tests/coder/test_steering_concurrent_operators.py::test_two_operator_threads_serialize_via_session_lock`.

### LD1 — Branch durability via `db.branch()` (CONSENSUS-LOCKED via codex R1, 2026-05-09)

`coder.branch(directive)` calls `self.db.branch(fork_at, [directive_assertion])`. Branched DBs tracked in per-session `branches: dict[branch_id, DB]` on `_CoderSteeringSession`. `coder.commit(branch_id)` is a session-level pointer swap (`session.active_db = branches[branch_id]`); parent DB stays unmodified as a counterfactual. Branches are per-session-durable only — lost on REPL session restart.

**`fork_at` is an explicit recorded boundary** in the `:repl/request` payload, NOT wall-clock-during-replay. This closes the byte-identity replay gap codex flagged: replay re-injects the recorded `fork_at`, not a fresh wall-clock value.

**Codex's decider (the load-bearing argument):**
> § 3.6 requires "writes on the branch can never leak into the parent store" and "both branches survive in the bitemporal log" (`design.md:240`). `DB.branch` is the only in-scope primitive that *actually* enforces non-leakage by construction via a fresh `InMemoryStore` (`db.py:543-582`).

**Codex's gotcha:** naming/expectation collision with Module 7's `repl/branch` (which is "cursor + depth marker, NOT a store fork" per ADR-13). 2.3d's op is `:coder/branch` (different op family), recorded with `fork_at` to keep replay meaningful.

**Acceptance signal G2 (codex revised):**
```python
def test_db_branch_isolation_and_determinism(parent_db, directive_assertion):
    fork_at = now()
    b1 = parent_db.branch(fork_at, [directive_assertion])
    b2 = parent_db.branch(fork_at, [directive_assertion])
    # Isolation: parent unchanged by any branch writes
    parent_pre  = canonical_dumps([d.to_dict() for d in parent_db.as_of(fork_at).datoms])
    _ = b1.transact([{"e":"x","a":":k","v":"v","valid_from":fork_at}])
    parent_post = canonical_dumps([d.to_dict() for d in parent_db.as_of(fork_at).datoms])
    assert parent_pre == parent_post
    # Determinism: two branches from same fork_at have byte-identical prefixes
    b1_seed = canonical_dumps([d.to_dict() for d in b1.as_of(fork_at).datoms])
    b2_seed = canonical_dumps([d.to_dict() for d in b2.as_of(fork_at).datoms])
    assert b1_seed == b2_seed
```

**W3-rescope deferrals:**
- SQLite-backed branch persistence (durable across REPL session restart) → v0.9.x. Acceptance signal: `xfail-strict` test that asserts a branched DB survives session-tear-down + reload.
- `db.merge(branch_db, into=main_db)` — promote branch's facts back to parent → v0.10.x (depends on substrate-level merge primitive that doesn't exist).

### LD2 — Pause attach point: `Coder.run()` iteration head, sync `threading.Event` (R0-fold B1: semantics inverted from initial draft)

`_check_pause()` is called at the START of each iteration in `Coder.run()` (`_session.py:66`), BEFORE `_observe()` — so the agent's view at pause time matches the operator's view through `coder.snapshot()` / `coder.context_at(t)`.

**Implementation (R0-fold B1):** the `threading.Event` semantic convention is `set` = "go ahead", `clear` = "block." Default state is **set** (not paused). `pause()` calls `_pause_event.clear()`; `resume()` calls `_pause_event.set()`. `_check_pause()` calls `_pause_event.wait()` which blocks while cleared and returns immediately while set. The initial draft had this inverted — `pause()` would have returned immediately. Fixed:

```python
class _CoderSteeringSession:
    def __init__(self, ...):
        self._pause_event = threading.Event()
        self._pause_event.set()  # default: not paused (event SET = pass through wait())

    def pause(self) -> None:
        self._pause_event.clear()  # subsequent _check_pause() calls block

    def resume(self) -> None:
        self._pause_event.set()  # unblocks any _check_pause() currently waiting

    def _check_pause(self) -> None:
        self._pause_event.wait()  # blocks while cleared; returns immediately while set
```

If no session is attached, `Coder._check_pause()` is a no-op (zero perf cost — bare `if session is None: return` guard).

**Why threading.Event (not asyncio):** `Coder.run()` is sync (per `_session.py:66`). The REPL WSServer is asyncio (per `_ws.py`). The event sits on a session shared between threads; `threading.Event` is thread-safe for cross-thread signaling. `asyncio.Event` would require either making the coder loop async (large blast radius) or `asyncio.run_coroutine_threadsafe` ceremony.

**Acceptance signal G1 (R0-fold B1: must FAIL without the inverted-fix):**
```python
def test_pause_blocks_then_resume_unblocks(coder, session):
    t0 = time.monotonic()
    session.pause()
    threading.Timer(0.15, session.resume).start()  # resume after 150ms
    # Run coder in a worker thread so we can timer-resume from the main thread.
    worker = threading.Thread(target=coder.run)
    worker.start()
    worker.join(timeout=2.0)
    assert not worker.is_alive(), "coder.run() failed to return after resume"
    t1 = time.monotonic()
    assert t1 - t0 >= 0.15, f"pause didn't actually block (elapsed={t1-t0:.3f}s)"
    assert coder._iter_count >= 1, "coder didn't advance past iter 0 after resume"
```

### LD3 — Capability tokens extend `Capability(op, qualifier)` shape (R0-fold B5: representability proof)

ADR-3 of the v0.7.0a1 REPL design defines the closed primitive: `Capability(op, qualifier)` where `op: Literal["inspect", "edit", "rewind", "branch"]` and `qualifier: str`. 2.3d EXTENDS this set:

- `op` literal extended to include `"coder"` (new value).
- `qualifier` for `op="coder"`: `"read" | "write" | "any"`.

Mapping table (the closed set 2.3d adds — R0-fold B5 representability):

| coder method | Required `Capability(op, qualifier)` |
|---|---|
| `pause()`, `resume()` | `("coder", "read")` (pause is observation-side) |
| `snapshot()`, `context_at()` | `("coder", "read")` |
| `branch()`, `fold()` | `("coder", "write")` |
| `commit()` | `("coder", "write")` |

`("coder", "any")` is the union — used for tokens issued to a trusted operator who needs the full surface. `_caps.py` validates the closed set via `UnknownCapability` raise on unrecognized `(op, qualifier)` per ADR-3.

**Acceptance signal G6 (R0-fold B5: schema-level representability test FIRST, then per-op denial):**
```python
def test_coder_caps_serialize_and_deserialize_via_existing_primitive():
    # B5 schema-level: prove (op="coder", qualifier=...) round-trips through _caps.py.
    cap_read  = Capability(op="coder", qualifier="read")
    cap_write = Capability(op="coder", qualifier="write")
    cap_any   = Capability(op="coder", qualifier="any")
    cap_set = CapabilitySet([cap_read, cap_write, cap_any], expires_at=...)
    blob = cap_set.to_token_blob()
    restored = CapabilitySet.from_token_blob(blob)
    assert restored == cap_set
    # Negative: unknown qualifier raises
    with pytest.raises(UnknownCapability):
        Capability(op="coder", qualifier="lol")

@pytest.mark.parametrize("op_name,required", [
    ("pause", ("coder", "read")),
    ("resume", ("coder", "read")),
    ("snapshot", ("coder", "read")),
    ("context_at", ("coder", "read")),
    ("branch", ("coder", "write")),
    ("fold", ("coder", "write")),
    ("commit", ("coder", "write")),
])
def test_coder_op_requires_capability(session_no_cap, op_name, required):

```

### LD4 — Audit chain integration via `:repl/request` / `:repl/response` (R0-fold N1: naming table; R0-fold I3: `fork_at` source pinned; R0-fold I4: G5 stream filter)

Each steering op emits **two audit-chained datoms**:
- `:repl/request` — operator-side directive payload (includes `fork_at` for branch ops, `branch_id` for fold/commit/snapshot/context_at, `n` for snapshot, etc.)
- `:repl/response` — agent-side result (includes new `branch_id` for branch, scores dict for fold, etc.)

Routed through the existing `make_audit_handler` stack — Merkle-linked, no new audit infra.

**Naming mapping table (R0-fold N1: 3 distinct naming scopes):**

| Scope | Pause | Resume | Snapshot | Context_at | Branch | Fold | Commit |
|---|---|---|---|---|---|---|---|
| WS JSON-RPC method | `coder/pause` | `coder/resume` | `coder/snapshot` | `coder/context_at` | `coder/branch` | `coder/fold` | `coder/commit` |
| Audit `:repl/request.op` field | `"pause"` | `"resume"` | `"snapshot"` | `"context_at"` | `"branch"` | `"fold"` | `"commit"` |
| Audit `:repl/response.op` field | mirrors `:repl/request.op` (single string) |
| AGENT-side commit datom | — | — | — | — | `:coder/branch` (only this op produces an AGENT-side substrate write via `db.branch()`) | — | — |

This makes `:coder/branch` the agent-side commit (real `db.branch()` call) while `:repl/request op="branch"` is the operator's directive. They co-exist on the audit chain in that order: `:repl/request` first, then `:coder/branch`, then `:repl/response`. Avoids the ADR-13 collision codex flagged.

**`fork_at` source pinned (R0-fold I3):** `fork_at = session.coder.substrate.clock()` read at `_steering.branch()` ENTRY (agent-side, not at WS transport entry). Substrate's `:clock/now` is pinnable for replay; wall-clock is not. Recorded value lands in `:repl/request.fork_at`. Replay re-injects the recorded `fork_at` byte-identically. Open W3 rescope: route `fork_at` through `:sys/now` (joins the existing 2.3b/2.3c.2 `:sys/now` bundle queued for 2.4a).

Per § 3.7's replay table row for `:repl/request`: "read recorded request, **re-inject** at the recorded decision boundary, read recorded response." Replay re-injects the operator directive; the agent's downstream effects audit-replay against the re-injected payload.

**Acceptance signal G5 (R0-fold I4: filter to `:repl/*` + `:coder/branch` only):**
```python
_REPL_OP_PREFIXES = (":repl/request", ":repl/response", ":coder/branch")

def filter_steering_audit(entries):
    return [e for e in entries if e.op.startswith(_REPL_OP_PREFIXES)]

def test_byte_identity_replay_branch_fold_commit(coder, session):
    # First run: live execution
    branch_id = session.branch({"directive": "explore option B"})
    scores = session.fold(probe=lambda db: len(list(db.log())))
    session.commit(branch_id)
    steering_run1 = filter_steering_audit(list(coder.substrate.audit.entries()))

    # Second run: replay mode, re-inject :repl/request payloads
    coder_replay = make_replay_coder(steering_run1, mode="audit-replay")
    coder_replay.run()
    steering_run2 = filter_steering_audit(list(coder_replay.substrate.audit.entries()))

    # Compare ONLY the steering-op stream — other audit entries (LLM call timings,
    # FS writes) may carry legitimately nondeterministic fields that aren't
    # the byte-identity contract for THIS gate.
    assert canonical_dumps(steering_run1) == canonical_dumps(steering_run2)
```

### LD5 — `coder.fold(probe)` is session-side, NOT `db.fold` (R0-fold I1: explicit FD; R0-fold B4: parent inclusion test pinned)

`coder.fold(probe: Callable[[DB], Any])` iterates `branches` dict, calls `probe(branch_db)` per entry, returns `dict[branch_id, Any]` of scores. The `branches` dict has a reserved key `"parent"` pointing at the parent DB (R0-fold B4: parent IS included).

`probe` signature: `Callable[[DB], Any]` — pure function over a DB snapshot. The score type is operator-defined (could be float for ranking, dict for richer scoring, etc.). Audit captures the probe's result hash via `result_summary` on `:repl/response`.

**FD-LD5 (Forced Spec Deviation, R0-fold I1):** Phase 2 design § 3.6 line 241 says `coder.fold(probe)` "uses #145 surface `s.txn.fold`." LD5 deviates: `coder.fold` is session-side dict iteration, NOT `s.txn.fold`. **Rationale:** `s.txn.fold(seed, items, fn)` is foldl-with-fact-emission — every iteration's `fn` returns `(new_acc, list[facts])` and those facts get committed via `transact_batch`. `coder.fold(probe)` is read-only scoring; emitting facts on every probe call would pollute branches with metadata about being scored, contradicting the "branches are immutable counterfactuals during fold" invariant. Substrate-true alternative would be a NULL-fact-list path on `s.txn.fold` allowing the `fn` to commit zero facts, but that's a substrate primitive change deferred to v0.10.x. **Acceptance signal:** the design choice is recorded in CHANGELOG-coder.md as FD-LD5; if a future refactor introduces `s.txn.fold`-backed `coder.fold`, the FD entry is removed and `coder.fold` calls `s.txn.fold(seed=None, items=branches.items(), fn=lambda acc,b,db: (probe(b[1]), []))`.

**Acceptance signal G4 (R0-fold B4: parent inclusion pinned to len==4 + key="parent"):**
```python
def test_fold_probe_over_branches_deterministic(session):
    b1 = session.branch({"d": "1"})
    b2 = session.branch({"d": "2"})
    b3 = session.branch({"d": "3"})

    scores = session.fold(probe=lambda db: len(list(db.log())))
    assert isinstance(scores, dict)
    # Parent inclusion (R0-fold B4): scores has 4 entries — parent + 3 children
    assert len(scores) == 4
    assert "parent" in scores
    assert b1 in scores and b2 in scores and b3 in scores
    # Determinism: re-run gives same result
    scores2 = session.fold(probe=lambda db: len(list(db.log())))
    assert scores == scores2
```
