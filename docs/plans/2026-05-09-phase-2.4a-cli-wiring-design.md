## Goal

Wire the production CLI (`python -m persistence.coder`) so it installs the same handler stack the test fixtures install, with `:sys/now`-driven `fork_at` defaults in steering and env-keyed Ed25519 audit signing. Closes 4 queued rescopes from earlier phases (2.3c.1, 2.3c.2, 2.3d, vertical-data-corpus LD-9).

## Architecture (one paragraph)

`__main__.main()` extends from "open + install LLM handler" to a 5-step bootstrap: (1) `Substrate.open(uri, audit_signer=‚Ä¶)` with signer derived from env, (2) install `make_skill_handler(skill_lib)` against the freshly-opened DB, (3) install the recursion-aware LLM dispatcher wrapping the chosen provider handler, (4) construct `Coder` (whose steering session picks up `:sys/now` for default `fork_at` via a one-line patch in `_steering.py`), (5) run. Signer plumbing is two kwarg-passthrough layers: `Substrate.open(audit_signer=‚Ä¶)` ‚Üí `canonical_audit_stack(entries, *, signer=‚Ä¶)` ‚Üí AuditEntry signing path. No new CLI flags. Mirrors 2.2b/2.3c.1 production-wiring precedent.

## Locked Decisions

### LD-1 ‚Äî Skill handler at CLI bootstrap

**Decision:** `__main__.main()` constructs `SkillLibrary(substrate._db)` against the freshly-opened substrate's DB and installs `make_skill_handler(skill_lib)` at `position="bottom"` immediately after `Substrate.open(...)`.

**Rationale:** mirrors 2.3c.1 fixture pattern (`tests/coder/test_loop_replay.py:137`). Identical setup as test fixtures means CLI-mode ‚â° test-mode for the `:skill/define` and `:skill/lookup` ops ‚Äî no behavioural drift between tested and shipped paths.

**Acceptance signal:** G1 (below). Falsifiable test that flips PASS ‚Üí FAIL if skill handler isn't installed.

**File changes:** `__main__.py` adds 2-3 lines (import `make_skill_handler` + `SkillLibrary`, construct, install).

### LD-2 ‚Äî Recursion-aware dispatcher wraps the provider handler

**Decision (FD-LD2 confirmed in T0 receipts):** the recursion-budget enforcement (`enter_call` / `exit_call` / `LLMRecursionBudgetExceeded`) **already lives inside `canonical_audit_stack`** at `effect/_audit_stack.py:297-370`. `Substrate.open(uri, audit=True)` (the default ‚Äî opt-out only via explicit `audit=False`) installs `canonical_audit_stack` automatically. **LD-2 collapses to an assertion test:** verify a CLI run installed via `__main__.main()` actually enforces the recursion budget. **No `__main__.py` change needed** for the recursion path.

**Rationale:** the recursion budget context is already active for production `:llm/call` invocations because the canonical_audit_stack is installed by default. The 2.3c.2 ship landed the wiring inside the audit stack, not as a separate dispatcher. The assertion test exists to lock the invariant: if a future change moves recursion enforcement out of the audit stack, the test fires.

**Acceptance signal:** G2 (below). Falsifiable: if `canonical_audit_stack` is bypassed (e.g., `audit=False` accidentally becomes the default), 3-deep compose succeeds ‚Üí test fails.

**File changes:** zero in `__main__.py`. New test only.

### LD-3 ‚Äî `:sys/now` substrate-time op for default `fork_at`

**T0 receipts finding (load-bearing for the decomposition below):**

- `_steering.py:294-299` already W3-rescoped `:sys/now` to 2.4a explicitly (verbatim comment: "R0-fold I3: source from agent-side wall-clock until :sys/now lands (W3 rescope to 2.4a, joining existing :sys/now bundle)").
- `Transaction.now()` at `txn/transaction.py:95-99` returns the **frozen `t_start` of a dosync body** ‚Äî it's deterministic-per-retry but ONLY valid inside `s.txn.dosync(...)`. `_steering.branch()` is not called inside a dosync, so `s.txn.now()` doesn't directly apply.
- This means LD-3 is NOT a one-line default-arg flip ‚Äî it requires landing `:sys/now` as a new substrate op (~2-3h scope).

**Two paths considered. Path B locked, ARIS to confirm.**

**Path A ‚Äî land `:sys/now` in 2.4a:** add `:sys/now` op to the substrate (effect handler returning monotonic substrate-time, replay-stable across re-runs); wire `_steering.branch()` to `s.effect.perform(":sys/now", {})` for default `fork_at`. ~2-3h additional scope. Brings 2.4a budget to ~8-9h. Tighter coupling between this phase and the harden track.

**Path B ‚Äî W3-rescope `:sys/now` to 2.4b, defer LD-3 from this phase (LOCKED, ARIS gate):** keep current wall-clock default in `_steering.py:299`; update the existing W3 comment to point at 2.4b instead of 2.4a; bundle `:sys/now` op-landing into 2.4b's spine alongside observability + `--confidence-threshold` deferrals. Reason: `:sys/now` is a substrate-level addition (new op, new handler, new test gate), not CLI wiring. 2.4a's theme is "production CLI wiring" ‚Äî `:sys/now` belongs in a phase that lands a substrate primitive cleanly. Drops 2.4a budget to ~5-6h, drops LD count from 4 to 3.

**Decision:** **Path B**. ARIS R0 will weigh in on whether the W3-rescope is honest under hard-mode-precedent rules (must queue with falsifiable acceptance signal ‚Äî see W3 Rescopes section below for the marker).

**Acceptance signal:** **deferred to 2.4b**. The 2.4a W3-rescope marker = `_steering.py:299` comment updated to `# noqa: wall-clock ‚Äî :sys/now W3 rescope to 2.4b` plus a 2.4b-tagged xfail-strict test in `tests/coder/test_steering_replay.py` that asserts substrate-time fork_at, currently expected to fail until 2.4b lands.

**File changes:** comment-tweak only in `_steering.py:299` (~1 line). xfail-strict test added in `tests/coder/test_steering_replay.py` (~30 LOC, marker `xfail(strict=True, reason=":sys/now W3 rescope 2.4b")`).

### LD-4 ‚Äî Env-keyed Ed25519 signer plumbed through

**Decision:** `__main__.main()` reads `PERSISTENCE_AUDIT_KEY` env var (URI-shaped: `file:/path/to/key.pem` for file-backed PEM); resolves to a `Signer` instance or `None`; passes to `Substrate.open(audit_signer=signer)`. `Substrate.open` adds `audit_signer: Signer | None = None` kwarg, threads to `canonical_audit_stack(entries, *, signer=audit_signer)`. `canonical_audit_stack` adds `signer: Signer | None = None` kwarg; AuditEntry construction in the wrapped/raw paths picks up the signer for `.sign(canonical_bytes)` to populate `AuditEntry.signature`.

**Rationale:** without this wiring, the Ed25519 audit-tamper-evidence work from 2.1c.5 / Phase B-demo can't reach end-users. Env-key surface mirrors `ANTHROPIC_API_KEY` precedent (`_provider.py:43`) ‚Äî zero CLI flags, zero docs to write, opt-in via env presence.

**URI scheme:** `PERSISTENCE_AUDIT_KEY=file:/path/to/key.pem` (file-backed PEM). This disambiguates from a future "raw PEM in env" form (e.g., `PERSISTENCE_AUDIT_KEY=pem:-----BEGIN PRIVATE KEY-----‚Ä¶`) without committing to it. Unknown URI schemes raise `SystemExit` at CLI bootstrap (loud-fail, like `--provider=anthropic` without `ANTHROPIC_API_KEY`).

**Acceptance signal:** G4 (below). Falsifiable: env-set vs env-unset ‚Üí signature presence differs.

**File changes:** ~30-50 LOC across `__main__.py` (env parse + Signer construction), `sdk/__init__.py` (kwarg passthrough), `_audit_stack.py` (kwarg + signing call site).

### G3 ‚Äî `:sys/now` xfail-strict marker (W3-rescoped to 2.4b)

**Setup:** `tests/coder/test_steering_sys_now.py::test_branch_default_fork_at_is_substrate_now` calls `session.branch(directive=...)` without `_fork_at_override`; asserts the resulting `:coder/branch` datom's `fork_at` equals `s.effect.perform(":sys/now", {})` (substrate-time op, lands in 2.4b). Marker: `@pytest.mark.xfail(strict=True, reason=":sys/now W3 rescope 2.4b")`.

**Falsifiability:** the strict marker is the falsifiability mechanism. Today (2.4a): xfail because `:sys/now` op doesn't exist + `_steering.branch()` defaults to `dt.datetime.now(dt.timezone.utc)`. When 2.4b lands `:sys/now` AND wires `_steering.branch()` to it, the xfail flips to PASS. If 2.4b lands `:sys/now` op but forgets to wire `_steering.branch()`, the xfail still fails ‚Üí 2.4b regression caught. If 2.4b lands wiring but op derives wall-clock, the xfail still fails ‚Üí wrong implementation caught.

**Test LOC:** ~40.

### G4 ‚Äî Env-keyed signer round-trip

**Setup:** `tests/effect/test_audit_signer_env.py::test_env_keyed_signer_signs_audit_entries`:
1. Generate ephemeral Ed25519 keypair, write private key to `/tmp/test-2.4a.pem`.
2. Set `PERSISTENCE_AUDIT_KEY=file:///tmp/test-2.4a.pem`.
3. Invoke `__main__.main` with a small task.
4. Read AuditEntries from substrate; assert each has non-None `signature` field.
5. Verify chain: `verify_audit_chain(entries, public_key)` returns True.
6. Tamper one entry's payload byte ‚Üí `verify_audit_chain` returns False.

**Falsifiability:** with env var unset, `signature` fields are None and `verify_audit_chain` short-circuits to True (signed-mode-disabled posture) ‚Äî but the env-set positive path is the load-bearing one.

**Test LOC:** ~80.
