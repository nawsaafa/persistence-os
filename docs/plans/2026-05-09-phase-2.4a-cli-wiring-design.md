## Goal

Wire the production CLI (`python -m persistence.coder`) so it installs the same handler stack the test fixtures install, with `:sys/now`-driven `fork_at` defaults in steering and env-keyed Ed25519 audit signing. Closes 4 queued rescopes from earlier phases (2.3c.1, 2.3c.2, 2.3d, vertical-data-corpus LD-9).

## Architecture (one paragraph)

`__main__.main()` extends from "open + install LLM handler" to a 5-step bootstrap: (1) `Substrate.open(uri, audit_signer=…)` with signer derived from env, (2) install `make_skill_handler(skill_lib)` against the freshly-opened DB, (3) install the recursion-aware LLM dispatcher wrapping the chosen provider handler, (4) construct `Coder` (whose steering session picks up `:sys/now` for default `fork_at` via a one-line patch in `_steering.py`), (5) run. Signer plumbing is two kwarg-passthrough layers: `Substrate.open(audit_signer=…)` → `canonical_audit_stack(entries, *, signer=…)` → AuditEntry signing path. No new CLI flags. Mirrors 2.2b/2.3c.1 production-wiring precedent.

## Locked Decisions

### LD-1 — Skill handler at CLI bootstrap

**Decision:** `__main__.main()` constructs `SkillLibrary(substrate._db)` against the freshly-opened substrate's DB and installs `make_skill_handler(skill_lib, name="skill")` at `position="bottom"` immediately after `Substrate.open(...)` and BEFORE the existing provider-handler install.

**Handler-ordering note (R0-fold I3):** `install_handler` at `sdk/_facade.py:143-166` is idempotent on `handler.name` (line 162: `rt.handlers = [h for h in rt.handlers if h.name != handler.name]` — replaces by name). `position="bottom"` always inserts at `handlers[0]`. Skill (`name="skill"`) and provider (`name="anthropic"` / `"claude-code"`) have distinct names, so they co-exist. Order matters only for op-routing-by-name at perform time — distinct ops (`:skill/*` vs `:llm/call`) don't conflict regardless of stack position. T1 install order: skill FIRST, provider SECOND, so provider ends up at `handlers[0]` (innermost for `:llm/call`) and skill at `handlers[1]` (innermost for `:skill/*`).

**Rationale:** mirrors 2.3c.1 fixture pattern (`tests/coder/test_loop_replay.py:137`). Identical setup as test fixtures means CLI-mode ≡ test-mode for the `:skill/define` and `:skill/lookup` ops — no behavioural drift between tested and shipped paths.

**Acceptance signal:** G1 (below). Falsifiable test that flips PASS → FAIL if skill handler isn't installed.

**File changes:** `__main__.py` adds 2-3 lines (import `make_skill_handler` + `SkillLibrary`, construct, install).

### LD-2 — Recursion-aware dispatcher wraps the provider handler

**Decision (FD-LD2 confirmed in T0 receipts, refined in R0-fold N1):** the recursion-budget enforcement (`enter_call` / `exit_call` / `LLMRecursionBudgetExceeded`) **already lives inside `canonical_audit_stack`** at `effect/_audit_stack.py:250-303`. The dispatcher handler is **a no-op pass-through when no `DispatcherContext` is bound** (`_audit_stack.py:303` short-circuit, gated on `current_dispatcher_context() is None` per `_recursion.py:230`). `Coder.run()` binds `with dispatcher_context(DispatcherContext()):` PER ITERATION at `_session.py:87`. So end-to-end CLI → Coder.run → DispatcherContext bound → recursion enforced through canonical_audit_stack. **LD-2 collapses to an assertion test:** verify the binding chain survives. **No `__main__.py` change needed.**

**Rationale (R0-fold N1 explicit precondition):** the recursion budget context is gated on TWO conditions: (a) `Substrate.open(audit=True)` (default; installs canonical_audit_stack) AND (b) `Coder.run()` binds `dispatcher_context(DispatcherContext())` at iteration entry. CLI inherits both for free; the assertion test locks both invariants.

**Acceptance signal:** G2 (below, rewritten in R0-fold B1 + B2). Falsifiability captures TWO regression classes: (1) audit stack bypass (`audit=False` at Substrate.open), (2) DispatcherContext binding removed from Coder.run.

**File changes:** zero in `__main__.py`. New test only.

### G2 — Recursion budget enforced through full CLI → Coder.run path (R0-fold B1+B2 rewrite)

**Setup:** `tests/coder/test_cli_wiring.py::test_cli_run_enforces_recursion_budget` invokes `__main__.main` with a task that triggers **4-deep nested `:llm/call`** (with default `MAX_LLM_CALL_DEPTH=3` per `_recursion.py:73`, allowed depths `{1,2,3}` and the 4th call raises). Asserts the run exits non-zero with `LLMRecursionBudgetExceeded(field="depth")` propagating through the CLI's stderr (or accessible via the test harness's exception capture).

**Real regression captured (R0-fold B1):** the recursion enforcement chain is:
1. `Substrate.open(audit=True)` (default) installs `canonical_audit_stack` (`_audit_stack.py:379`).
2. `canonical_audit_stack` includes a `_make_dispatcher_handler` (`_audit_stack.py:250`) that **only enforces when `current_dispatcher_context() is not None`** (`_recursion.py:230`; pass-through otherwise per `_audit_stack.py:303`).
3. `Coder.run()` binds `with dispatcher_context(DispatcherContext()):` per iteration at `_session.py:87`.

**Falsifiability — TWO regression classes both flip the test PASS → FAIL:**
- **Class A:** if `Coder.run` drops the `dispatcher_context(DispatcherContext())` wrap (`_session.py:87`), then `current_dispatcher_context() is None` at perform time → dispatcher handler short-circuits to pass-through → 4-deep compose succeeds → no exception → test fails.
- **Class B:** if `__main__.main` accidentally calls `Substrate.open(uri, audit=False)`, the canonical_audit_stack is never installed → no dispatcher middleware → 4-deep compose succeeds → test fails.

**Exception name (R0-fold B2):** `LLMRecursionBudgetExceeded` (per `_recursion.py:164`), NOT `RecursionBudgetExceeded`. `field="depth"` per `_recursion.py:293`.

**Test LOC:** ~60.
