## Locked Decisions

### LD-1 — Narrow banner-mask for echo-mode `ValueError` (codex consensus Q1)

**Decision:** In `__main__.py:200-224`, wrap the `Coder(...).run()` block with a `try/except ValueError` that triggers ONLY when `provider_name == "echo"`. On catch, print a single-line stderr message (e.g., `"persistence-coder: echo handler can't drive a real loop — set ANTHROPIC_API_KEY or sign in to Claude Code"`) and return non-zero exit.

**Codex consensus path:** Controller initial proposal was Approach B (`--provider=echo` with clean termination — change echo to return a valid termination tool-call). Codex Round-1 verdict: **REJECT-FOR-OPTION-C**. Three load-bearing arguments:
1. **Semantic repurpose of `raw-echo`**: `raw.py:20-24` docstring intentionally says "echoes last message content" — used by other tests for deterministic echo assertions. Approach B silently breaks that invariant.
2. **Misleading exit-0**: Approach B normalizes "no provider available" into exit-0 success. A user without credentials would get a "successful" run that did no work. Wrong UX signal.
3. **Error taxonomy preservation**: Approach A's broad `ValueError` catch would hide real programmer errors. Option C narrows the mask to the known-special-case (echo mode) using `provider_name == "echo"` which the CLI already tracks at `__main__.py:164-170`.

**Implementation sketch (R0-fold B1 + B2 corrected):**

Two changes required:

**(a)** Change `_build_substrate_and_handlers` signature to return `(substrate, provider_name)` tuple. Currently it returns only `substrate` at `__main__.py:199` and `provider_name` is local to the helper at `__main__.py:164`. R0-fold B2 plumbing:

```python
def _build_substrate_and_handlers(args) -> tuple["Substrate", str]:
    # ... existing body ...
    handler, provider_name = detect_or_explicit(args.provider)
    # ... existing body ...
    return substrate, provider_name
```

**(b)** Update `main()` with valid Python control-flow (R0-fold B1) — outer `try / except CoderStubNotImplemented / except ValueError / no-finally` plus inner `try / finally` for substrate.close():

```python
def main(argv: list[str] | None = None) -> int:
    parser = build_parser()
    args = parser.parse_args(argv)

    # R0-fold I1: pre-bind so the except ValueError below can safely
    # reference provider_name even if _build_substrate_and_handlers
    # itself raises ValueError before the unpacking succeeds. In that
    # case provider_name stays None → mask check is False → re-raise.
    provider_name: str | None = None

    try:
        substrate, provider_name = _build_substrate_and_handlers(args)
        try:
            Coder(
                task=args.task,
                substrate=substrate,
                model=args.model,
                max_iters=args.max_iters,
            ).run()
        finally:
            substrate.close()
    except CoderStubNotImplemented as exc:
        print(f"persistence-coder skeleton: {exc}", file=sys.stderr)
        return 1
    except ValueError as exc:
        # Phase 2.4b.1 LD-1 (codex consensus Option C, R0-fold I1):
        # narrow banner-mask. Only swallow ValueError when echo handler
        # is in use (no real LLM provider). Other ValueErrors are real
        # programmer errors and must propagate. Substrate-construction
        # ValueErrors → provider_name stays None → falls through to raise.
        if provider_name == "echo":
            print(
                "persistence-coder: echo handler can't drive a real "
                "agent loop. Set ANTHROPIC_API_KEY or sign in to Claude "
                "Code, then re-run.",
                file=sys.stderr,
            )
            return 1
        raise
    return 0
```

Note on control-flow shape (R0-fold B1): Python does NOT permit `try / finally / except`. The valid forms are:
- `try / except / [else] / [finally]` (one block, all clauses)
- Nested: outer `try / except`, inner `try / finally` (or vice versa)

This impl uses nested: outer `try / except CoderStubNotImplemented / except ValueError` wraps inner `try / finally`. The `substrate.close()` finally only runs if substrate construction succeeded (Python doesn't enter the inner try otherwise — `substrate` may be unbound, but we don't reference it in the except handlers).

**Acceptance signal:** G1 — `python -m persistence.coder --task hello` (no provider, no API key) exits non-zero with single-line stderr message (no Python traceback). Falsifier: revert the `except ValueError` block → traceback re-appears.

## Forced Deviations (declared at T0; updated post-R0-fold)

- **FD-LD1-provider-name-plumbing** (R0-fold B2): `provider_name` is NOT in `main()` scope by default — it's local to `_build_substrate_and_handlers()` at `__main__.py:164`. Helper signature changes from `-> Substrate` to `-> tuple[Substrate, str]` returning `(substrate, provider_name)`. `main()` unpacks: `substrate, provider_name = _build_substrate_and_handlers(args)`. Pre-bind `provider_name: str | None = None` outside the try-block so the except clause can reference it safely if construction itself raises ValueError.
- **FD-LD1-control-flow** (R0-fold B1): the implementation uses nested try-blocks (outer try / except CoderStubNotImplemented / except ValueError wraps inner try / finally substrate.close()). Python does NOT permit `try / finally / except` flat — see the LD-1 sketch.
- **FD-LD1-mask-narrowness** (locked, codex consensus DECIDER): the mask catches `ValueError` ONLY when `provider_name == "echo"`. Other exception classes (`KeyError`, `AttributeError`, etc.) and non-echo ValueErrors propagate.
- **FD-LD1-mask-message-narrowness** (R0-fold I2 acknowledged): the mask currently catches ANY ValueError in echo mode, not specifically the "malformed act payload" one. Codex I2 notes this could hide unrelated echo-mode bugs. Tightening to message-match (e.g., `"malformed act payload" in str(exc)`) is deferred to v0.9.x per codex N1 — LD-1 is consensus-locked at the broader narrowness boundary (provider_name guard) which already constitutes meaningful scoping vs Approach A's global mask.
- **FD-LD2.a-version-bump:** `ALLOWLIST_VERSION` will change. The `shell.py:73-77` comment documents allowlist changes as the explicit observable signal. Tests already import `ALLOWLIST_VERSION` dynamically, so version-bump propagates automatically. R0-fold I3 sanity note: no caller hard-codes the old hash — dynamic import is the project convention.
- **FD-LD2.b-PYTHONPATH-form:** use `{**os.environ, "PYTHONPATH": ...}` (mirrors `test_cli_smoke.py:21`) rather than overriding os.environ entirely.
