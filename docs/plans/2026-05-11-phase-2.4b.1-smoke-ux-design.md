## Architecture (one paragraph)

The CLI's `__main__.py:221` only banner-masks `CoderStubNotImplemented` — a sentinel that disappeared when Phase 2.3b replaced stubs with real impls. Post-2.3b, the no-provider path runs the echo handler whose `{"text": "echo:..."}` response falls through `_session._decide`'s tiers to a tier-3 fallback that creates `LLMDecision(kind="act", payload={"raw_text": ...})` — no `op` field. `_act()` then raises raw `ValueError`. LD-1 adds a NARROW banner-mask specifically for `ValueError` when `provider_name == "echo"`, preserving `raw-echo`'s deterministic-echo semantics and the deliberate error taxonomy (`CoderStubNotImplemented` = banner-masked sentinel; other exceptions = real failures). LD-2 aligns three test-harness inconsistencies — two CLI smoke tests with stale banner-mask assertions, one HTTP subprocess test missing `PYTHONPATH`, and the shell allowlist not tracking Python 3.14 stems.

## Locked Decisions

### LD-1 — Narrow banner-mask for echo-mode `ValueError` (codex consensus Q1)

**Decision:** In `__main__.py:200-224`, wrap the `Coder(...).run()` block with a `try/except ValueError` that triggers ONLY when `provider_name == "echo"`. On catch, print a single-line stderr message (e.g., `"persistence-coder: echo handler can't drive a real loop — set ANTHROPIC_API_KEY or sign in to Claude Code"`) and return non-zero exit.

**Codex consensus path:** Controller initial proposal was Approach B (`--provider=echo` with clean termination — change echo to return a valid termination tool-call). Codex Round-1 verdict: **REJECT-FOR-OPTION-C**. Three load-bearing arguments:
1. **Semantic repurpose of `raw-echo`**: `raw.py:20-24` docstring intentionally says "echoes last message content" — used by other tests for deterministic echo assertions. Approach B silently breaks that invariant.
2. **Misleading exit-0**: Approach B normalizes "no provider available" into exit-0 success. A user without credentials would get a "successful" run that did no work. Wrong UX signal.
3. **Error taxonomy preservation**: Approach A's broad `ValueError` catch would hide real programmer errors. Option C narrows the mask to the known-special-case (echo mode) using `provider_name == "echo"` which the CLI already tracks at `__main__.py:164-170`.

**Implementation sketch:**

```python
# __main__.py — after Coder(...).run() block
try:
    Coder(
        substrate=substrate,
        model=args.model,
        max_iters=args.max_iters,
    ).run()
finally:
    substrate.close()
except CoderStubNotImplemented as exc:
    print(f"persistence-coder skeleton: {exc}", file=sys.stderr)
    return 1
except ValueError as exc:
    # Phase 2.4b.1 LD-1: echo-mode ValueError is the only ValueError
    # path that should be banner-masked. Other ValueErrors are real
    # programmer errors and must propagate. provider_name is in scope
    # from the detect_or_explicit() call earlier in main().
    if provider_name == "echo":
        print(
            f"persistence-coder: echo handler can't drive a real "
            f"agent loop. Set ANTHROPIC_API_KEY or sign in to Claude "
            f"Code, then re-run.",
            file=sys.stderr,
        )
        return 1
    raise
return 0
```

`provider_name` is already a local in `main()` per the existing echo-warning at `__main__.py:165-170` — no plumbing change needed.

**Acceptance signal:** G1 — `python -m persistence.coder --task hello` (no provider, no API key) exits non-zero with single-line stderr message (no Python traceback). Falsifier: revert the `except ValueError` block → traceback re-appears.


Load-bearing argument: the hermetic_subprocess + allowlist failures aren't fuzzy environmental drift — they're direct consequences of today's test-harness and allowlist contracts that will re-bite every verification run until fixed. The cheapest time to remove the recurring tax is alongside LD-1's already-in-flight smoke edits. Both fixes are bounded:

## Test Gates

### G1 — Echo-mode banner-mask + exit-non-zero (LD-1)

**Test:** `test_cli_smoke.py::test_cli_runs_skeleton_and_emits_banner_on_first_stub` (REWRITTEN per LD-2.c).

Asserts that `python -m persistence.coder --task hello` (no provider):
- Exits with returncode != 0.
- Stderr does NOT contain Python traceback markers (`Traceback (most recent call last):`, `File "/...py", line N`).
- Stderr contains the new banner message (`"echo handler can't drive a real agent loop"` or similar).
- Stderr still contains the existing pre-loop warnings (`"warning: no --db-path"`, `"warning: no LLM provider available"`, `"echo handler"`).

**Falsifiability matrix:**
- If LD-1's `except ValueError` block is forgotten → raw traceback in stderr → "no traceback markers" assertion fails.
- If the mask catches non-echo ValueErrors (broad-catch regression) → tests that intentionally raise ValueError in non-echo mode will start incorrectly exiting 1 instead of propagating; a separate test below catches this.
- If the exit code is 0 (Approach B regression) → returncode != 0 assertion fails.

## Forced Deviations (declared at T0)

- **FD-LD1-provider-name-scope:** `provider_name` is already a local in `main()` per the existing echo-warning at `__main__.py:165-170`. The `except ValueError` block references it via lexical scope — no plumbing change needed.
- **FD-LD1-mask-narrowness:** the mask is ONLY for `ValueError`. Other exception classes (`KeyError`, `AttributeError`, etc.) propagate. This is intentional per codex consensus DECIDER (preserve the deliberate error taxonomy).
- **FD-LD2.a-version-bump:** `ALLOWLIST_VERSION` will change. The `shell.py:73-77` comment documents this as the explicit signal for allowlist changes. No backwards-compat shim — the change is the contract.
- **FD-LD2.b-PYTHONPATH-form:** use `{**os.environ, "PYTHONPATH": ...}` (mirrors `test_cli_smoke.py:21`) rather than overriding os.environ entirely.
