## Locked Decisions

### LD-1 — Wheel-build + fresh-venv install + dual-CLI smoke (codex consensus Q1 NEW-OPTION-Z)

**Decision:** "Lockfile freeze verified" means **no lock drift + buildable artifacts + fresh-env install/run smokes for the shipping surfaces** — not "sync everything and run one module."

**Gate steps (G1 test body, R0-fold B3b: HTTP smoke moved to W3 sister):**

1. **`uv lock --check`** — assert current `uv.lock` matches `pyproject.toml`; no drift.
2. **`uv build`** — produce `dist/persistence-0.9.0a1-py3-none-any.whl` + `dist/persistence-0.9.0a1.tar.gz`.
3. **Fresh-venv install from the built wheel** (NOT editable, NO `[http]` extras — base wheel only):
   ```bash
   python -m venv /tmp/2.4c-smoke-venv
   /tmp/2.4c-smoke-venv/bin/pip install dist/persistence-0.9.0a1-py3-none-any.whl
   ```
4. **Coder CLI smoke:**
   - `python -m persistence.coder --task hello` (echo-mode banner-mask + exit 1, no traceback — reuses 2.4b.1 G1 invariants)
   - HTTP smoke is **NOT** in base G1 (R0-fold B3b: `persistence.http` requires `[http]` extras `fastapi/uvicorn/pydantic` not in the base wheel). W3 sister test `tests/sdk/test_lockfile_distribution_smoke_http.py` installs `dist/*.whl[http]` and exercises the HTTP entry-point.

**Rationale (codex-tipped):** Two arguments tipped the verdict:
1. **`uv sync` installs editable-from-repo** — sidesteps the entire "what does an installed wheel look like" failure surface (missing package-data, broken extras constraints, module-only-fails-when-imported-from-installed-dist). My original Option A would give false confidence.
2. **`--all-extras` is cost-imbalanced** — pulls dspy/aiohttp/hypothesis/psycopg-binary[bin] just to verify the coder smoke runs. Disproportionate.

**R0-fold note on lockfile-vs-wheel separation (B3a defense):** the gate validates TWO separate contracts in two distinct steps:
- **Step 1 (`uv lock --check`)** validates the DEV-environment reproducibility contract — `uv.lock` matches `pyproject.toml`, so `uv sync` in any contributor's workspace produces an identical resolved env.
- **Step 3 (`pip install dist/*.whl`)** validates the CONSUMER-side install contract — the BUILT WHEEL installs cleanly into a fresh interpreter using wheel metadata for transitive resolution. This is what Phase 7 / decision-ops / agent-stack-first-class will do.

These are different consumers using different mechanisms; conflating them (e.g., `uv pip install --requirement uv.lock`) doesn't test the wheel artifact downstream consumers actually receive. Codex's B3a suggestion to use a "lockfile-respecting `uv pip install` flow" was conflating these contracts.

**Why competitors lose:**
- **Original Option A (`uv sync --all-extras --reinstall`)**: editable install + ~250MB extras download, doesn't validate distribution.
- **Option C (`uv lock --check` only)**: passes while install-time/import-time/packaging-time failures exist. The 2.4b.1 LD-2.a allowlist regression that re-bit fresh-install would have been missed.

**Falsifiable acceptance signal:** `tests/sdk/test_lockfile_distribution_smoke.py::test_built_wheel_installs_and_runs_dual_cli` — `@pytest.mark.xfail(strict=True, reason="Phase 2.4c LD-1: built wheel install + dual-CLI smoke")` until impl ships. Test creates `tmp_path / "smoke-venv"`, subprocess-runs `uv build` in repo, installs `dist/*.whl` into the venv, runs both CLI modules, asserts banner-mask + no-traceback. **Regression vector**: if `pyproject.toml` `[tool.setuptools.packages.find]` is broken or `[tool.setuptools.package-data]` omits a critical resource, install fails or runtime fails — test catches it.

**FD-LD1-wheel-build-isolation (declared T0):** the test must run `uv build` in a copy of the repo (or with `--no-build-isolation` flag controlled) so it doesn't pollute the source tree. Spec: use `tmp_path` + `shutil.copytree(repo_root, tmp_path / "src")` with `ignore=shutil.ignore_patterns(".git", "*.pyc", "__pycache__", ".venv", "dist", "build")`, then `uv build` inside the copy.

from __future__ import annotations
import tomllib
from pathlib import Path

import pytest

from persistence.sdk._facade import Substrate

MANIFEST_PATH = Path(__file__).resolve().parents[1] / "preflight_manifest.toml"


def _load_manifest() -> dict:
    with MANIFEST_PATH.open("rb") as fh:
        return tomllib.load(fh)


def test_manifest_is_closed_allowlist_and_resolves():
    """G2 — every dotted entry in [allowed.*] resolves to a callable
    attribute on Substrate.open(...); no entry starts with `escape.`;
    escape_callsites is empty.
    """
    manifest = _load_manifest()
    assert manifest["meta"]["version"] == "0.9.0a1"
    assert manifest.get("escape_callsites") == [], (
        "v0.9.0a1 contract: escape_callsites MUST be empty"
    )
    assert "allowed" in manifest

    s = Substrate.open("memory")
    try:
        for namespace_name, methods in manifest["allowed"].items():
            assert namespace_name != "escape", (
                f"escape namespace in [allowed] violates v0.9.0a1 contract"
            )
            namespace = getattr(s, namespace_name, None)
            assert namespace is not None, (
                f"manifest entry [allowed.{namespace_name}] but Substrate "
                f"has no `{namespace_name}` attribute"
            )
            for method_name in methods:
                attr = getattr(namespace, method_name, None)
                assert attr is not None, (
                    f"manifest entry [allowed.{namespace_name}.{method_name}] "
                    f"but Substrate.{namespace_name} has no `{method_name}` attribute"
                )
                assert callable(attr), (
                    f"manifest entry [allowed.{namespace_name}.{method_name}] "
                    f"is not callable on Substrate.{namespace_name}"
                )
    finally:
        s.close()


def test_escape_callsites_remain_empty():

### G1 — Wheel-build + fresh-venv install + dual-CLI smoke (LD-1)

**File:** `tests/sdk/test_lockfile_distribution_smoke.py` (NEW)

**Test sketch:**

```python
import shutil
import subprocess
import sys
from pathlib import Path
import pytest

REPO_ROOT = Path(__file__).resolve().parents[2]


@pytest.mark.xfail(strict=True, reason="Phase 2.4c LD-1: built wheel install + dual-CLI smoke not yet implemented")
@pytest.mark.slow  # 30-60s
def test_built_wheel_installs_and_runs_dual_cli(tmp_path: Path) -> None:
    """G1 — uv lock --check + uv build + fresh-venv install of built
    wheel + run python -m persistence.coder + python -m persistence.http.
    Catches packaging-time / install-time / import-time failures that
    `uv lock --check` alone misses.
    """
    # (1) Copy repo to tmp_path (avoid polluting source tree with dist/)
    repo_copy = tmp_path / "repo"
    shutil.copytree(
        REPO_ROOT,
        repo_copy,
        ignore=shutil.ignore_patterns(
            ".git", "*.pyc", "__pycache__", ".venv", "dist", "build",
            "*.egg-info", ".pytest_cache", ".mypy_cache",  # R0-fold N1
        ),
    )

    # (2) uv lock --check
    r = subprocess.run(
        ["uv", "lock", "--check"],
        cwd=str(repo_copy),
        capture_output=True,
        text=True,
        timeout=60,
    )
    assert r.returncode == 0, f"uv lock --check failed:\n{r.stderr}"

    # (3) uv build
    r = subprocess.run(
        ["uv", "build", "--no-progress"],
        cwd=str(repo_copy),
        capture_output=True,
        text=True,
        timeout=180,
    )
    assert r.returncode == 0, f"uv build failed:\n{r.stderr}"
    wheels = list((repo_copy / "dist").glob("persistence-0.9.0a1-*.whl"))
    assert len(wheels) == 1, f"expected one wheel, got {wheels}"

    # (4) Fresh venv install of the BUILT WHEEL (not editable)
    venv_dir = tmp_path / "smoke-venv"
    subprocess.run([sys.executable, "-m", "venv", str(venv_dir)], check=True, timeout=60)
    venv_py = venv_dir / "bin" / "python"
    venv_pip = venv_dir / "bin" / "pip"
    r = subprocess.run(
        [str(venv_pip), "install", "--no-input", str(wheels[0])],
        capture_output=True,
        text=True,
        timeout=120,
    )
    assert r.returncode == 0, f"pip install of built wheel failed:\n{r.stderr}"

    # (5) python -m persistence.coder --task hello — echo-mode banner-mask
    # R0-fold B3b: HTTP smoke is NOT in base G1; sister test G1[http]
    # at tests/sdk/test_lockfile_distribution_smoke_http.py (W3 rescope)
    # installs dist/*.whl[http] and exercises the FastAPI startup surface.
    r = subprocess.run(
        [str(venv_py), "-m", "persistence.coder", "--task", "hello"],
        capture_output=True,
        text=True,
        timeout=30,
    )
    assert r.returncode == 1, (
        f"coder smoke: expected exit 1 (echo-floor), got {r.returncode}:\n"
        f"stdout={r.stdout}\nstderr={r.stderr}"
    )
    assert "Traceback (most recent call last):" not in r.stderr, (
        f"coder smoke: raw traceback leaked:\n{r.stderr}"
    )
    assert (
        "persistence-coder: echo handler can't drive a real agent loop"
        in r.stderr
    )
```

**Falsifier:** if `pyproject.toml` `[tool.setuptools.packages.find]` omits a submodule OR `[tool.setuptools.package-data]` is missing critical resources OR `pyproject.toml` `dependencies` is incomplete → install or runtime fails → test fails loudly.
