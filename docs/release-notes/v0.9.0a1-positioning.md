# What v0.9.0a1 Unlocks

> *Substrate properties no model upgrade will give you.*

Better models will not fix unattributable actions, unbounded blast
radius, or unreplayable history. v0.9.0a1 ships a substrate where each
of those is a structural guarantee, not a runtime hope. The contract is
held by code that runs locally, signs locally, and replays locally —
not by a vendor's API. Five capabilities ship in this release; each is
tied to a named phase and a falsifiable test. If a cited test fails on
your clone, the guarantee is gone. That is the contract.

## 1. Agent-vs-human identity

Phase 2.4a wires an Ed25519 signer into the canonical audit stack via
the standard PEM URI form `file:///abs/path/to/key.pem`. Every datom
an agent emits is signed with a key the agent holds; every datom a
human commits is signed with a key the human holds. Tamper with a
single byte and verification breaks. There is no shared "system" actor
and no opaque service trust — actions carry a verifiable origin by
construction. Unsigned mode is supported for local development and
fails closed in production: an unknown URI scheme or a missing key
file aborts process start rather than degrading to anonymous writes.

**Proof:** `tests/effect/test_audit_signer_env.py::test_tamper_breaks_signature_verification`
— flips a single byte in a frozen audit entry and asserts
`verify_chain` rejects it. The round-trip, unknown-URI, and
missing-PEM `SystemExit` paths are pinned in sibling tests in the
same module.

## 2. Governed action

Phase 2.3d introduces the `Capability(op=..., qualifier=...)` lattice
that gates every coder operation at perform-time. An agent without
`Capability(op="coder", qualifier="branch")` cannot branch the
substrate, full stop — the call denies before the side effect runs,
and the denial itself is an audit-chain entry. This is not
authorization-as-middleware sitting above an unconstrained substrate;
it is the substrate's own perform-time gate. The lattice composes:
capabilities can be issued, revoked, and scoped per branch without
crossing trust boundaries.

**Proof:** `tests/coder/test_steering_capability.py::test_op_requires_capability`
— parametrized over the coder op surface; asserts that a missing
capability raises before the op executes and that the denial lands
on the audit chain.

## 3. Audit replay

Phase B established the canonical audit stack; Phase 2.4a wired it
through every coder entrypoint. The chain is byte-identical on
replay: hand a v0.9.0a1 trace back to a fresh substrate, replay it
from the first datom, and the resulting canonical form matches the
original byte for byte. Two falsifying siblings make the property
real — swap a handler implementation and replay breaks; skew the
recorded clock by one nanosecond and replay breaks. This is the
property that lets a regulator, a buyer's auditor, or your own
incident-response team reconstruct exactly what happened, not "what
we think happened from logs."

**Proof:** `tests/coder/test_loop_replay.py::test_coder_loop_audit_replay_byte_identity`
— replays a recorded coder loop and asserts byte-equal canonical
audit form before and after. The two paired falsifiers
(`..._handler_swap_mismatch`, `..._clock_skew_mismatch`) sit beside
it in the same module.

## 4. Kill switch

Phase 2.3d ships REPL pause/resume as first-class substrate
operations, emitting `:repl/request` and `:repl/response` audit
entries on the same chain as ordinary work. An operator can pause a
running coder mid-action, inspect substrate state at the exact halt
point, branch a probe, and resume — and the pause itself is
replayable. The pause/resume pair is not a runtime flag tacked on to
a long-running loop; it is a recorded substrate event that
participates in the canonical audit chain like any other op.

**Proof:** `tests/coder/test_steering_replay.py::test_pause_emits_repl_request_then_response_in_order`
— asserts the pause/resume pair lands on the audit chain in
canonical order with an intact Merkle linkage.

## 5. Substrate time

Phase 2.4b promotes `:sys/now` from an ambient wall-clock call to a
substrate-level operation, resolved through the canonical audit
stack and delegated to the replay clock during replay. No coder
path reads the wall clock directly; the read is itself an audit
entry. This is the property that makes section 3 (audit replay)
actually byte-identical — without substrate time, replay drifts the
moment any timestamp appears in a recorded action.

**Proof:** `tests/effect/test_sys_now_handler.py::test_sys_now_delegates_to_replay_clock`
— asserts that under a replay context, `:sys/now` returns the
recorded clock value, not the wall clock.
