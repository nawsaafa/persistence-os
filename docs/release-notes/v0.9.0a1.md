`docs/release-notes/v0.9.0a1-positioning.md` install sentence.

## Highlights

Five substrate capabilities, each with a falsifiable proof. See
[`docs/release-notes/v0.9.0a1-positioning.md`](./v0.9.0a1-positioning.md) for the
full mapping of capability → test.

1. **Agent-vs-human identity** — Ed25519 audit signing via
   `PERSISTENCE_AUDIT_KEY=file:///path/to/key.pem`; `verify_chain` validates both
   prev-hash AND signature; tamper-evident at single-byte resolution.
2. **Governed action** — `Capability(op, qualifier)` ADT lattice; ops gated at the
   `:capability/denied` boundary; replayable denial entries.
3. **Audit replay** — `canonical_audit_stack` produces a chain that replays
   byte-identically under a pinned clock + same signing key (the "canonical
   replay byte-identity" contract).
4. **Kill switch** — `_CoderSteeringSession` 7-op live debugger (pause / resume /
   snapshot / branch / commit / fold / status); pause is fail-fast at `_check_pause`
   between every coder step.
5. **Substrate time** — `:sys/now` view op (NOT wrapped); single canonical-time
   source; replay-clock symmetry across coder + REPL + observe paths.

## Breaking changes from v0.8.5a1

- **`pyproject.toml`**: version `0.8.5a1 → 0.9.0a1`; `pydantic>=2.9,<3.0` promoted
  from `[http]` extras to base dependencies (Phase 2.4c F2; G1 wheel-smoke surfaced
  the v0.8.5a1 distribution defect).
- **`uv.lock`**: now tracked (Phase 2.4c F1). Lockfile is the reproducibility contract
  for v0.9.0a1.
- **`tests/preflight_manifest.toml`**: NEW. Hand-authored allowed-entrypoints contract
  for the persistence-coder agent and Phase 7 skill consumers. Adding / removing /
  renaming a curated method requires explicit manifest edit + reviewer sign-off.
- **`_build_substrate_and_handlers(args)`** (`coder/__main__.py:124-199`): signature
  changed from `-> Substrate` to `-> tuple[Substrate, str]`. 5 test callsites updated.
  Phase 2.4b.1 LD-1.
- **Shell handler allowlist**: `python3.X` version stems added (2.4b.1 LD-2.a).
