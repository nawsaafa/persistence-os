## Highlights

Five substrate capabilities, each with a falsifiable proof. See
[`docs/release-notes/v0.9.0a1-positioning.md`](./v0.9.0a1-positioning.md) for the
full mapping of capability → test.

1. **Agent-vs-human identity** — Ed25519 audit signing via
   `PERSISTENCE_AUDIT_KEY=file:///path/to/key.pem` (PEM is the operator-surface
   format; runtime extracts raw 32-byte private key bytes at the env-decode
   boundary). `verify_chain(entries)` validates the prev-hash chain by default;
   pass `public_keys={signer_id: raw_pub_bytes}` to additionally validate Ed25519
   signatures on every signed entry. Tamper-evident at single-byte resolution.
2. **Governed action** — `Capability(op, qualifier)` ADT lattice. Two enforcement
   sites with different semantics: (a) `_CoderSteeringSession._check_cap` denies
   pre-emission — denied steering ops never reach the `:repl/*` audit chain
   (`coder/_steering.py:95-112`); (b) the Phase 7 orchestrate runner emits a
   `:capability/denied` signaling op for skipped chain steps and records a
   signed audit entry via its own audit middleware
   (`orchestrate/_emit.py:245-255` + `_install_orchestrate_handlers`).
3. **Audit replay** — `canonical_audit_stack` produces a chain that replays
   byte-identically under a pinned clock + same signing key (the "canonical
   replay byte-identity" contract).
4. **Kill switch** — `_CoderSteeringSession` 7-op live debugger (pause / resume /
   snapshot / branch / commit / fold / status); the `Coder.run` loop calls
   `_check_pause` at the head of each iteration so the next observe→decide→act
   cycle is gated.
5. **Substrate time** — `:sys/now` is a VIEW handler at `position="bottom"` in
   `canonical_audit_stack`, NOT in `CANONICAL_AUDIT_WRAPPED_OPS`. It synthesizes
   substrate-frozen UTC by nested-performing `:clock/now`. Reads emit NO
   per-call audit entry; instead, audit-wrapped ops that read `:sys/now` carry
   the time into their own entry's `recorded_at`. Replay-clock symmetry holds
   across the coder + REPL + observe paths.

## New surfaces in v0.9.0a1

Substrate behaviour added since v0.8.5a1 falls into 4 buckets:

**A. Canonical audit-wrapped ops added** (`CANONICAL_AUDIT_WRAPPED_OPS` in
`src/persistence/effect/_audit_stack.py:104-168`):

| Op | Phase | Substantive return? | Notes |
|---|---|---|---|
| `:llm/call` | 2.1b | yes (provider-supplied) | Wrapping op; provider handler installed `position="bottom"` by `coder/__main__.py`. |
| `:claim/emit` | 2.1c.6 | no | Audit-only; request datom IS the audit signal. Facts committed separately via `s.fact.transact`. |
| `:blob/put` | 2.1c.6 | no | Same audit-only shape as `:claim/emit`. |
| `:fs/read`, `:fs/write`, `:fs/glob`, `:fs/grep` | 2.2a | yes | Substantive-return; bottom-of-stack handler via `make_fs_handler`. |
| `:shell/exec` | 2.2a | yes | Allowlisted stems via `make_shell_handler`; python3.X stems added in 2.4b.1. |
| `:code/run` | 2.2b | yes | NEW op (NOT shared with legacy `:code/exec`); substantive-return wrapper. |
| `:git/diff`, `:git/status`, `:git/log`, `:git/commit` | 2.2b | yes | Thin wrappers over `:shell/exec` (codex Q2 verdict). |
| `:skill/define`, `:skill/lookup` | 2.3c.1 | yes | Skill-library content-addressed ops. |
| `:repl/request`, `:repl/response`, `:coder/branch` | 2.3d | no (raw) | REPL steering audit-only ops; in `CANONICAL_AUDIT_RAW_OPS`. |

**B. View handlers (NOT audit-wrapped)** installed by `canonical_audit_stack`:

| Op | Phase | Position | Notes |

| `:sys/now` | 2.4b | between `clock` and `audit` | View over `:clock/now` returning UTC `datetime.datetime`. No per-call audit entry by design (mirrors `:clock/now` exclusion to avoid audit-tier recursion). |

**C. Fact-layer datoms added** (not substrate ops; written via
`s.fact.transact` into the coder's session DB):

| Attribute | Phase | Source |
|---|---|---|
| `:llm/messages` | 2.1b | Conversation history before `_decide`. |
| `:llm/decision` | 2.1b | Provider decision shape after `_decide`. |

**D. Phase 7 orchestrate-emitted signaling op** (emitted by the orchestrate skill
runner's `_install_orchestrate_handlers`, NOT a canonical substrate op):

| Op | Phase | Notes |
|---|---|---|
| `:capability/denied` | 7 | Signaling op for skipped chain steps; recorded by the orchestrate runner's own audit middleware which reuses the canonical stack's signer. |

## Breaking changes from v0.8.5a1

- **`pyproject.toml`**: version `0.8.5a1 → 0.9.0a1`; `pydantic>=2.9,<3.0` promoted
  from `[http]` extras to base dependencies (Phase 2.4c F2; G1 wheel-smoke surfaced
  the v0.8.5a1 distribution defect).
- **`uv.lock`**: now tracked (Phase 2.4c F1). Lockfile is the reproducibility contract
  for v0.9.0a1.
- **`tests/preflight_manifest.toml`**: NEW. Hand-authored allowed-entrypoints contract
  for the persistence-coder agent and Phase 7 skill consumers. Adding / removing /
  renaming a curated method requires explicit manifest edit + reviewer sign-off.
- **`_build_substrate_and_handlers(args)`** (`coder/__main__.py:124-199`): signature
  changed from `-> Substrate` to `-> tuple[Substrate, str]`. 5 test callsites updated.
  Phase 2.4b.1 LD-1.
- **Shell handler allowlist**: `python3.X` version stems added (2.4b.1 LD-2.a).
