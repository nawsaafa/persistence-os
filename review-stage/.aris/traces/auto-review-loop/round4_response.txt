We present \textbf{Persistence}, a neurosymbolic substrate where symbolic state --- bitemporal datoms, content-addressed plan ASTs, policy-as-data, and boundary specs --- carries formal properties on top of which neural agency composes. \section{Introduction}\label{sec:intro}

\textbf{Research question.} When neural agency is scaffolded by a symbolic substrate, what formal properties does the combined system inherit that neither layer provides alone? We argue that six substrate invariants --- immutable bitemporal datoms, algebraic-effect handlers with audit semantics, homoiconic plan ASTs, transactional shared state, machine-readable boundary specs, and a REPL --- yield derived capabilities (accountability, replay, counterfactual fidelity, composable safety, compositional skill learning, multi-agent coordination) that are consequences of the substrate, not independently-engineered features.

LLM agents in production expose recurring gaps: regulators asking \emph{``what did the agent believe at 14:03, and why?''} get prose-log narratives; post-incident counterfactuals are simulated by prompt perturbation that changes more than the intended variable; self-rewriting skill libraries accumulate drift~\citep{hannecke2026ssgm}; single-layer guardrails cannot compose policy/dry-run/cache/rate-limit/audit. The field offers partial solutions~\citep{kurtic2025zep,cheng2025pangolin,khattab2023dspy,wang2023voyager,anon2026camo} but none are composed: five substrates, five consistency models, five failure modes.

\textbf{Contribution.} We present \textbf{Persistence}, a cognitive-runtime substrate built around a single claim: accountability, replay, counterfactuals, composable safety, and compositional skill learning can be derived from \emph{one} substrate rather than re-engineered per system. v0.1.0a1 + v0.4.0a1 + v0.5.1 ships six of seven modules (\textbf{931 tests green, 7 xfailed}); Plan execution and REPL remain Phase 2.C / 2.E. The substrate is asserted once; each derived capability is a property we check against the log, not a feature engineered per agent. Neural agency operates over an explicitly symbolic substrate: bitemporal datom queries with a Datalog-shaped surface, EDN AST grammars for plans, policy-as-data, and Malli-style specs with an \C{explain\_for\_llm} self-healing surface. We do not claim a Datalog rule engine or Z3-discharged \C{verify} leaves~\citep{fan2024proofofthought} in Phase 1; both are adjacent future work.

\section{Related Work}\label{sec:related}

\textbf{Agent memory}\label{sec:memory} has evolved from vector-store RAG to graph-structured memory (Graphiti, Neo4j-based agent graphs). Zep~\citep{kurtic2025zep} and Memento~\citep{n1n2026memento} introduce bitemporal edges with \C{valid\_at}/\C{invalid\_at} timestamps. Mem0 and A-Mem pursue mutable memory updated by LLM-driven consolidation, which~\citet{hannecke2026ssgm} shows produces unbounded drift without a formal write-gate. Persistence sits on the immutable end with a critical extension: it treats \emph{every} datom (memory, audit, plan, skill, transaction) as bitemporal --- memory is one consumer; the substrate serves all.

\textbf{Algebraic effects in LLM programming}\label{sec:effects-rw} were brought to LLM scripting by~\citet{wang2025composable} and Pangolin~\citep{cheng2025pangolin}: separating what an effect is from how it is handled enables stacked interceptors for retry, caching, multi-shot sampling, observability. Neither integrates effect handlers with a temporal memory substrate, policy-as-data, or a formal audit chain. Persistence does: with the audit handler wrapping the full catalog (the regulated-deployment default), every effect emits a datom, every handler is policy-reviewable, every run is replayable.

\textbf{Structured agent programming.}\label{sec:structured} DSPy~\citep{khattab2023dspy} treats agents as declarative programs compiled by optimizers (BootstrapFewshot, MIPROv2). LLM+PDDL integrations~\citep{liu2023llmp,valmeekam2023planbench} route LLMs through classical planners; \citet{kambhampati2024canllm} argues LLMs-as-planners are unsound without a symbolic verifier. The neurosymbolic reasoning literature --- Logic Tensor Networks~\citep{badreddine2022ltn}, DeepProbLog~\citep{manhaeve2018deepproblog}, NeuPSL~\citep{pryor2023neupsl} --- integrates differentiable neural components with symbolic inference at finer granularity than we target; our contribution is orthogonal: a substrate for agent-level symbolic scaffolding with LLM leaves, not a neural-symbolic reasoning hybrid at the inference layer.

\textbf{Skill libraries and continual learning.}\label{sec:skills} Voyager~\citep{wang2023voyager} demonstrated LLM agents can accumulate executable skill libraries; Memento-Skills~\citep{memento2026skills} extended this with Read-Write Reflective Learning. Voyager promotes skills on first successful use, producing a long tail of single-use skills under noise. Persistence's Plan module enforces a four-gate criterion over versioned, content-addressed skills with parent pointers preserving lineage.

\textbf{Counterfactual replay.}\label{sec:counterfactual-rw} CAMO~\citep{anon2026camo} introduced simulator-internal counterfactuals with aligned randomness; AgentHER~\citep{anon2026agenther} adapted Hindsight Experience Replay~\citep{andrychowicz2017her}; AgenTracer~\citep{anon2025agentracer} uses counterfactual fault injection; Abduct-Act-Predict~\citep{anon2025aap} operationalizes Pearl's causal ladder. Persistence is stronger on this axis: because all state is immutable bitemporal datoms, aligned randomness is a substrate property, not a simulator hack. For the NO-OP intervention case, the canonical hash of the replayed trajectory is byte-identical to the factual trajectory's --- a structural guarantee, not a statistical one.

\section{The Persistence Thesis}\label{sec:thesis}

We state six substrate invariants. Invariants 1, 2, 5 are fully realized in Phase 1; invariant 3 is realized at the shape + parse + validate layer by Phase 2.B (execution Phase 2.C); 4 and 6 are frozen at the spec-registry boundary with runtime in Phase 2.

\begin{enumerate}
\item \textbf{Every fact is immutable, temporal, content-addressed.} Memory, audit, plan, skill, transaction are all datoms; provenance, counterfactual branching, audit trails are derived. \emph{[Phase 1: shipped.]}
\item \textbf{Every action is an effect.} LLM calls, tool calls, memory writes, and side effects pass through a composable handler stack; policy, safety, caching, dry-run, retry are handler instances. \emph{[Phase 1: shipped, 15-op catalog.]}
\item \textbf{Every plan is an EDN AST.} Agents read, edit, and evolve their own plans; skills are named AST subtrees promoted under statistical evidence; plan optimization is structural search. \emph{[Phase 2.B v0.2.0a1: parse/walk/spec-validation over the \C{[tag \{attrs\} *children]} AST with 128-bit content-addressing; execution Phase 2.C.]}
\item \textbf{Every shared state change is a transaction.} STM coordinates multi-agent belief updates without locks or message passing. \emph{[Phase 2.D v0.5.1: shipped --- snapshot-isolated dosync (CM + decorator + factory forms) over datom-backed refs; commit gate \C{any\_datoms\_since(t\_start)} guarded by an atomic \C{store.\_lock} window; intent-log read-set emitted into commit-datom provenance; \C{:effect/txn-commit} promoted to a first-class \C{AuditEntry} field threading transactional commits into the audit chain.]}
\item \textbf{Every LLM boundary has a spec.} Parse-don't-validate at trust boundaries; generative testing from specs; self-healing via spec-error feedback. \emph{[Phase 1: shipped, 186 tests.]}
\item \textbf{Everything is REPL-live.} Inspection, editing, rewind, and speculative branching apply to running production agents. \emph{[Rewind subsumed by \C{as\_of}; REPL Phase 2.]}
\end{enumerate}

\textbf{Thesis.} These six invariants, implemented together as a single runtime, produce a system where accountability, replay, counterfactuals, composable safety, compositional skill promotion, coordinated multi-agency, and live production steering are \emph{properties of the substrate} --- not features engineered per agent. Six are shipped: accountability via the Merkle-hashed audit chain; replay via byte-identical NO-OP; counterfactual branching via \C{branch()}; composable safety via the Effect handler stack with a machine-checkable well-formedness property; homoiconic plan-as-data via the Phase 2.B Plan module's content-addressed parse/walk/spec-validation; and transactional shared-state coordination via the Phase 2.D Txn module's snapshot-isolated \C{dosync} with intent-log provenance threaded into the audit chain. Phase 2.C / 2.E ship the remaining capabilities (Plan-execution, REPL) on top of the same substrate with no schema changes --- that non-requirement is the testable form of the thesis.

\section{Formalization}\label{sec:formal}

The formalization below tracks the Phase 1 reference implementation. Where a claim depends on Phase 2 work, we mark it explicitly.

\subsection{Datoms and bitemporal queries}\label{sec:datoms}

A \textbf{datom} is an eight-tuple:

\begin{equation}\label{eq:datom}
d = \langle e,\ a,\ v,\ \tau,\ \tau_{sys},\ \nu_{from},\ \nu_{to},\ \omega \rangle
\end{equation}

where $e$ is an entity, $a$ a namespaced attribute, $v$ an EDN value, $\tau$ a monotonic transaction id, $\tau_{sys}$ the ingestion time, $\nu_{from},\nu_{to}$ the valid-time interval, and $\omega \in \{\text{assert}, \text{retract}\}$. A provenance record $\pi$ ships as the \C{Provenance} \C{TypedDict} (\C{total=False}) over seven known keys --- \C{source}, \C{tx\_time}, \C{handler\_id}, \C{canonical\_call}, \C{parent\_provenance\_hash}, \C{superseded\_by\_tx}, plus a free-form \C{extra} map. The schema is deliberately model-pluggable: \C{canonical\_call} is a content hash of (model, prompt, tools, params); no \C{:provider} field --- the neural model is a plug-in, not embedded in the substrate's shape. Phase 1 seals provenance with SHA-256; per-transaction ed25519 signing is Phase 2 (\sectionref{sec:privacy}). Core queries: $\text{asOf}(D, t) = \{d \mid \tau_{sys}(d) \leq t\}$; $\text{validAsOf}(D, t) = \{d \mid \omega = \text{assert},\, \nu_{from} \leq t < \nu_{to}\}$; $\text{history}(D, e) = \{d \mid \text{entity}(d) = e\}$ sorted by $\tau$; $\text{branch}(D, t, \Delta) = \text{asOf}(D, t) \cup \Delta$. Projection-layer counterfactuals are supported via \C{ProjectionAdapter.fork(branch\_id)} (v0.4.0a1) --- a Protocol method returning a fresh empty adapter the caller populates by \C{rebuild()} against a branched DB.

\begin{proposition}[Branch is a logical operation over the shipped store]\label{prop:branch}
\C{branch(D, t, $\Delta$)} returns a new \C{DB} value backed by a fresh in-memory store seeded with \C{asOf(D, t)} and extended with $\Delta$; writes to the branched value cannot leak back into the parent. \emph{Complexity:} on the Phase 1 \C{InMemoryStore}, $O(|D|)$ seed plus $O(|\Delta|)$ additions. \emph{Phase 2 upgrade:} a persistent HAMT backing store reduces seed to $O(|\Delta| \log |D|)$; the \C{Store} Protocol makes this drop-in.
\end{proposition}

\emph{Remark.} The Phase 1 complexity is honest rather than ideal --- we ship a correct list-backed implementation rather than pull in \C{pyrsistent}. The \emph{isolation} property holds unconditionally and is the load-bearing one for counterfactual reasoning.

\subsection{Effects, handlers, and well-formedness}\label{sec:effects}

An effect signature is a typed pair $\langle \text{op}: \kappa, \text{args}: \sigma_{in} \rightharpoonup \sigma_{out} \rangle$. The minimum catalog $K$ comprises 15 ops: \C{llm/call}, \C{tool/call}, \C{mem/read}, \C{mem/write}, \C{decide}, \C{ask-user}, \C{emit-artifact}, \C{sleep}, \C{random}, \C{env/read}, \C{net/fetch}, \C{secret/use}, \C{cost/charge}, \C{clock/now}, \C{audit/emit}. A \textbf{handler} $h$ is a partial function $(\kappa, \sigma_{in}, k: \sigma_{out} \to \alpha, \mathrm{ctx}) \to \alpha$ with continuation semantics. A stack $H = [h_1, \dots, h_n]$ ($h_1$ outermost) dispatches $\kappa$ to the outermost handler whose domain contains it; that handler invokes the continuation, delegating to the remaining stack.

\begin{proposition}[Well-formedness; machine-checkable on the shipped runtime]\label{prop:wellformed}
A stack $H$ over catalog $K$ is well-formed iff for every $\kappa \in K$, at least one handler above the raw base handles $\kappa$. The shipped \C{Runtime.is\_well\_formed(catalog)} (\C{src/persistence/effect/runtime.py}) decides this property in $O(|H| \cdot |K|)$ time --- \emph{linear in stack size for fixed catalog} ($|K|=15$ in the shipped runtime); \C{Runtime.uncovered\_ops(catalog)} returns the witness set. At runtime, \C{Runtime.perform(op, \dots)} raises \C{Unhandled} when no handler covers $\kappa$ --- the property is not merely asserted but \emph{enforced on every call}.
\end{proposition}

This is the paper's strongest formal contribution on the Phase-1 artifact: a decidable, linear-time, machine-checked completeness property with runtime enforcement via \C{Unhandled}. Properties above (audit-chain integrity \sectionref{sec:audit}, replay determinism \sectionref{sec:replay}) build on well-formedness. Handlers compose \emph{masked} (Koka semantics): $\text{mask}_a(h)$ prevents $h$ from intercepting operations of attribute $a$ within its own body, eliminating re-entrant loops.

\subsection{The Merkle-hashed audit chain}\label{sec:audit}

When the \C{audit} handler wraps a set of operations $W \subseteq K$, each effect on $\kappa \in W$ emits a datom recording \C{(op, args, verdict, provenance, prev\_hash)} before the continuation fires. Each entry's \C{id} is computed as \C{sha256(canonical(fields \textbackslash{} \{id\}))} --- i.e.\ the canonical serialization of every entry field except the \C{id} itself, including \C{prev\_hash}; chaining is enforced by setting \C{prev\_hash := previous.id} (Proposition~\ref{prop:tamper}). Phase 1 ships \C{make\_audit\_handler(wraps, \dots)} (default \C{("llm/call",)}), \C{verify\_chain(entries) $\to$ bool} (re-derives \C{prev\_hash}; detects field-mutation tamper), and \C{audit\_entry\_to\_datom} flowing audit entries into the Fact log. As of v0.4.0a1, \C{audit\_entry\_to\_datom} also emits a \C{parent\_provenance\_hash} alias alongside \C{:prev-hash}, bridging audit-entry datoms into the typed \C{Provenance} schema so \C{DB.causal\_history(e)} walks audit chains and fact-log derivation chains under one query primitive. Authenticity (\emph{who} signed) is distinct from integrity and not claimed for Phase 1; per-transaction ed25519 is Phase 2 (\sectionref{sec:privacy}).

\begin{proposition}[Audit-chain immutability]\label{prop:tamper}
For any audit-chain $C = \langle e_0, \dots, e_n \rangle$ produced by \C{make\_audit\_handler}, \C{verify\_chain(C)} returns true iff for all $i$, $e_i.\text{id} = \text{sha256}(\text{canonical}(e_i.\text{fields} \setminus \{\text{id}\}))$ and $e_i.\text{prev\_hash} = e_{i-1}.\text{id}$ --- i.e. no entry has been mutated, deleted, reordered, or truncated from the middle. Exercised end-to-end by \C{tests/effect/test\_audit.py} (tampering / deletion / reordering tests). Tail-truncation is allowed by construction and must be detected by regulators comparing a separately-recorded expected length.
\end{proposition}

\textbf{Universality contract.} ``Every effect emits a datom'' holds when $W$ covers the full catalog $K$; \C{Runtime.is\_well\_formed} checks this, and regulated deployments (Case A in \sectionref{sec:eval-cases}) wrap all 15 ops.

\subsection{Plans (Phase 2.B shipped: parse/walk/spec-validation; execution Phase 2.C)}\label{sec:plans}

A plan is a labeled tree: internal nodes are \emph{control operators} ($\text{seq}$, $\text{par}$, $\text{choice}$, $\text{loop}$, $\text{race}$, $\text{let}$, $\text{branch}$); leaves are \emph{effect invocations} ($\text{tool-call}$, $\text{llm-call}$, $\text{code}$, $\text{checkpoint}$) or \emph{cognitive operators} ($\text{reflect}$, $\text{verify}$, $\text{call-skill}$). Every node $n$ carries content identity $\text{id}(n) = \text{sha256}(\text{canonical}(n))[:32]$ as its Merkle-DAG handle. The \C{:persistence.plan/node} spec is registered in Phase 1; the Phase 2.B \C{persistence.plan} module (v0.2.0a1) ships parse, unparse, depth-first walk, and spec validation against this shape. Encoding nodes as EDN vectors \C{[:tag \{attrs\} \& children]} rather than maps makes plan ASTs \emph{homoiconic}: the homoiconicity contract's allowed self-edits (read, splice, compose, rewrite, fork, promote) reduce to list-splicing over well-typed vectors.

\begin{proposition}[Plan content-addressing with descendant propagation]\label{prop:plan}
For any plan node $n = \langle \text{tag}, \text{attrs}, \text{children} \rangle$ constructed through the shipped \C{Node} constructor, $\text{id}(n) = \text{sha256}(\text{canonical}(n)).\text{hexdigest}()[{:}32]$ (i.e.\ the first 32 hex characters $=$ 128-bit prefix of the SHA-256 digest) over $n$'s tag, alphabetically-sorted attr map (non-finite floats rejected; non-string / colon-prefixed / empty / reserved \C{id} keys rejected at both construct and parse as defense-in-depth against EDN hash-poisoning), and its children's ids recursively. Hence for any descendant modification at depth $d$ producing $T'$, every ancestor id on the root-to-leaf path differs between $T$ and $T'$. Property-checked at \C{max\_examples=200} on round-trip id preservation and descendant-mutation propagation over all 16 node kinds. Spec conformance against \C{:persistence.plan/node} is enforced at parse when \C{strict=True}. The 7 \C{xfail} cases are tightening tests for Phase-2.C semantics (per-kind required-attr enforcement, e.g.\ \C{:loop} cardinality bounds, \C{:branch} exhaustiveness, \C{:case} disjointness); the shipped claim covers parse / walk / spec-validation / content-addressing only --- \emph{not} execution semantics --- so the xfails do not mask any property machine-checked above. Birthday-collision probability at 1\% reaches $\approx 2.6 \times 10^{18}$ plans at 128 bits, argued in the corresponding ship commit's CHANGELOG.
\end{proposition}

\emph{Dispatch seam (v0.4.0a1).} The \C{Dispatcher} class layers handler-per-tag dispatch over the AST walker (renamed \C{\_walk.py}; back-compat shim retained); walk order $\equiv$ dispatch order, hypothesis-tested at \C{max\_examples=50}. This is the seam Phase 2.C binds effect handlers into without AST shape changes; \C{PLAN\_CANONICAL\_VERSION} stays at 1.

\emph{Neurosymbolic reading.} Proposition~\ref{prop:plan} is the formal identity law for a homoiconic neurosymbolic AST: the skeleton (tag, attrs, children) is symbolic --- editable, optimizable, regulator-readable; leaves (\C{:llm-call}, \C{:code}, \C{:reflect}) are neural. Descendant propagation makes every structural edit produce a new ancestor-id path --- a verifiable identity surface Phase-2.C optimizers (MIPROv2, MCTS-over-Plan) walk over while leaves remain neural. The homoiconicity contract permits read, splice, compose, rewrite, fork, promote; it forbids edits to \C{:verify} nodes (would defeat Proof-of-Thought~\citep{fan2024proofofthought}), edits to ancestors of the executing frame, and unbounded \C{:loop}.

A \textbf{skill} is a quadruple $\langle \text{name}, \text{version}, \text{ast}, \text{stats} \rangle$ encoded by \C{:persistence.plan/skill}. Promotion (Phase 2 runtime) requires: $\text{uses}(T) \geq 3$, $T$ in $\geq 2$ distinct parents, rolling success rate $\geq 0.8$, and an LLM-written docstring of $T$ that retrieves $T$ as top-1 over the existing skill library --- the last gate enforcing semantic non-collision cheaply.

\subsection{Trajectories and replay}\label{sec:replay}

A \textbf{trajectory} is an ordered sequence of effect datoms sharing a run-id, plus a seed vector $\sigma = \langle \sigma_{llm}, \sigma_{tool}, \sigma_{env} \rangle$. An \textbf{intervention set} $I = [\langle \text{step}, \text{field}, \text{new-value} \rangle, \dots]$ is a list of per-step modifications, sorted by step. The replay operator copies $T_i$ verbatim for $i < I.\text{step}$, applies the intervention at $I.\text{step}$, and re-executes the suffix with effect calls routed through a \emph{replay handler} that returns cached responses keyed by $\langle \text{step-index}, \text{op}, \text{args-hash} \rangle$ (full cache model and trajectory hash given in Proposition~\ref{prop:noop}).

\begin{proposition}[Replay determinism --- stronger NO-OP form]\label{prop:noop}
If all non-determinism in the agent routes through effects in catalog $K$ \textbf{and} the per-step rng-consumption pattern is recorded, $\text{replay}(T, I)$ produces a deterministic counterfactual trajectory that diverges from $T$ exactly at $I.\text{step}$. \emph{Cache model:} the replay handler keys cached responses by $\langle \text{step-index}, \text{op}, \text{args-hash} \rangle$ rather than args-hash alone, so that two distinct steps issuing the same op with identical args still resolve to their respective recorded outcomes; the RNG-consumption pattern is what guarantees the cache key sequence reproduces deterministically. \emph{Trajectory hash:} $\text{trajectory\_hash}(T) = \text{sha256}(\text{canonical\_dumps}(\langle e_{\text{run-id}}, \text{op}, \text{args}, \text{value}, \tau, \text{prev-hash} \rangle_{i=0..n}))$ over the effect-datom sequence; it does not include wall-clock or non-recorded environment reads. Phase 1 strengthens the proposition for the NO-OP case: for an intervention that applies but changes no value, $\text{trajectory\_hash}(\text{replay}(T, I_{\text{noop}})) = \text{trajectory\_hash}(T)$ --- \emph{byte-identical} on the canonical serialization, not merely statistically close. Verified by \C{tests/replay/test\_determinism.py} (NO-OP byte-identity test) on toy agents with fixed per-step rng-consumption; LLM-leaf generalization is Phase 2 (\sectionref{sec:eval}).
\end{proposition}

This strengthens CAMO's aligned-randomness on the axis NeSy reviewers care about: a structural canonical-hash guarantee, not a statistical outcome-delta claim. Extending from toy agents (per-step rng-consumption fixed in \C{conftest.py}) to real LLM trajectories requires replacing \C{\_advance\_rngs\_to\_match} with a recorded per-step rng-state vector --- Phase 2 work, prerequisite for the \sectionref{sec:eval} camera-ready numbers.

\subsection{Transactions}\label{sec:transactions}

A \textbf{ref} is a pair $\langle \iota, \tau_{\text{current}} \rangle$ where $\iota$ is an identity and $\tau_{\text{current}}$ the most recent transaction id. A transaction body is a thunk that reads and writes refs; commit emits an atomic set of datoms, CAS-ing on all read refs' $\tau$. On conflict, the transaction is retried. Composition is automatic: a transaction that calls another transaction inherits the outer's write set. The Txn-module runtime ships in Phase 2.D (v0.5.1: snapshot-isolated \C{dosync} with read-set + intent-log provenance and the \C{:effect/txn-commit} audit-field hook; see \sectionref{sec:impl-txn}). Multi-process coordination via Postgres \texttt{SERIALIZABLE} is deferred to v0.6.x.

\subsection{Specs and the self-healing LLM contract}\label{sec:specs}

A \textbf{spec} $s$ is a predicate-generator pair. Conforming $x$ against $s$ yields a refined value or $\bot$ via a discriminated union (\C{Conformed} \textbar{} \C{ConformError}, not silent coercion). The boundary contract: business logic is typed against conformed values; raw inputs cannot reach it. Specs are first-class EDN --- composable, diffable, versionable.

\textbf{Self-healing contract (shipped).} On conform failure, \C{spec.explain\_for\_llm(err)} returns a structured message with field path, failure reason, and a Fix-clause-annotated hint. When an LLM emits a malformed tool call, the spec layer returns a prompt-ready correction the agent can retry against --- a \textbf{conform $\to$ explain $\to$ retry} contract. The symbolic layer emits hints the neural layer consumes; this is a concrete neurosymbolic contribution distinct from post-hoc error logging.

\textbf{Self-conforming producers (shipped).} Every load-bearing producer calls \C{spec.conform} on its return value and raises on mismatch --- a defect fails at the producer's site, not later inside a consumer that read a bad wire form.

\textbf{Forward-compatible spec-first commitment.} The two plan specs were registered in Phase 1 \emph{before} the Plan module existed --- a parse-don't-validate methodology that locks the data shape before code depends on it.

\section{Implementation}\label{sec:impl}

Persistence is implemented across seven modules. The reference implementation is in Python for integration parity with existing agent stacks. Phase 1 ships four modules (Fact, Effect, Spec, Replay); Phase 2.B ships Plan (minimal: parse / unparse / walk / spec-validation); Phase 2.D ships Txn (snapshot-isolated STM); Phase 2.C and 2.E remain (Plan execution; REPL).

\subsection{Fact (Phase 1 --- shipped)}\label{sec:impl-fact}

Fact is designed for an append-only Postgres datom log with five covering indexes (EAVT, AEVT, AVET, VAET) plus a bitemporal VT-E range index (Datomic Index Model;~\citealp{tonsky2023datomic}). Phase 1 ships a \C{Store} Protocol with two backends: \C{InMemoryStore} and \C{SQLiteStore} (against SQLite 3.37+; Postgres adapter and Zstd segment compression are Phase 2). Writes pass through a transactor computing auto-retractions for cardinality-one overwrites, preserving history via \C{invalidated-by} pointers. Projection is a \C{ProjectionAdapter} Protocol with a reference \C{DictProjection}; v0.4.0a1 adds \C{fork(branch\_id)} returning a fresh adapter the caller populates by \C{rebuild()} against a branched DB --- closing the projection-side counterpart to \C{branch()}. \C{DB.causal\_history(e, max\_depth=16)} ships as the substrate-level lineage query (single-level v0.4.0a1; multi-level deferred to v0.5).

\textbf{Concurrent-writer safety.} \C{Store.allocate\_and\_append(datoms)} runs \C{MAX(tx)+1} and row inserts inside a single atomic transaction (SQLite \C{BEGIN IMMEDIATE}; in-memory \C{threading.Lock}), closing the TOCTOU window a split \C{next\_tx()}/\C{append} would leave open. Exercised by 16 threads $\times$ 50 transacts producing 800 unique tx ids with zero collisions. This race-free allocator is the concurrency invariant Proposition~\ref{prop:noop} implicitly depends on. Phase 1 tests check correctness, not latency; per-operation p95 targets and reference numbers across 1k/10k/100k corpora are [TBD] for the camera-ready.

\subsection{Effect (Phase 1 --- shipped)}\label{sec:impl-effect}

The Effect module implements a handler-stack runtime. Handlers are declared in EDN with clauses keyed by operation. The canonical stack for regulated domains is:

The canonical regulated stack is \C{audit $\to$ policy $\to$ dry-run $\to$ cache $\to$ retry $\to$ rate-limit $\to$ raw}. Each handler is a pure function over operation, arguments, and continuation; \C{audit} emits a datom into the Fact module chaining via \C{prev-hash}. Policies are declarative EDN interpreted by a $\sim$200-line evaluator (\C{policy\_eval.py}) supporting principal attributes, op-matching, and conditional effects (\C{:op=}, \C{:op-in}, \C{:mode=}, \C{:and}, \C{:or}, \C{:not}, \C{:contains?}, \C{:matches?}). \C{Runtime.is\_well\_formed} and \C{Runtime.uncovered\_ops} are part of the public API.

\subsection{Spec (Phase 1 --- shipped)}\label{sec:impl-spec}

The Spec module provides Malli-equivalent functionality in Python: predicate-generator pairs, composable registry, conformance. Specs are EDN data; a generative harness produces examples for every registered spec, and failed conforms emit self-healing retry hints (\C{spec.explain\_for\_llm}). Ten canonical specs are registered across the namespaces \C{persistence.\{fact, effect, replay, plan\}} --- with the two plan specs ahead of the Plan module.

\subsection{Replay (Phase 1 --- shipped)}\label{sec:impl-replay}

The Replay module records trajectories as sequences of effect datoms and implements the replay operator from \sectionref{sec:replay}. A replay handler intercepts effects during branching, returning cached responses keyed by $\langle \text{step-index}, \text{op}, \text{args-hash} \rangle$ (Proposition~\ref{prop:noop} cache model: distinct steps issuing the same op with identical args resolve to their respective recorded outcomes). DPO pairs are to be extracted when paired trajectories differ in outcome beyond a configurable threshold: prefix must match exactly; suffix divergence feeds the \C{chosen}/\C{rejected} dataset automatically. \C{gen\_regression\_test(trajectory)} emits a pytest-source snapshot-test string that asserts against the loaded trajectory (this is honest snapshot-replay rather than agent-re-run; full regression re-execution is Phase 2). The byte-identical NO-OP determinism invariant (\sectionref{sec:replay} corollary) is the headline test result for this module.

\subsection{Plan (Phase 2.B --- shipped; Phase 2.C --- designed)}\label{sec:impl-plan}

Plan v0.2.0a1 ships a minimal parse/walk/spec surface over the uniform \C{[tag \{attrs\} *children]} EDN AST across 16 node kinds (\C{:seq}, \C{:par}, \C{:choice}, \C{:case}, \C{:loop}, \C{:race}, \C{:let}, \C{:branch}, \C{:ref}, \C{:tool-call}, \C{:llm-call}, \C{:code}, \C{:checkpoint}, \C{:reflect}, \C{:verify}, \C{:call-skill}). A frozen-slots \C{Node} dataclass with content-addressed \C{id} (Proposition~\ref{prop:plan}); an \C{edn\_format}-backed parser with optional alias lowering (\C{:phase}/\C{:workstream} $\to$ \C{:seq}); a symmetric canonical unparser; a depth-first \C{walk(node, visitor)} trace primitive; \C{PlanSpecError} conformance against \C{:persistence.plan/node}. Attr keys are plain strings internally, re-prefixed at the spec/emit boundary; user-supplied top-level \C{:id} is stripped at parse. NaN/Inf in attrs are rejected at id computation (\C{allow\_nan=False}). 172 plan-module tests ship alongside the full-repo suite of \textbf{931 passed + 7 xfailed} (v0.5.1).

v0.2.0a1$\to$v0.3.0a1 closed the reserved-attr-key round-trip falsifier and shipped a \C{Node.id} coercion registry for non-JSON-native types (\C{datetime}, \C{bytes}, \C{Decimal}, \C{UUID}, \C{frozenset}, \C{edn\_format.Symbol}), plus an alias escape hatch and a shared \C{SpecError} base. Phase 2.C (designed): a \C{fold} primitive for speculation/rollback/checkpointing; an effect-dispatched evaluator emitting OpenTelemetry spans keyed by \C{node.id}; the optimizer ladder (MIPROv2 on \C{:llm-call} via DSPy, UCT-MCTS on \C{:choice}/\C{:branch} with Pareto dominance, evolutionary topology search); the 4-gate skill-promotion pipeline.

\subsection{Txn (Phase 2.D --- shipped, v0.5.1)}\label{sec:impl-txn}

The Txn module implements software transactional memory over refs backed by the Fact store. Commits are snapshot-isolated against the read-set: a transaction body is a thunk that reads and writes refs; on commit, the runtime checks \C{any\_datoms\_since(t\_start)} against the recorded read-set inside an atomic \C{store.\_lock} window (the TOCTOU-free commit gate), retries on conflict via the \C{dosync} retry loop, and emits the write-set as a single tx-commit datom. Three surface forms ship: \C{with db.dosync()} (single-shot CM), \C{@db.dosync(max\_retries=N)} (retryable decorator), and \C{db.dosync()(callable)} (factory). Read-set + intent-log emit into the commit datom's provenance (v0.5.1 N1); per-ref attribute specs route through \C{Ref.spec\_attr} (N3); a Hypothesis property at \C{max\_examples=200} pins single-shot \C{assoc} byte-identity (N4); and \C{:effect/txn-commit} is promoted to a first-class \C{AuditEntry} field (N2), threading transactional commits through the audit chain. Commit semantics follow the Multiverse~\citep{bieniusa2026multiverse} hybrid of versioned and unversioned execution; multi-process coordination via Postgres SERIALIZABLE is deferred to v0.6.x. The integration with Fact means every transaction is itself a datom; the entire multi-agent coordination history is queryable.

\subsection{REPL (Phase 2 --- designed)}\label{sec:impl-repl}

The REPL module will expose a capability-gated WebSocket API for live inspection, editing, rewind, and branching of running agents. The UI extends a thread-routing Claude Code client. Rewind resumes an agent from an earlier $\tau$ using the already-shipped \C{as-of} view; branch runs a speculative path in a sandboxed handler stack with dry-run forced. Production invariant: a REPL session cannot write to the authoritative fact store without a signed operator approval.

\section{Evaluation}\label{sec:eval}

\textbf{Posture.} The submission reports the v0.1.0a1 + v0.4.0a1 + v0.5.1 shipped artifact with Propositions~\ref{prop:wellformed} and~\ref{prop:noop} machine-checked in the bundled test suite (\textbf{931 passed / 7 xfailed} from a clean clone in under one minute). Three numeric evaluations are staged for the 2026-07-20 camera-ready: \textbf{LongMemEval} (Fact vs.~MemGPT/Mem0/Zep/Memento, gating the mem0 projection adapter), \textbf{distributional CAMO} (1000 paired rollouts on a toy agent with per-step rng-state recording, then 100 trajectories on Claude Haiku if budget permits), and \textbf{regulator-replay} (a 50-trajectory synthetic project-finance corpus released alongside this paper under a permissive open-data license; metric: byte-for-byte reconstruction under \C{effect.canonical.canonical\_dumps}; hypothesis $H_1$: fidelity $\geq 99\%$). Unlike pure-trace replay~\citep{anon2026camo,anon2026agenther}, regulator-replay requires the symbolic plan AST and the neural effect trace to recover byte-identically together --- one fidelity metric on symbolic-neural coupling.

\section{Discussion}\label{sec:discussion}

\textbf{Limitations.} Phase 1 imposes costs we disclose frankly. Write latency rises by a constant per-transaction overhead from transactor validation and auto-retraction; exact figures on \C{InMemoryStore}/\C{SQLiteStore} are [TBD] for the camera-ready. Append-only storage is $\approx 4\times$ a mutable-memory baseline, partially offset by content-addressing and (Phase 2) Zstd segment compression. \C{branch} is $O(|D|)$-copy (Proposition~\ref{prop:branch}); HAMT backing is the Phase 2 upgrade. Replay byte-identity holds only for toy agents (Proposition~\ref{prop:noop}); LLM-trajectory generalization requires per-step rng-state recording (Phase 2). Audit universality is a stack-configuration contract, not a substrate invariant. Policy-as-EDN presumes EDN/Clojure literacy --- JSON is a subset, so interop is straightforward. Substrate lock-in is real, but the migration path is additive.

\textbf{Privacy architecture.} The authoritative datom log runs on operator-controlled infrastructure; projections are regenerable. Phase 1 provides chain \emph{integrity} (SHA-256 per-datom hashes, \C{verify\_chain}); cryptographic \emph{authenticity} via per-transaction ed25519 (\C{sign\_transaction(tx\_id, tx\_hash, key)} verified inside \C{verify\_chain}) is Phase 2. \C{:privacy :local} on \C{:llm-call} nodes routes inference to local models (Qwen, DeepSeek, Llama); \C{:visibility :private} on skills enforces never-to-cloud semantics; telemetry exits to an operator-controlled OpenTelemetry collector with no SaaS egress. With the audit handler wrapping the full catalog, regulated deployments stay fully auditable inside the client's VPC with zero third-party exposure.

\textbf{Adoption path.} An existing agent stack migrates additively in three phases: (1) stand up Fact alongside the existing memory system with a datom-emitting interceptor on writes ($\sim$2 days, non-breaking); (2) backfill from mutable state with \C{tx-time}/\C{valid-from} := \C{created\_at} ($\sim$3 days); (3) build \C{as\_of}/\C{history}/\C{branch} as thin functions over the log ($\sim$1 week). Effect, Spec, and Replay then migrate one at a time at the operator's pace; Plan, Txn, REPL land in Phase 2.

\textbf{Relation to neurosymbolic AI.} Persistence is neurosymbolic in a specific technical sense: the substrate is symbolic, the agency is neural, and they couple through three concrete contracts beyond ``storage and audit'' (already named in \sectionref{sec:specs}, \sectionref{sec:plans}, \sectionref{sec:replay}): spec-driven self-healing, plan-as-data with content identity, and replay determinism over neural leaves. The shipped substrate carries accountability, composition, and formal guarantees; the neural layer carries expressiveness. Two items we do \emph{not} claim shipped: a Datalog rule engine (the datom model is Datalog-ready; Phase 1 is comprehensions over the log), and Z3-discharged \C{verify} leaves~\citep{fan2024proofofthought}. Both are adjacent Phase-2 work. This distinguishes Persistence from pure-neural approaches (no guarantees) and pure-symbolic approaches (no expressiveness), and positions it as a candidate substrate for neurosymbolic systems beyond LLM agents.

\section{Conclusion}\label{sec:conclusion}

We presented Persistence, a cognitive-runtime substrate for accountable neurosymbolic agents. v0.1.0a1 + v0.4.0a1 + v0.5.1 (\textbf{931 tests green + 7 xfailed}) ships six modules --- Fact, Effect, Spec, Replay, Plan (minimal), Txn --- demonstrating six substrate-derived capabilities: queryable bitemporal history, counterfactual branching with parent-store isolation, composable policy-gated effects with machine-checkable well-formedness, boundary-checked contracts with LLM-self-healing hints, content-addressed homoiconic plan ASTs with byte-identical round-trip and descendant-propagation identity, and snapshot-isolated transactional shared state with intent-log read-set provenance threaded into the audit chain. Five formal propositions are stated and machine-checked on the shipped artifact; the headline two: well-formedness is decidable in linear time in stack size for fixed catalog by \C{Runtime.is\_well\_formed} and enforced at runtime via \C{Unhandled}; NO-OP replay produces a canonical-hash byte-identical to the factual trajectory's (toy agents with recorded per-step rng-consumption). Two further modules (Plan execution, REPL) are specified, partially registered in the spec registry ahead of the runtime, and scheduled for Phase 2.C / 2.E (2026-Q3). The core claim is that accountability, counterfactual replay, composable safety, compositional skill learning, multi-agent coordination, and live production steering derive from \emph{one} substrate rather than six --- and the six already shipped are properties of the log, checkable against the artifact, not features engineered per agent.

\textbf{Future work.} (1) A persistent-trie \C{Store} making Proposition~\ref{prop:branch}'s $O(\log n)$ branch a shipped property; (2) per-step rng-state recording unblocking byte-identical LLM-trajectory replay; (3) ed25519 per-transaction signing; (4) a Datalog query surface; (5) Z3-discharged \C{verify} leaves following Proof-of-Thought~\citep{fan2024proofofthought}; (6) embodied-agent extensions with partial-observability effects; (7) distributed deployment preserving bitemporal semantics across regions.
